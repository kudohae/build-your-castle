# T47 — [Phase 4] — Push Notifications

## Overview
This task implements all four push notification types for the sand castle game — tide warnings, guestbook arrival alerts, new star rating alerts, and breakwater expiry warnings — using the AIT Push SDK. It also wires up the tide alert scheduler that calculates the 10-minute pre-tide window and uses `setTimeout` for delivery timing.

## Scope
Create `src/ait/push.ts` with four exported notification functions and one scheduler. `sendTideAlert()` sends a push with title "🌊 밀물이 10분 후에 옵니다", body "낮은 곳의 모래성이 잠길 수 있어요. 지금 확인해보세요!", and deeplink `intoss://sand-castle`. `sendGuestbookAlert(authorNickname)` sends title "📝 방명록이 도착했어요", body `${authorNickname}님이 내 성에 메시지를 남겼어요.`. `sendScoreAlert(newScore)` sends title "⭐ 새로운 별점을 받았어요" with body that includes the new score value. `breakwaterExpiryAlert()` sends "🛡 방파제가 3일 후 만료됩니다" to remind players to renew. `scheduleTideAlert(nextHighTide: Date)` computes `alertTime = nextHighTide - 10 minutes`, checks if alertTime is in the future, and uses `setTimeout` to call `sendTideAlert()` at the computed delay. Each function calls `AIT.Push.send()` from the Toss host environment.

## Implementation Details

### AIT Push Type Declaration & Helper

```typescript
// src/ait/push.ts

// AIT SDK type augmentation (see also T41)
// window.AIT.Push.send is already declared in T41's type block.

interface PushPayload {
  title: string;
  body: string;
  deeplink?: string;
}

const GAME_DEEPLINK = 'intoss://sand-castle';
const MAX_BODY_LENGTH = 200; // AIT limit (assumed)

function getAITPush() {
  if (!window.AIT?.Push) {
    throw new Error('AIT Push SDK가 초기화되지 않았습니다.');
  }
  return window.AIT.Push;
}

async function sendPush(payload: PushPayload): Promise<void> {
  const push = getAITPush();

  // Truncate body defensively
  const body = payload.body.length > MAX_BODY_LENGTH
    ? payload.body.slice(0, MAX_BODY_LENGTH - 1) + '…'
    : payload.body;

  try {
    await push.send({ ...payload, body });
  } catch (err: any) {
    // Push failures are non-fatal; log and continue
    console.warn('[push] Failed to send push notification:', err?.message ?? err);
  }
}
```

Push failures are logged but never thrown to the caller — a failed notification must never interrupt gameplay or purchase flow.

### scheduleTideAlert

```typescript
// Active scheduler handle — only one tide alert may be scheduled at a time
let tideAlertTimeoutId: ReturnType<typeof setTimeout> | null = null;

export function scheduleTideAlert(nextHighTide: Date): void {
  // Cancel any existing scheduled alert
  if (tideAlertTimeoutId !== null) {
    clearTimeout(tideAlertTimeoutId);
    tideAlertTimeoutId = null;
  }

  const ALERT_ADVANCE_MS = 10 * 60 * 1000; // 10 minutes
  const alertTime = nextHighTide.getTime() - ALERT_ADVANCE_MS;
  const delay = alertTime - Date.now();

  if (delay <= 0) {
    // Tide is less than 10 min away or already passed — send immediately
    sendTideAlert();
    return;
  }

  tideAlertTimeoutId = setTimeout(async () => {
    tideAlertTimeoutId = null;
    await sendTideAlert();
  }, delay);

  console.debug(
    `[push] Tide alert scheduled in ${Math.round(delay / 1000)}s ` +
    `(at ${new Date(alertTime).toLocaleTimeString('ko-KR')})`
  );
}

export function cancelTideAlert(): void {
  if (tideAlertTimeoutId !== null) {
    clearTimeout(tideAlertTimeoutId);
    tideAlertTimeoutId = null;
  }
}
```

## Key Constraints
- All push functions are fire-and-forget from the game's perspective — they must never throw to callers; `sendPush` catches and logs internally.
- Only one `tideAlertTimeoutId` may be active at a time — `scheduleTideAlert` cancels the previous one before scheduling a new one.
- If `nextHighTide` is less than 10 minutes in the future when `scheduleTideAlert` is called, the alert fires immediately (delay ≤ 0 case).
- The AIT Push SDK is expected to handle user permission checks — this module does not request push permission explicitly; that is handled at app startup by the Toss host.
- `authorNickname` in `sendGuestbookAlert` is sanitized to remove `<>` characters before interpolation into the body string.
- Push notifications do not carry sensitive data (no block coordinates, no receipt tokens) in the body or deeplink.
- `cancelTideAlert()` must be called when the player's session ends or the app is closed to prevent stale alerts.
- Cross-user push targeting (sending to the plot owner's device) is a known limitation — see scope note above about the AIT session model.
