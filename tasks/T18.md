# T18 — [Phase 2] — Pixel-aware block renderer

## Overview
This task upgrades the block rendering pipeline from solid `fillRect` calls to a full per-pixel rendering loop driven by each block's `pixelMask` Float32Array. This is the visual foundation for all carving effects — without pixel-level rendering, carved shapes appear as solid rectangles regardless of mask data. Every subsequent carving and visual feature depends on this renderer being correct and performant.

## Scope
Modify `src/canvas/renderer.ts` to replace the existing solid `fillRect` block-drawing path with a pixel-accurate rendering loop. For every block in the scene, iterate all 1024 cells (32 columns × 32 rows) of its `pixelMask`. A cell value of `1.0` means the pixel is fully solid and should be drawn with the block's full base color. A value of `0.5` means the pixel is engraved and should be drawn darkened by 35% (multiply each RGB channel by `0.65`). A value of `0.0` means the pixel is transparent and should be skipped entirely. To avoid 1,024 individual `fillRect` calls per block (which would be catastrophic for frame rate), all pixels are written into a shared `ImageData` buffer that is flushed with a single `putImageData` call per block. Implement a standalone `darken(hexColor: string, amount: number): string` helper that converts a hex color string to RGB, scales each channel by `(1 - amount)`, and returns a new hex string. `amount=0.35` produces the engraving shade. The renderer must continue to support block transforms (x/y offset) and camera pan, compositing blocks in painter's order (background first).

## Implementation Details

### darken() helper
Converts a CSS hex color to RGB, scales each channel, and returns a new hex string. Used for both engraved pixels (amount=0.35) and any future tint effects.

```typescript
// src/canvas/renderer.ts

export function darken(hexColor: string, amount: number): string {
  // Strip leading '#' if present
  const hex = hexColor.replace(/^#/, '');
  const r = parseInt(hex.slice(0, 2), 16);
  const g = parseInt(hex.slice(2, 4), 16);
  const b = parseInt(hex.slice(4, 6), 16);

  const scale = 1 - amount;
  const nr = Math.round(Math.min(255, r * scale));
  const ng = Math.round(Math.min(255, g * scale));
  const nb = Math.round(Math.min(255, b * scale));

  return `#${nr.toString(16).padStart(2, '0')}${ng.toString(16).padStart(2, '0')}${nb.toString(16).padStart(2, '0')}`;
}

// Pre-parse a hex color string into an [r, g, b] triple for fast pixel writes
function hexToRgb(hex: string): [number, number, number] {
  const h = hex.replace(/^#/, '');
  return [
    parseInt(h.slice(0, 2), 16),
    parseInt(h.slice(2, 4), 16),
    parseInt(h.slice(4, 6), 16),
  ];
}
```

### Per-block ImageData pixel loop
Each block gets its own 32×32 `ImageData`. We pre-parse the block's base color and its darkened variant once, then write directly into the buffer's `Uint8ClampedArray`, avoiding any string operations in the hot loop.

```typescript
const BLOCK_SIZE = 32;
const ENGRAVE_DARKEN = 0.35;

function renderBlock(
  ctx: CanvasRenderingContext2D,
  block: Block,
  screenX: number,
  screenY: number
): void {
  const imageData = ctx.createImageData(BLOCK_SIZE, BLOCK_SIZE);
  const data = imageData.data; // Uint8ClampedArray, 4 bytes per pixel (RGBA)

  const [br, bg, bb] = hexToRgb(block.color);
  const scale = 1 - ENGRAVE_DARKEN; // 0.65
  const [dr, dg, db] = [
    Math.round(br * scale),
    Math.round(bg * scale),
    Math.round(bb * scale),
  ];

  for (let py = 0; py < BLOCK_SIZE; py++) {
    for (let px = 0; px < BLOCK_SIZE; px++) {
      const maskIndex = py * BLOCK_SIZE + px;
      const maskValue = block.pixelMask[maskIndex];

      if (maskValue === 0.0) continue; // transparent — leave alpha=0

      const pixelIndex = maskIndex * 4;

      if (maskValue === 1.0) {
        data[pixelIndex]     = br;
        data[pixelIndex + 1] = bg;
        data[pixelIndex + 2] = bb;
        data[pixelIndex + 3] = 255;
      } else {
        // maskValue === 0.5 → engraved dark
        data[pixelIndex]     = dr;
        data[pixelIndex + 1] = dg;
        data[pixelIndex + 2] = db;
        data[pixelIndex + 3] = 255;
      }
    }
  }

  ctx.putImageData(imageData, Math.round(screenX), Math.round(screenY));
}
```

### Intermediate mask values
Although the primary mask values are `0.0`, `0.5`, and `1.0`, the renderer should gracefully handle any float in `[0, 1]`. Treat values in `(0, 0.75)` as engraved (dark) and values in `[0.75, 1]` as full. This guards against rounding drift during mask serialization.

```typescript
// Generalised threshold logic (replace the if/else above if needed):
if (maskValue < 0.01) {
  // transparent — skip
} else if (maskValue < 0.75) {
  // engraved
  data[pixelIndex]     = dr;
  data[pixelIndex + 1] = dg;
  data[pixelIndex + 2] = db;
  data[pixelIndex + 3] = 255;
} else {
  // solid
  data[pixelIndex]     = br;
  data[pixelIndex + 1] = bg;
  data[pixelIndex + 2] = bb;
  data[pixelIndex + 3] = 255;
}
```

### Performance considerations
- `ctx.createImageData(32, 32)` allocates a new buffer each call. For scenes with hundreds of blocks, consider maintaining a single reusable `ImageData` (32×32) and clearing it manually between blocks.
- Pre-compute `[dr, dg, db]` once per block, not per-pixel.
- Avoid calling `darken()` inside the render loop; it is only used for one-off utility calls (e.g., HUD colors).
- Profile with the browser's Performance tab; at 60 fps with 200 blocks the pixel loop budget is ~0.08 ms per block.

## Key Constraints
- `pixelMask` is a `Float32Array` of exactly 1024 elements; never access index >= 1024.
- Pixel coordinates are row-major: index = `py * 32 + px`.
- Camera pan must still be applied — `screenX/screenY` are computed before calling `renderBlock`.
- The `darken` helper is exported for reuse in T24 (stability HUD color gradient) and T26 (water tint).
- Do not use `ctx.globalAlpha` to fake engraving; the mask pixel write approach must be used so that composite operations work correctly over background layers.
- `putImageData` ignores the current canvas transform, so coordinates passed must already be in canvas pixel space (after camera offset).

## Acceptance Criteria
- [ ] A block with all-1.0 mask renders as a solid 32×32 rectangle of the block's base color.
- [ ] A block with all-0.0 mask renders as fully transparent (no pixels drawn).
- [ ] A block with a 0.5-valued circular region in the center renders that region 35% darker than the surrounding solid pixels.
- [ ] `darken('#ffcc00', 0.35)` returns `'#a68500'` (or within ±1 per channel due to rounding).
- [ ] Frame rate with 200 blocks on screen stays above 55 fps on a mid-range mobile device (measured via `performance.now()` delta in `renderFrame`).
- [ ] Blocks at non-integer `screenX/screenY` values (due to camera panning sub-pixel) render without visible tearing (coordinates are rounded before `putImageData`).
- [ ] No regression: existing block placement, camera pan, and zone boundaries still render correctly.
- [ ] `renderBlock` is the single canonical pixel-drawing path; no residual `fillRect` calls exist for block bodies.
