# T48 — [Phase 4] — Erosion Edge Function

## Overview
This task implements the server-side erosion mechanic as a Supabase Edge Function running in Deno. Triggered daily by a `pg_cron` job, it identifies plots that have been inactive for 7+ days and no longer have active breakwater protection, then randomly removes up to 5 exposed blocks from each such plot, simulating natural erosion of abandoned castles.

## Scope
Create `supabase/functions/erosion/index.ts` as a Deno-based Supabase Edge Function. On invocation: (1) query the `plots` table for all rows where `last_visited_at < NOW() - INTERVAL '7 days'`; (2) for each such plot, query the `breakwaters` table for an active row (`expires_at > NOW()`); (3) if no active breakwater, query the `blocks` table for that plot, identify "exposed" blocks (blocks that have no neighbor block directly above them — i.e., the `y-1` cell is empty — or have no neighbor directly to their left and right), randomly select up to 5 of these exposed blocks, and `DELETE` them from the `blocks` table; (4) update `plots.erosion_last_run = NOW()` for each processed plot; (5) return a JSON summary `{ affectedPlots: number, totalBlocksRemoved: number }`. Uses the Supabase admin client initialized with the `SUPABASE_SERVICE_ROLE_KEY` environment variable. The `pg_cron` job is defined separately in the migration file.

## Output Files
- `supabase/functions/erosion/index.ts` — new Deno Edge Function file

## Depends On
- T10 — establishes the `blocks` table schema with `x`, `y`, `plot_id`, `block_type` columns used by the erosion query

## Implementation Details

### Deno Edge Function Bootstrap

```typescript
// supabase/functions/erosion/index.ts

import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const SUPABASE_URL = Deno.env.get('SUPABASE_URL')!;
const SERVICE_ROLE_KEY = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!;
const INACTIVITY_DAYS = 7;
const MAX_BLOCKS_PER_PLOT = 5;

const adminClient = createClient(SUPABASE_URL, SERVICE_ROLE_KEY, {
  auth: { persistSession: false },
});

Deno.serve(async (req: Request) => {
  // Only allow POST (from pg_cron or manual trigger)
  if (req.method !== 'POST') {
    return new Response('Method Not Allowed', { status: 405 });
  }

  // Optional: verify a shared secret header to prevent unauthorized triggers
  const authHeader = req.headers.get('Authorization');
  const expectedSecret = Deno.env.get('EROSION_SECRET');
  if (expectedSecret && authHeader !== `Bearer ${expectedSecret}`) {
    return new Response('Unauthorized', { status: 401 });
  }

  const summary = await runErosion();

  return new Response(JSON.stringify(summary), {
    headers: { 'Content-Type': 'application/json' },
    status: 200,
  });
});
```

### fetchInactivePlots

```typescript
interface PlotRow {
  id: string;
  owner_id: string;
}

async function fetchInactivePlots(): Promise<PlotRow[]> {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - INACTIVITY_DAYS);

  const { data, error } = await adminClient
    .from('plots')
    .select('id, owner_id')
    .lt('last_visited_at', cutoff.toISOString());

  if (error) {
    console.error('[erosion] Failed to fetch inactive plots:', error.message);
    return [];
  }

  return data ?? [];
}
```

### checkActiveBreakwater

```typescript
async function checkActiveBreakwater(plotId: string): Promise<boolean> {
  const { data } = await adminClient
    .from('breakwaters')
    .select('id')
    .eq('plot_id', plotId)
    .gt('expires_at', new Date().toISOString())
    .maybeSingle();

  return data !== null;
}
```

Returns `true` if there is an active breakwater — the plot is protected and erosion is skipped.

### fetchExposedBlocks

```typescript
interface BlockRow {
  id: string;
  x: number;
  y: number;
  block_type: string;
}

async function fetchExposedBlocks(plotId: string): Promise<BlockRow[]> {
  // Fetch all blocks for the plot
  const { data: allBlocks, error } = await adminClient
    .from('blocks')
    .select('id, x, y, block_type')
    .eq('plot_id', plotId);

  if (error || !allBlocks) return [];

  // Build a Set of occupied positions for O(1) lookup
  const occupied = new Set<string>(
    allBlocks.map((b) => `${b.x},${b.y}`)
  );

  // "Exposed" = no block directly above (y - BLOCK_SIZE)
  const BLOCK_SIZE = 32;
  const exposed = allBlocks.filter((block) => {
    const aboveKey = `${block.x},${block.y - BLOCK_SIZE}`;
    return !occupied.has(aboveKey);
  });

  return exposed;
}
```

A block is "exposed" if its `(x, y - 32)` neighbor slot is empty. This models surface blocks that are open to the sky — the most realistic targets for weathering.

### selectRandomBlocks & deleteBlocks

```typescript
function selectRandom<T>(arr: T[], count: number): T[] {
  // Fisher-Yates partial shuffle
  const copy = [...arr];
  const result: T[] = [];
  for (let i = 0; i < Math.min(count, copy.length); i++) {
    const j = i + Math.floor(Math.random() * (copy.length - i));
    [copy[i], copy[j]] = [copy[j], copy[i]];
    result.push(copy[i]);
  }
  return result;
}

async function deleteBlocks(blockIds: string[]): Promise<number> {
  if (blockIds.length === 0) return 0;

  const { error, count } = await adminClient
    .from('blocks')
    .delete({ count: 'exact' })
    .in('id', blockIds);

  if (error) {
    console.error('[erosion] Block deletion failed:', error.message);
    return 0;
  }

  return count ?? 0;
}
```

### Main Erosion Runner

```typescript
interface ErosionSummary {
  affectedPlots: number;
  totalBlocksRemoved: number;
  processedAt: string;
}

async function runErosion(): Promise<ErosionSummary> {
  const inactivePlots = await fetchInactivePlots();
  let affectedPlots = 0;
  let totalBlocksRemoved = 0;

  for (const plot of inactivePlots) {
    const hasBreakwater = await checkActiveBreakwater(plot.id);
    if (hasBreakwater) {
      console.log(`[erosion] Plot ${plot.id} is protected by breakwater — skipping.`);
      continue;
    }

    const exposedBlocks = await fetchExposedBlocks(plot.id);
    if (exposedBlocks.length === 0) continue;

    const targets = selectRandom(exposedBlocks, MAX_BLOCKS_PER_PLOT);
    const removed = await deleteBlocks(targets.map((b) => b.id));

    if (removed > 0) {
      affectedPlots++;
      totalBlocksRemoved += removed;

      // Update erosion_last_run timestamp
      await adminClient
        .from('plots')
        .update({ erosion_last_run: new Date().toISOString() })
        .eq('id', plot.id);

      console.log(
        `[erosion] Plot ${plot.id}: removed ${removed} blocks (` +
        `${exposedBlocks.length} exposed total).`
      );
    }
  }

  return {
    affectedPlots,
    totalBlocksRemoved,
    processedAt: new Date().toISOString(),
  };
}
```

### pg_cron Job Definition (Migration File)

```sql

-- Schedule erosion function to run daily at 03:00 UTC
SELECT cron.schedule(
  'daily-erosion',
  '0 3 * * *',
  $$
  SELECT net.http_post(
    url    := current_setting('app.supabase_url') || '/functions/v1/erosion',
    headers := '{"Authorization": "Bearer ' || current_setting('app.erosion_secret') || '"}',
    body   := '{}'
  )
  $$
);
```

The `pg_net` extension must be enabled. The `app.supabase_url` and `app.erosion_secret` settings are set via `ALTER DATABASE ... SET`.

## Key Constraints
- The edge function uses `SUPABASE_SERVICE_ROLE_KEY` to bypass RLS — this is necessary for admin-level cross-user operations. The key must never be exposed to client-side code.
- A maximum of 5 blocks is removed per plot per day — this is a soft erosion, not an immediate destruction.
- Only "exposed" blocks (top surface, no neighbor above) are eligible for removal — buried blocks are protected by surrounding material, modeling realistic weathering.
- The function is idempotent with respect to a single run: deleting up to 5 blocks and then being called again in the same day will find fewer exposed blocks (or none on an empty plot) and simply remove fewer or none.
- The `EROSION_SECRET` header check is optional but strongly recommended to prevent unauthorized function invocations from the public internet.
- `last_visited_at` must be updated on every legitimate player visit — the client is responsible for this; the edge function only reads the timestamp.
- The Deno runtime import uses `esm.sh` for the Supabase client — do not use npm-style imports in Deno edge functions.

## Acceptance Criteria
- [ ] The function returns `{ affectedPlots, totalBlocksRemoved, processedAt }` JSON on every successful invocation.
- [ ] Plots with `last_visited_at` within the last 7 days are NOT processed.
- [ ] Plots with an active `breakwaters` row (`expires_at > NOW()`) are skipped with a log message.
- [ ] For an eligible plot with 20 exposed blocks, exactly 5 are randomly selected and deleted.
- [ ] For an eligible plot with 3 exposed blocks, all 3 are deleted (fewer than max).
- [ ] For a plot with no blocks at all, the function skips deletion without error.
- [ ] `plots.erosion_last_run` is updated to `NOW()` only for plots that had blocks actually deleted.
- [ ] An unauthorized request (missing or wrong `Authorization` header) returns HTTP 401.
