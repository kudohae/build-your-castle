# T26 — [Phase 2] — Tide UI & HUD countdown

## Overview
This task makes the tide system visible to the player through two complementary elements: a translucent water overlay rendered directly on the game canvas, and a tide countdown timer in the top HUD strip. The animated water gives the castle zone an immersive environmental feel, while the countdown tells players exactly how long they have before the tide changes, enabling strategic decision-making about when to place wet-sensitive materials.

## Scope
Modify `src/canvas/renderer.ts` to draw a semi-transparent blue rectangle (`rgba(100, 180, 255, 0.3)`) from the current `waterY` down to `GROUND_Y`, spanning the SHORELINE and CASTLE_ZONE x-range (`x: 800` to the right edge of the canvas). Apply a gentle wave animation by oscillating the water surface by ±3px using `Math.sin(Date.now() / 600)` — a slow sine with a ~3.8-second period. This oscillation is purely visual: it affects only the drawn rectangle top edge, not the actual `waterY` used for physics or block conversion checks. Modify `src/ui/hud.ts` to add a tide countdown display in the top-right corner of the HUD strip. Using `getTideInfo(new Date())` from T25, determine whether the next event is high tide or low tide and compute the MM:SS remaining. Display `"다음 밀물까지 MM:SS"` (next high tide) or `"다음 썰물까지 MM:SS"` (next low tide). The countdown text updates every frame (the HUD renders every frame anyway). Also draw a small tide arrow icon (↑ rising, ↓ falling) beside the countdown in the HUD color scheme.

## Output Files
- `src/canvas/renderer.ts (modify)` — adds `renderWaterOverlay(ctx, cameraX)` called each frame after background, before blocks
- `src/ui/hud.ts (modify)` — adds `renderTideCountdown(ctx, canvasWidth)` called inside the main HUD render function

## Depends On
- T25 — provides `getTideLevel`, `getWaterY`, `getTideInfo`, and `TideInfo` used in both the water overlay and the HUD countdown
- T24 — the HUD structure extended here; the countdown is positioned relative to the HUD strip layout established by T24's stability gauge addition

## Implementation Details

### Water overlay rendering

The water overlay is drawn after the background layer and before block rendering, so blocks appear to sit inside/under the water rather than floating on top of it.

```typescript
// src/canvas/renderer.ts (addition)

import { getTideLevel, getWaterY } from '../systems/weather';
import {
  SHORELINE_START_X,
  GROUND_Y,
} from '../types/zones';

const WATER_WAVE_AMPLITUDE = 3; // px
const WATER_WAVE_PERIOD = 600;  // ms per radian (≈3.8s full cycle)

function renderWaterOverlay(
  ctx: CanvasRenderingContext2D,
  canvasWidth: number,
  cameraX: number
): void {
  const now = new Date();
  const tideLevel = getTideLevel(now);
  const waterY = getWaterY(tideLevel);

  // Visual wave oscillation (purely cosmetic)
  const waveOffset = Math.sin(Date.now() / WATER_WAVE_PERIOD) * WATER_WAVE_AMPLITUDE;
  const visualWaterY = waterY + waveOffset;

  // Convert world x to screen x accounting for camera
  const shorelineScreenX = SHORELINE_START_X - cameraX;
  const waterWidth = canvasWidth - shorelineScreenX; // water extends to right edge

  if (waterWidth <= 0) return; // camera panned past water zone

  const waterHeight = GROUND_Y - visualWaterY;
  if (waterHeight <= 0) return; // tide so low it's below ground — skip

  // Main water fill
  ctx.fillStyle = 'rgba(100, 180, 255, 0.3)';
  ctx.fillRect(shorelineScreenX, visualWaterY, waterWidth, waterHeight);

  // Subtle water surface gradient (lighter at top, deeper below)
  const gradient = ctx.createLinearGradient(0, visualWaterY, 0, GROUND_Y);
  gradient.addColorStop(0, 'rgba(180, 220, 255, 0.15)');
  gradient.addColorStop(1, 'rgba(40, 100, 200, 0.20)');
  ctx.fillStyle = gradient;
  ctx.fillRect(shorelineScreenX, visualWaterY, waterWidth, waterHeight);

  // Wave crests: two sine-wave lines at the surface
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.5)';
  ctx.lineWidth = 1.5;
  ctx.beginPath();

  const t = Date.now() / 400; // scrolling phase
  for (let sx = shorelineScreenX; sx <= canvasWidth; sx += 2) {
    const wx = sx + cameraX; // world x for consistent wave position
    const wy = visualWaterY + Math.sin(wx / 40 + t) * 2;
    if (sx === shorelineScreenX) {
      ctx.moveTo(sx, wy);
    } else {
      ctx.lineTo(sx, wy);
    }
  }
  ctx.stroke();
}
```


### Tide countdown in HUD

```typescript
// src/ui/hud.ts (addition)

import { getTideInfo } from '../systems/weather';

function formatCountdown(ms: number): string {
  const totalSeconds = Math.max(0, Math.floor(ms / 1000));
  const minutes = Math.floor(totalSeconds / 60);
  const seconds = totalSeconds % 60;
  return `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`;
}

function renderTideCountdown(
  ctx: CanvasRenderingContext2D,
  canvasWidth: number
): void {
  const info = getTideInfo(new Date());
  const label = info.isRising ? '다음 썰물까지' : '다음 밀물까지';
  const countdown = formatCountdown(info.nextChangeMs);
  const arrowChar = info.isRising ? '↑' : '↓';
  const arrowColor = info.isRising ? '#64b4ff' : '#ffb340';

  const textX = canvasWidth - 16; // right-aligned
  const textY = 20;               // top HUD strip baseline

  ctx.save();

  // Arrow icon
  ctx.fillStyle = arrowColor;
  ctx.font = 'bold 14px "Apple SD Gothic Neo", sans-serif';
  ctx.textAlign = 'right';
  ctx.fillText(arrowChar, textX - 90, textY);

  // Label text
  ctx.fillStyle = 'rgba(255,255,255,0.85)';
  ctx.font = '11px "Apple SD Gothic Neo", sans-serif';
  ctx.textAlign = 'right';
  ctx.fillText(label, textX, textY - 6);

  // Countdown value
  ctx.fillStyle = '#ffffff';
  ctx.font = 'bold 14px "SF Mono", "Courier New", monospace';
  ctx.fillText(countdown, textX, textY + 8);

  ctx.restore();
}
```

### Integration into HUD render

```typescript
// src/ui/hud.ts — inside the main renderHUD(ctx, canvas, ...) function
// After existing top strip elements:
renderTideCountdown(ctx, canvas.width);
```

### Tide level visual indicator
Optionally draw a small tide-level bar beside the countdown to give players an analogue sense of the cycle position:

```typescript
function renderTideLevelBar(
  ctx: CanvasRenderingContext2D,
  tideLevel: number,
  x: number,
  y: number
): void {
  const barH = 24;
  const barW = 6;

  // Background
  ctx.fillStyle = 'rgba(0,0,0,0.3)';
  ctx.fillRect(x, y, barW, barH);

  // Fill from bottom
  const fillH = tideLevel * barH;
  ctx.fillStyle = '#64b4ff';
  ctx.fillRect(x, y + barH - fillH, barW, fillH);
}
```

## Key Constraints
- The `waveOffset` (±3px) must only affect the visual rectangle top, never the `waterY` value used in physics calculations or block conversion (`applyTideEffect` uses the unmodified `getWaterY` result).
- `renderWaterOverlay` must be called before block rendering in the frame order — blocks should appear to sit inside the water body, not float over it.
- The countdown label language must match the tide direction: "밀물" (incoming tide / high tide approach), "썰물" (outgoing tide / low tide approach).
- Do not cache `getTideInfo` across frames — call it fresh each render to keep the countdown accurate.
- The wave crest rendering uses a scrolling phase (`Date.now() / 400`) for motion; this does not need to be synchronized with `waveOffset`.
- If `waterWidth <= 0` (camera fully panned left of the shoreline), skip rendering to avoid negative rect dimensions.
