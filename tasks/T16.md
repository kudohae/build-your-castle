# T16 — [Phase 2] — PixelMask data structure & utilities

## Overview
The pixel mask system enables sub-block carving — instead of removing entire blocks, players can engrave patterns, cut channels, and sculpt fine details into their sand castle. This task defines the core data structure (`Float32Array` of 1024 values representing a 32×32 per-block pixel grid), implements all manipulation primitives, and provides serialization to/from base64 for Supabase storage. Every carving and rendering operation in Phase 2 depends on this foundation.

## Scope
Implement `src/carving/pixelMask.ts` with the `PixelMask` type alias, all accessor/mutator functions, and serialization utilities. `PixelMask` is a `Float32Array` of length 1024 (32×32 pixels, row-major). Values encode material state: `1.0` = full original material, `0.5` = engraved/indented (rendered darker), `0.0` = fully removed (transparent). Export `createFullMask()`, `getPixel(mask, px, py)`, `setPixel(mask, px, py, value)`, `serializeMask(mask): string`, `deserializeMask(data: string): PixelMask`. Also export `MASK_SIZE = 32` and `MASK_LENGTH = 1024`.

## Output Files
- `src/carving/pixelMask.ts` — PixelMask type, constants, all accessor/mutator/serialization functions

## Implementation Details

### Type definition and constants

```typescript
// src/carving/pixelMask.ts

/** A per-block pixel grid. Float32Array of length MASK_LENGTH (32×32). */
export type PixelMask = Float32Array;

/** Pixel dimension of one side of the mask grid. */
export const MASK_SIZE = 32;

/** Total number of float values in a mask (MASK_SIZE²). */
export const MASK_LENGTH = MASK_SIZE * MASK_SIZE; // 1024

/**
 * Pixel value semantics:
 *   1.0 → full material (opaque, original color)
 *   0.5 → engraved / indented (render at ~60% brightness)
 *   0.0 → removed (fully transparent / no collision contribution)
 */
```

### createFullMask

Creates a new mask with all pixels set to `1.0` — appropriate for any newly placed block.

```typescript
export function createFullMask(): PixelMask {
  return new Float32Array(MASK_LENGTH).fill(1.0);
}
```

### getPixel

Returns the value at pixel `(px, py)` within the 32×32 grid. Returns `0` for out-of-bounds coordinates rather than throwing, to simplify brush boundary handling in the carving tool.

```typescript
export function getPixel(mask: PixelMask, px: number, py: number): number {
  if (px < 0 || px >= MASK_SIZE || py < 0 || py >= MASK_SIZE) return 0;
  return mask[py * MASK_SIZE + px];
}
```

### setPixel

Clamps the value to `[0, 1]` before writing. Does nothing for out-of-bounds coordinates.

```typescript
export function setPixel(
  mask: PixelMask,
  px: number,
  py: number,
  value: number,
): void {
  if (px < 0 || px >= MASK_SIZE || py < 0 || py >= MASK_SIZE) return;
  mask[py * MASK_SIZE + px] = Math.max(0, Math.min(1, value));
}
```

### fillRect — batch mutator

Fills a rectangular region of the mask with a single value. Used by carving brushes for efficiency.

```typescript
export function fillRect(
  mask: PixelMask,
  px: number,
  py: number,
  width: number,
  height: number,
  value: number,
): void {
  const clampedValue = Math.max(0, Math.min(1, value));
  for (let row = py; row < py + height; row++) {
    for (let col = px; col < px + width; col++) {
      if (col < 0 || col >= MASK_SIZE || row < 0 || row >= MASK_SIZE) continue;
      mask[row * MASK_SIZE + col] = clampedValue;
    }
  }
}
```

### serializeMask

Converts the Float32Array to a base64 string for Supabase text column storage. The raw buffer is 4096 bytes (1024 floats × 4 bytes each), which base64-encodes to ~5460 characters — well within PostgreSQL `text` column limits.

```typescript
export function serializeMask(mask: PixelMask): string {
  // Use Buffer in Node.js environments (test runner); btoa + Uint8Array in browser
  const bytes = new Uint8Array(mask.buffer);

  if (typeof Buffer !== 'undefined') {
    // Node.js / Vite SSR
    return Buffer.from(bytes).toString('base64');
  }

  // Browser: encode byte-by-byte
  let binary = '';
  for (let i = 0; i < bytes.length; i++) {
    binary += String.fromCharCode(bytes[i]);
  }
  return btoa(binary);
}
```

### deserializeMask

Reconstructs a `Float32Array` from a base64 string. Validates the decoded length before constructing the view.

```typescript
export function deserializeMask(data: string): PixelMask {
  let bytes: Uint8Array;

  if (typeof Buffer !== 'undefined') {
    // Node.js
    bytes = new Uint8Array(Buffer.from(data, 'base64'));
  } else {
    // Browser
    const binary = atob(data);
    bytes = new Uint8Array(binary.length);
    for (let i = 0; i < binary.length; i++) {
      bytes[i] = binary.charCodeAt(i);
    }
  }

  const expectedBytes = MASK_LENGTH * Float32Array.BYTES_PER_ELEMENT; // 4096
  if (bytes.length !== expectedBytes) {
    console.warn(
      `[PixelMask] deserializeMask: expected ${expectedBytes} bytes, got ${bytes.length}. Returning full mask.`,
    );
    return createFullMask();
  }

  // Copy bytes into a correctly aligned buffer (avoids alignment issues from atob)
  const aligned = new ArrayBuffer(expectedBytes);
  new Uint8Array(aligned).set(bytes);
  return new Float32Array(aligned);
}
```

### Utility: isMaskFull

A quick check to skip mask rendering overhead for blocks that have never been carved.

```typescript
export function isMaskFull(mask: PixelMask): boolean {
  for (let i = 0; i < MASK_LENGTH; i++) {
    if (mask[i] < 1.0) return false;
  }
  return true;
}
```

## Key Constraints
- `PixelMask` is a `Float32Array` (not `Uint8Array`) because carving operations produce intermediate values like `0.5` that require fractional precision.
- The row-major index formula `py * MASK_SIZE + px` must be used consistently — using `px * MASK_SIZE + py` (column-major) would silently transpose all masks.
- `deserializeMask` must handle buffer alignment carefully — slicing `Uint8Array` bytes into `Float32Array.buffer` directly can fail if the source buffer is not 4-byte aligned. The copy-to-`ArrayBuffer` step in the implementation above is mandatory.
- `serializeMask`/`deserializeMask` must be round-trip lossless — a mask serialized and deserialized must be bit-for-bit identical to the original. Verify this with a float precision test.
- Do not import from any other project module here — `pixelMask.ts` has no dependencies and must remain importable from any context including server-side migration scripts.
- `MASK_SIZE = 32` must match `BLOCK_SIZE` from T01. If block size ever changes, all stored masks would need migration.

## Acceptance Criteria
- [ ] `createFullMask()` returns a Float32Array of length 1024 with all values equal to `1.0`.
- [ ] `getPixel(mask, -1, 0)` returns `0` without throwing.
- [ ] `setPixel(mask, 0, 0, 1.5)` clamps to `1.0`; `setPixel(mask, 0, 0, -0.3)` clamps to `0.0`.
- [ ] `getPixel(mask, px, py)` returns the value previously set by `setPixel(mask, px, py, value)`.
- [ ] `deserializeMask(serializeMask(mask))` produces a Float32Array bit-for-bit identical to `mask`.
- [ ] `isMaskFull(createFullMask())` returns `true`.
- [ ] `isMaskFull(mask)` returns `false` after any `setPixel` call with value < `1.0`.
- [ ] TypeScript compiles with zero errors; no `any` types.
