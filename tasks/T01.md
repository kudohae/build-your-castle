# T01 — [Phase 1] — Project constants & block types

## Overview
This task establishes the foundational type system and spatial constants that every other module depends on. It defines the physical properties of all block materials, the geographic layout of the three game zones, and the core `Block` data interface that flows through physics, rendering, and persistence layers.

## Scope
Define and export all shared constants and types used across the entire codebase. This includes `BLOCK_SIZE` (the universal 32px tile dimension), the `BLOCK_PHYSICS` lookup map keyed by `BlockType` with per-material mass, friction, restitution, and color values, and the `MAP_ZONES` array describing each zone's horizontal extent and background colors. Also define the `Block` interface (id, type, x, y, plotId, isWet, health — with `pixelMask` deferred to T17), the `BlockType` union type, the `ZoneId` union type, and the `getZoneAtX(worldX: number)` utility that returns the enclosing zone for any world x-coordinate. Ground layout constants `GROUND_Y` and `MAP_HEIGHT` are also exported here.

## Output Files
- `src/constants/blocks.ts` — block type union, Block interface, BLOCK_SIZE, and BLOCK_PHYSICS lookup table
- `src/constants/maps.ts` — MAP_ZONES array, ZoneId type, GROUND_Y, MAP_HEIGHT, and getZoneAtX helper

## Depends On
- none

## Implementation Details

### Block types and physics properties

The `BlockType` union is a string literal union covering every material in the game. The `BLOCK_PHYSICS` record maps each type to its physical simulation parameters plus the CSS hex color used by the renderer when no pixel-mask texture is present.

```typescript
// src/constants/blocks.ts

export const BLOCK_SIZE = 32; // pixels, used everywhere

export type BlockType =
  | 'dry_sand'
  | 'wet_sand'
  | 'wood_plank'
  | 'stone'
  | 'shell';

export interface BlockPhysics {
  mass: number;
  friction: number;
  restitution: number;
  color: string; // CSS hex
}

export const BLOCK_PHYSICS: Record<BlockType, BlockPhysics> = {
  dry_sand:  { mass: 1.0, friction: 0.4, restitution: 0.10, color: '#E8C87A' },
  wet_sand:  { mass: 2.5, friction: 0.9, restitution: 0.05, color: '#B8956A' },
  wood_plank:{ mass: 1.5, friction: 0.6, restitution: 0.20, color: '#8B6914' },
  stone:     { mass: 5.0, friction: 0.8, restitution: 0.00, color: '#8A8A8A' },
  shell:     { mass: 0.5, friction: 0.3, restitution: 0.30, color: '#F0E0D0' },
};

export interface Block {
  id: string;          // UUID v4
  type: BlockType;
  x: number;           // world-space left edge in pixels
  y: number;           // world-space top edge in pixels
  plotId: string;      // FK → plots.id; empty string for unclaimed blocks
  isWet: boolean;      // true while inside SHORELINE tide range or after bucket soak
  health: number;      // 0–100; blocks are destroyed at 0
  // pixelMask: PixelMask  ← added in T17
}
```

### Zone constants and helper

`MAP_ZONES` is ordered left-to-right. `CASTLE_ZONE` uses `width: Infinity` because the map can grow rightward as new plots are allocated. `getZoneAtX` performs a simple sequential scan; with only three zones this is negligible.

```typescript
// src/constants/maps.ts

export const GROUND_Y = 520;   // y-coordinate of the top surface of the static ground body
export const MAP_HEIGHT = 600; // total canvas world height in pixels

export type ZoneId = 'BEACH' | 'SHORELINE' | 'CASTLE_ZONE';

export interface ZoneDefinition {
  id: ZoneId;
  label: string;       // Korean display name
  startX: number;
  width: number;       // Infinity for CASTLE_ZONE
  bgColor: string;     // sky / water background fill
  groundColor: string; // ground surface color
}

export const MAP_ZONES: ZoneDefinition[] = [
  {
    id: 'BEACH',
    label: '백사장',
    startX: 0,
    width: 800,
    bgColor: '#F5E6C8',
    groundColor: '#E8C87A',
  },
  {
    id: 'SHORELINE',
    label: '해안',
    startX: 800,
    width: 400,
    bgColor: '#B8DFF0',
    groundColor: '#7EC8E3',
  },
  {
    id: 'CASTLE_ZONE',
    label: '성 구역',
    startX: 1200,
    width: Infinity,
    bgColor: '#D4E8C2',
    groundColor: '#C8A96E',
  },
];

/**
 * Returns the ZoneDefinition that contains the given world x-coordinate.
 * Falls back to CASTLE_ZONE for any x beyond the defined zones.
 */
export function getZoneAtX(worldX: number): ZoneDefinition {
  for (const zone of MAP_ZONES) {
    const endX = zone.startX + zone.width;
    if (worldX >= zone.startX && worldX < endX) {
      return zone;
    }
  }
  // Should not be reached given Infinity width of CASTLE_ZONE
  return MAP_ZONES[MAP_ZONES.length - 1];
}
```

## Key Constraints
- `BLOCK_SIZE` must be exactly 32 throughout the entire codebase — never reference a raw `32` literal outside this file.
- `GROUND_Y` is the top surface of the static physics ground body; Matter.js body center is at `GROUND_Y + 10` (see T07).
- `BlockType` must be exhaustive — adding a new material requires updating `BLOCK_PHYSICS`, `BlockType`, and potentially `BLOCK_PHYSICS` colors simultaneously.
- `pixelMask` is intentionally absent from the `Block` interface here; T17 adds it via a single targeted `modify` step to avoid circular imports with `src/carving/pixelMask.ts`.
- `CASTLE_ZONE.width = Infinity` means callers must never compute `startX + width` directly without a guard; `getZoneAtX` handles this safely.
- All color values must be valid 6-digit CSS hex strings — they are consumed directly by `ctx.fillStyle` in the renderer.
