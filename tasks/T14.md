# T14 — [Phase 1] — Inventory UI (bottom slot bar)

## Overview
The inventory slot bar is the primary persistent UI element players interact with on every session — it shows what they're carrying and lets them switch between tools and materials. This task implements the visual rendering and touch handling for the 20-slot bar that sits at the bottom of the screen, using the renderer's UI drawer API rather than DOM elements so it integrates seamlessly with the Canvas 2D rendering pipeline.

## Scope
Implement `src/ui/inventory.ts` as a self-contained UI module. On import, register a UI drawer with `renderer.registerUIDrawer(drawInventory)`. The `drawInventory(ctx, deltaMs)` function draws a semi-transparent dark background bar 80px tall at the bottom of the screen, then renders all 20 slots in a horizontal row. Each occupied slot shows a colored material swatch, a short abbreviation label, and a quantity badge in the corner. The currently selected slot (`inventory.selectedSlotIndex`) is highlighted with a 2px bright yellow border. Touch events on the inventory bar set `selectedSlotIndex` and must not propagate to the world input layer. Export `getInventoryBarHeight(): number` (returns 80) for other UI modules to use as a layout constant.

### Constants and layout math

```typescript
// src/ui/inventory.ts
import { registerUIDrawer, getCanvas, getContext } from '../canvas/renderer';
import { inventory, setSelectedSlot, InventorySlot } from '../systems/inventory';
import { BLOCK_PHYSICS, BlockType } from '../constants/blocks';

export const INVENTORY_BAR_HEIGHT = 80;
const SLOT_COUNT = 20;
const SLOT_PADDING = 4;

// Short display names for items (fits in a 40px slot)
const ITEM_LABELS: Record<string, string> = {
  dry_sand:   '모래',
  wet_sand:   '습모',
  wood_plank: '나무',
  stone:      '돌',
  shell:      '조개',
  shovel:     '삽',
  branch:     '나뭇가지',
  bucket:     '양동이',
};

// Colors for tool items (materials use BLOCK_PHYSICS colors)
const TOOL_COLORS: Record<string, string> = {
  shovel:  '#8B6914',
  branch:  '#5C4A1E',
  bucket:  '#4A90D9',
};
```

### Slot geometry helper

```typescript
function getSlotRect(index: number, canvasWidth: number, canvasHeight: number) {
  const barY = canvasHeight - INVENTORY_BAR_HEIGHT;
  const totalSlotWidth = canvasWidth / SLOT_COUNT;
  const slotSize = Math.min(totalSlotWidth, INVENTORY_BAR_HEIGHT) - SLOT_PADDING * 2;

  return {
    x: index * totalSlotWidth + SLOT_PADDING,
    y: barY + (INVENTORY_BAR_HEIGHT - slotSize) / 2,
    w: slotSize,
    h: slotSize,
    barY,
  };
}
```

### Main draw function

```typescript
function drawInventory(ctx: CanvasRenderingContext2D, _deltaMs: number): void {
  const canvas = getCanvas();
  const W = canvas.width;
  const H = canvas.height;
  const barY = H - INVENTORY_BAR_HEIGHT;

  // Background bar
  ctx.fillStyle = 'rgba(0, 0, 0, 0.65)';
  ctx.fillRect(0, barY, W, INVENTORY_BAR_HEIGHT);

  // Slot separator lines
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.1)';
  ctx.lineWidth = 1;
  const slotWidth = W / SLOT_COUNT;
  for (let i = 1; i < SLOT_COUNT; i++) {
    ctx.beginPath();
    ctx.moveTo(i * slotWidth, barY);
    ctx.lineTo(i * slotWidth, H);
    ctx.stroke();
  }

  // Slots
  for (let i = 0; i < SLOT_COUNT; i++) {
    const rect = getSlotRect(i, W, H);
    const slot: InventorySlot | null = inventory.slots[i];

    // Selected highlight
    if (i === inventory.selectedSlotIndex) {
      ctx.strokeStyle = '#FFD700'; // gold
      ctx.lineWidth = 2;
      ctx.strokeRect(rect.x - 1, rect.y - 1, rect.w + 2, rect.h + 2);
    }

    if (!slot) {
      // Empty slot: dark inset
      ctx.fillStyle = 'rgba(255, 255, 255, 0.05)';
      ctx.fillRect(rect.x, rect.y, rect.w, rect.h);
      continue;
    }

    // Colored swatch
    const color = TOOL_COLORS[slot.itemType]
      ?? BLOCK_PHYSICS[slot.itemType as BlockType]?.color
      ?? '#888888';
    ctx.fillStyle = color;
    ctx.fillRect(rect.x, rect.y, rect.w, rect.h);

    // Item label (abbreviated, centered)
    ctx.fillStyle = '#FFFFFF';
    ctx.font = `bold ${Math.max(9, rect.w * 0.22)}px sans-serif`;
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText(
      ITEM_LABELS[slot.itemType] ?? slot.itemType.slice(0, 3),
      rect.x + rect.w / 2,
      rect.y + rect.h / 2,
    );

    // Quantity badge (bottom-right corner)
    if (slot.quantity > 1) {
      const badgeText = slot.quantity >= 100 ? '99+' : String(slot.quantity);
      ctx.fillStyle = 'rgba(0, 0, 0, 0.7)';
      ctx.font = `${Math.max(8, rect.w * 0.18)}px sans-serif`;
      ctx.textAlign = 'right';
      ctx.textBaseline = 'bottom';
      ctx.fillText(badgeText, rect.x + rect.w - 2, rect.y + rect.h - 2);
    }
  }
}
```

### Touch handling — slot selection

Touch events on the inventory bar are handled here and must not reach the world input layer. The `touchstart` listener checks if the y-coordinate is within the bar before consuming the event.

```typescript
function initInventoryTouch(): void {
  const canvas = getCanvas();

  canvas.addEventListener('touchstart', (e: TouchEvent) => {
    const touch = e.changedTouches[0];
    const barY = canvas.height - INVENTORY_BAR_HEIGHT;

    if (touch.clientY < barY) return; // tap is above the inventory bar — let it through

    e.stopPropagation(); // consume the event before world handlers see it
    e.preventDefault();

    const slotWidth = canvas.width / SLOT_COUNT;
    const slotIndex = Math.floor(touch.clientX / slotWidth);

    if (slotIndex >= 0 && slotIndex < SLOT_COUNT) {
      setSelectedSlot(slotIndex);
    }
  }, { passive: false, capture: true }); // capture phase to intercept before world handlers
}
```

### Module initialization

Auto-register when the module is first imported.

```typescript
// Runs once at import time
registerUIDrawer(drawInventory);
initInventoryTouch();

export function getInventoryBarHeight(): number {
  return INVENTORY_BAR_HEIGHT;
}
```

## Key Constraints
- The touch listener for slot selection must use `capture: true` so it intercepts events before the world-space handlers (shovel dig, block placement) process them.
- `e.stopPropagation()` (not `e.preventDefault()` alone) is needed to prevent the touch from reaching renderer.ts's `touchstart` handlers.
- The `drawInventory` function must not apply any `ctx.translate`/`ctx.scale` transforms — it draws in screen space and assumes the camera transform has already been restored.
- The slot width calculation `canvas.width / SLOT_COUNT` produces non-integer widths on many screens; use `Math.floor` for pixel-aligned drawing to avoid sub-pixel blurriness.
- `getInventoryBarHeight()` must be used by `hud.ts` (T15) and any other module that needs to avoid overlapping with the inventory bar.
- The quantity badge should show `'99+'` when `quantity >= 100` to stay within the badge bounds.
- `ITEM_LABELS` Korean strings must be verified for display correctness on mobile devices — ensure the canvas font supports Korean characters (use system-ui or a web font).
