# T15 — [Phase 1] — Basic HUD & mini-map

## Overview
The HUD gives the player spatial awareness — knowing which zone they're in and approximately where they are on the full map. This task implements two lightweight UI elements drawn in canvas screen-space: a top bar showing the current zone name, and a mini-map strip in the bottom-left corner showing the three zones with a position cursor. Both are designed as extensible foundations for richer indicators added in later tasks.

## Scope
Implement `src/ui/hud.ts` as a self-contained UI module that registers two UI drawers with the renderer. The top HUD bar is 40px tall, semi-transparent, and displays the Korean zone label for the zone at the current camera center. The mini-map is 200×30px in the bottom-left corner (above the inventory bar), rendered as three proportionally-sized colored bands corresponding to `MAP_ZONES`, with a small white dot indicating the current camera center world position. Export `getHUDTopHeight(): number` (returns 40). The module is designed for extension in later tasks — placeholder slots for stability indicator, tide countdown, and season label are included as `//TODO` comments.

## Output Files

### Constants and layout

```typescript
// src/ui/hud.ts
import { registerUIDrawer, getCanvas } from '../canvas/renderer';
import { getOffset, getScale } from '../canvas/camera';
import { MAP_ZONES, getZoneAtX, ZoneDefinition } from '../constants/maps';
import { getInventoryBarHeight } from './inventory'; // from T14

export const HUD_TOP_HEIGHT = 40;
const MINIMAP_WIDTH = 200;
const MINIMAP_HEIGHT = 30;
const MINIMAP_MARGIN = 8; // px from edge

// Total world width to represent in mini-map (visible portion, not Infinity)
const MINIMAP_WORLD_WIDTH = 4000;
```

### Top HUD bar

Displays the current zone's Korean label centered in the bar. The background is slightly darker on scroll (uses a fixed opacity for Phase 1). Placeholder slots for future indicators are placed on the right side.

```typescript
function drawHUDTop(ctx: CanvasRenderingContext2D): void {
  const canvas = getCanvas();
  const W = canvas.width;
  const { offsetX } = getOffset();
  const scale = getScale();

  // Background
  ctx.fillStyle = 'rgba(0, 0, 0, 0.5)';
  ctx.fillRect(0, 0, W, HUD_TOP_HEIGHT);

  // Bottom border line
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.15)';
  ctx.lineWidth = 1;
  ctx.beginPath();
  ctx.moveTo(0, HUD_TOP_HEIGHT);
  ctx.lineTo(W, HUD_TOP_HEIGHT);
  ctx.stroke();

  // Current zone label (center-screen world x → zone lookup)
  const cameraCenterWorldX = (offsetX + W / 2) / scale;
  const zone: ZoneDefinition = getZoneAtX(cameraCenterWorldX);

  ctx.fillStyle = '#FFFFFF';
  ctx.font = 'bold 16px system-ui, sans-serif';
  ctx.textAlign = 'center';
  ctx.textBaseline = 'middle';
  ctx.fillText(zone.label, W / 2, HUD_TOP_HEIGHT / 2);
```

### Mini-map

The mini-map represents the full `MINIMAP_WORLD_WIDTH` as a 200px strip. Each zone occupies a proportional width. A white dot marks the camera center. The mini-map is placed above the inventory bar using `getInventoryBarHeight()`.

```typescript
function drawMinimap(ctx: CanvasRenderingContext2D): void {
  const canvas = getCanvas();
  const H = canvas.height;
  const { offsetX } = getOffset();
  const scale = getScale();
  const invBarH = getInventoryBarHeight();

  const mmX = MINIMAP_MARGIN;
  const mmY = H - invBarH - MINIMAP_MARGIN - MINIMAP_HEIGHT;

  // Background
  ctx.fillStyle = 'rgba(0, 0, 0, 0.55)';
  roundRect(ctx, mmX, mmY, MINIMAP_WIDTH, MINIMAP_HEIGHT, 4);
  ctx.fill();

  // Zone color bands
  for (const zone of MAP_ZONES) {
    const zoneStartRatio = zone.startX / MINIMAP_WORLD_WIDTH;
    const zoneWidthPx = isFinite(zone.width)
      ? (zone.width / MINIMAP_WORLD_WIDTH) * MINIMAP_WIDTH
      : MINIMAP_WIDTH - (zone.startX / MINIMAP_WORLD_WIDTH) * MINIMAP_WIDTH;
    const zoneX = mmX + zoneStartRatio * MINIMAP_WIDTH;

    ctx.fillStyle = zone.groundColor;
    ctx.fillRect(zoneX, mmY, zoneWidthPx, MINIMAP_HEIGHT);
  }

  // Camera position dot
  const cameraCenterWorldX = (offsetX + canvas.width / 2) / scale;
  const dotRatio = Math.max(0, Math.min(1, cameraCenterWorldX / MINIMAP_WORLD_WIDTH));
  const dotX = mmX + dotRatio * MINIMAP_WIDTH;
  const dotY = mmY + MINIMAP_HEIGHT / 2;

  ctx.fillStyle = '#FFFFFF';
  ctx.beginPath();
  ctx.arc(dotX, dotY, 3, 0, Math.PI * 2);
  ctx.fill();

  // Mini-map border
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.3)';
  ctx.lineWidth = 1;
  roundRect(ctx, mmX, mmY, MINIMAP_WIDTH, MINIMAP_HEIGHT, 4);
  ctx.stroke();
}
```

### Utility: rounded rectangle (Canvas 2D API compat shim)

`CanvasRenderingContext2D.roundRect` is available in modern browsers but may not be typed in older `lib.dom.d.ts`. Use a manual path as a fallback.

```typescript
function roundRect(
  ctx: CanvasRenderingContext2D,
  x: number, y: number,
  w: number, h: number,
  r: number,
): void {
  ctx.beginPath();
  ctx.moveTo(x + r, y);
  ctx.lineTo(x + w - r, y);
  ctx.quadraticCurveTo(x + w, y, x + w, y + r);
  ctx.lineTo(x + w, y + h - r);
  ctx.quadraticCurveTo(x + w, y + h, x + w - r, y + h);
  ctx.lineTo(x + r, y + h);
  ctx.quadraticCurveTo(x, y + h, x, y + h - r);
  ctx.lineTo(x, y + r);
  ctx.quadraticCurveTo(x, y, x + r, y);
  ctx.closePath();
}
```

### Module initialization

```typescript
// Auto-register at import time
registerUIDrawer((ctx: CanvasRenderingContext2D) => {
  drawHUDTop(ctx);
  drawMinimap(ctx);
});

export function getHUDTopHeight(): number {
  return HUD_TOP_HEIGHT;
}
```

## Key Constraints
- The mini-map Y position must account for `getInventoryBarHeight()` — hardcoding `80` here would break if the inventory bar height changes.
- The zone label uses `zone.label` (Korean string from `MAP_ZONES`), not `zone.id` — do not display the English ID to end users.
- `MINIMAP_WORLD_WIDTH = 4000` caps the mini-map display; the castle zone theoretically extends to Infinity, but the mini-map must not attempt to render infinite width. The dot will clamp to the right edge if the player's camera is beyond 4000px.
- Both drawers must complete in under 1ms per frame — no expensive operations (loops over blocks, DOM access, etc.) are allowed here.
- `getHUDTopHeight()` is consumed by the renderer's touch handler to determine the "world touch zone" (touches above the HUD top bar should not register as world interactions).
- The HUD is screen-space — no camera transform is applied. All coordinates are in `canvas.width × canvas.height` space.
- Later tasks (T24 stability, T26 tide, T28 season) will add content to the right side of the top bar using the `//TODO` slots; keep the center label centered and the right region free.
