# T35 — [Phase 3] — Guestbook DB & API

## Overview
This task implements the Supabase data layer for the guestbook feature, allowing visitors to leave short messages on other players' castle plots. It establishes the `guestbook` table contract, enforces message length constraints at the application layer, and delegates ownership-based deletion to Supabase Row Level Security policies.

## Scope
Create `src/supabase/guestbook.ts` with three exported async functions and one exported TypeScript interface. `addMessage(plotId, authorId, message)` validates that the message is between 1 and 50 characters (inclusive), then inserts a row into the `guestbook` table; it throws a typed `GuestbookError` for validation failures or DB errors. `getMessages(plotId)` selects the latest 20 rows ordered by `created_at DESC`, joining with the `users` table to resolve `author_nickname`; it returns an array of `GuestbookEntry`. `deleteMessage(id, requesterId)` issues a DELETE filtered by both `id` and a sub-select confirming `requesterId` is the plot owner — this check is redundant with RLS but provides a clear error at the application layer. The `GuestbookEntry` interface captures all fields returned by `getMessages`. The Supabase client instance is imported from `src/supabase/client.ts` (established in T02).

## Output Files
- `src/supabase/guestbook.ts` — new file containing all guestbook DB logic and the `GuestbookEntry` interface

## Depends On
- T02 — provides the initialized Supabase client (`src/supabase/client.ts`) and base DB connection configuration

## Implementation Details

### GuestbookEntry Interface & Error Types

```typescript
// src/supabase/guestbook.ts

import { supabase } from './client';

export interface GuestbookEntry {
  id: string;               // UUID primary key
  plotId: string;           // FK → plots.id
  authorId: string;         // FK → users.id
  authorNickname: string;   // resolved from users.nickname JOIN
  message: string;          // 1–50 chars
  createdAt: string;        // ISO 8601 timestamp string
}

export class GuestbookError extends Error {
  constructor(
    message: string,
    public readonly code: 'VALIDATION' | 'NOT_FOUND' | 'UNAUTHORIZED' | 'DB_ERROR'
  ) {
    super(message);
    this.name = 'GuestbookError';
  }
}

const MAX_MESSAGE_LENGTH = 50;
const MAX_MESSAGES_RETURNED = 20;
```

### addMessage — Validation & Insert

```typescript
export async function addMessage(
  plotId: string,
  authorId: string,
  message: string
): Promise<GuestbookEntry> {
  const trimmed = message.trim();

  if (trimmed.length === 0) {
    throw new GuestbookError('메시지를 입력해주세요.', 'VALIDATION');
  }
  if (trimmed.length > MAX_MESSAGE_LENGTH) {
    throw new GuestbookError(
      `메시지는 ${MAX_MESSAGE_LENGTH}자 이하로 입력해주세요.`,
      'VALIDATION'
    );
  }

  const { data, error } = await supabase
    .from('guestbook')
    .insert({
      plot_id: plotId,
      author_id: authorId,
      message: trimmed,
    })
    .select(`
      id,
      plot_id,
      author_id,
      message,
      created_at,
      users!author_id ( nickname )
    `)
    .single();

  if (error || !data) {
    throw new GuestbookError(`메시지 저장 실패: ${error?.message}`, 'DB_ERROR');
  }

  return mapRow(data);
}
```

The `.trim()` is applied before length validation so that a 50-space string is correctly rejected. The insert uses `.select()` with an inline join so the returned row is immediately usable as a `GuestbookEntry` without a second round-trip.

### getMessages — Paginated Fetch with JOIN

```typescript
export async function getMessages(plotId: string): Promise<GuestbookEntry[]> {
  const { data, error } = await supabase
    .from('guestbook')
    .select(`
      id,
      plot_id,
      author_id,
      message,
      created_at,
      users!author_id ( nickname )
    `)
    .eq('plot_id', plotId)
    .order('created_at', { ascending: false })
    .limit(MAX_MESSAGES_RETURNED);

  if (error) {
    throw new GuestbookError(`메시지 불러오기 실패: ${error.message}`, 'DB_ERROR');
  }

  return (data ?? []).map(mapRow);
}
```

### deleteMessage — Owner-Gated Deletion

```typescript
export async function deleteMessage(
  id: string,
  requesterId: string
): Promise<void> {
  // RLS policy enforces this, but we add an explicit app-level check
  // to provide a meaningful error before hitting the DB.
  const { data: entry, error: fetchError } = await supabase
    .from('guestbook')
    .select('id, plots!plot_id ( owner_id )')
    .eq('id', id)
    .single();

  if (fetchError || !entry) {
    throw new GuestbookError('메시지를 찾을 수 없습니다.', 'NOT_FOUND');
  }

  const ownerId = (entry as any).plots?.owner_id;
  if (ownerId !== requesterId) {
    throw new GuestbookError('삭제 권한이 없습니다.', 'UNAUTHORIZED');
  }

  const { error: deleteError } = await supabase
    .from('guestbook')
    .delete()
    .eq('id', id);

  if (deleteError) {
    throw new GuestbookError(`삭제 실패: ${deleteError.message}`, 'DB_ERROR');
  }
}
```

### mapRow — Raw DB Row to GuestbookEntry

```typescript
function mapRow(row: any): GuestbookEntry {
  return {
    id: row.id,
    plotId: row.plot_id,
    authorId: row.author_id,
    authorNickname: row.users?.nickname ?? '알 수 없음',
    message: row.message,
    createdAt: row.created_at,
  };
}
```

### Required Supabase Schema

The `guestbook` table must exist with these columns:

```sql
CREATE TABLE guestbook (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plot_id    UUID NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
  author_id  UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  message    TEXT NOT NULL CHECK (char_length(message) <= 50),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- RLS: only the plot owner may delete entries
CREATE POLICY "plot_owner_delete" ON guestbook
  FOR DELETE USING (
    EXISTS (
      SELECT 1 FROM plots WHERE plots.id = guestbook.plot_id
        AND plots.owner_id = auth.uid()
    )
  );

-- RLS: anyone authenticated may insert
CREATE POLICY "authenticated_insert" ON guestbook
  FOR INSERT WITH CHECK (auth.uid() = author_id);

-- RLS: public read
CREATE POLICY "public_select" ON guestbook
  FOR SELECT USING (true);
```

## Key Constraints
- Message length validation occurs BEFORE the DB call to avoid unnecessary network round-trips.
- The 50-character limit is enforced at three layers: UI counter (T36), application validation (`addMessage`), and DB `CHECK` constraint.
- `deleteMessage` validates ownership at the application layer for a clear error message; RLS serves as the authoritative enforcement layer.
- `getMessages` always returns at most 20 rows; there is no cursor-based pagination in this task (a future task may add it).
- The `users!author_id` join syntax requires that the Supabase foreign key is named or hinted correctly — verify FK hint matches project schema.
- All functions throw `GuestbookError` (never raw DB errors) so callers can type-narrow on `.code`.
- The function does NOT expose a bulk-delete or admin-delete path; moderation is out of scope for this task.

## Acceptance Criteria
- [ ] `addMessage` with a message of exactly 50 characters inserts successfully and returns a valid `GuestbookEntry`.
- [ ] `addMessage` with a 51-character message throws `GuestbookError` with code `'VALIDATION'` without making a DB call.
- [ ] `addMessage` with an empty string or whitespace-only string throws `GuestbookError` with code `'VALIDATION'`.
- [ ] `getMessages` returns entries ordered newest-first and never more than 20 rows.
- [ ] `getMessages` for a plot with no entries returns an empty array without throwing.
- [ ] `deleteMessage` called by the plot owner successfully removes the row from DB.
- [ ] `deleteMessage` called by a non-owner throws `GuestbookError` with code `'UNAUTHORIZED'` before issuing a DELETE.
- [ ] All returned `GuestbookEntry` objects have `authorNickname` populated (fallback `'알 수 없음'` if the user row is missing).
