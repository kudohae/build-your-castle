# T13 — [Phase 1] — Block placement in castle zone

## Overview
Block placement is the central creative action of the game — the player drags a block from their inventory and drops it in their castle plot. This task implements the full placement pipeline: ghost preview while dragging, inventory deduction on release, physics body creation, a 2-second settling period, and final position persistence to Supabase. Placement is restricted to the player's own plot boundaries to prevent griefing.

## Scope
Extend the input handling in `src/canvas/renderer.ts` to detect drags in the CASTLE_ZONE when a block type is selected. During the drag, render a semi-transparent 32×32 ghost rectangle following the finger. On `touchend`: validate the release position is within the current user's plot x-bounds; deduct one unit from inventory; create a new `Block` object with a UUID; call `createBody(block)` to hand it to physics; wait 2000ms for the block to settle under gravity; call `syncBodyToBlock(block)` to capture the final position; call `saveBlock(block)` to persist it. Extend `src/physics/blockBodies.ts` and `src/supabase/blocks.ts` with any additional hooks needed to support this flow.

## Output Files
- `src/physics/blockBodies.ts (modify)` — no new exports needed; used as-is from T08
- `src/supabase/blocks.ts (modify)` — no structural changes; `saveBlock` used as-is from T10

## Implementation Details

### Placement mode state (in renderer.ts)

```typescript
// Module-level placement state
interface PlacementState {
  active: boolean;
  itemType: string | null;   // the block type being placed
  ghostX: number;            // current finger world-x
  ghostY: number;            // current finger world-y
  startClientX: number;      // finger down screen position (for tap vs drag detection)
  startClientY: number;
}

const placement: PlacementState = {
  active: false,
  itemType: null,
  ghostX: 0,
  ghostY: 0,
  startClientX: 0,
  startClientY: 0,
};
```

### Touch event integration for placement

These handlers are added alongside the shovel handler in `initInputHandlers()`. Placement mode activates when:
1. The selected item is a `BlockType` (not a tool).
2. The touch-down world-x is within the CASTLE_ZONE.

```typescript
import { v4 as uuidv4 } from 'uuid';
import { createBody, syncBodyToBlock } from '../physics/blockBodies';
import { saveBlock } from '../supabase/blocks';
import { getUser } from '../ait/auth';
import { loadPlot } from '../supabase/plots';
import { BlockType, BLOCK_SIZE } from '../constants/blocks';

const TOOL_TYPES_SET = new Set(['shovel', 'branch', 'bucket']);

canvas.addEventListener('touchstart', (e: TouchEvent) => {
  const touch = e.changedTouches[0];
  const { x: worldX, y: worldY } = screenToWorld(touch.clientX, touch.clientY);
  const selected = getSelectedItem();

  if (!selected || TOOL_TYPES_SET.has(selected.itemType)) return;
  if (getZoneAtX(worldX).id !== 'CASTLE_ZONE') return;

  placement.active = true;
  placement.itemType = selected.itemType;
  placement.ghostX = worldX;
  placement.ghostY = worldY;
  placement.startClientX = touch.clientX;
  placement.startClientY = touch.clientY;
}, { passive: false });

canvas.addEventListener('touchmove', (e: TouchEvent) => {
  if (!placement.active) return;
  const touch = e.changedTouches[0];
  const { x: worldX, y: worldY } = screenToWorld(touch.clientX, touch.clientY);
  placement.ghostX = worldX;
  placement.ghostY = worldY;
}, { passive: false });

canvas.addEventListener('touchend', async (e: TouchEvent) => {
  if (!placement.active) return;
  placement.active = false;

  const { x: worldX, y: worldY } = screenToWorld(
    e.changedTouches[0].clientX,
    e.changedTouches[0].clientY,
  );

  await commitPlacement(worldX, worldY);
});
```

### commitPlacement — full placement pipeline

```typescript
async function commitPlacement(worldX: number, worldY: number): Promise<void> {
  const user = getUser();
  if (!user) return;

  // Load the user's plot to verify bounds
  const plot = await loadPlot(/* plotId from session */);
  if (!plot) return;

  // Check the drop position is within the user's plot
  if (worldX < plot.positionX || worldX > plot.positionX + plot.width) {
    console.warn('[Placement] Drop position outside plot bounds');
    return;
  }

  const itemType = placement.itemType as BlockType;

  // Deduct from inventory (atomic check + remove)
  if (!removeItem(itemType, 1)) {
    console.warn('[Placement] Not enough items in inventory');
    return;
  }

  // Create the block data object
  const block: Block = {
    id: uuidv4(),
    type: itemType,
    x: worldX - BLOCK_SIZE / 2,   // center the block on the drop position
    y: worldY - BLOCK_SIZE / 2,
    plotId: plot.id,
    isWet: false,
    health: 100,
  };

  // Add to renderer immediately so the player sees it
  addBlock(block);

  // Create physics body — block will fall under gravity
  createBody(block);

  // Wait for physics to settle
  await new Promise(resolve => setTimeout(resolve, 2000));

  // Sync final physics position back to block object
  syncBodyToBlock(block);

  // Persist the settled position
  await saveBlock(block);
}
```

### Ghost preview rendering

The ghost is drawn in the world-space draw layer (inside the `ctx.save()/restore()` transform block in `render()`):

```typescript
// Inside render() — after block drawing, before ctx.restore():
if (placement.active && placement.itemType) {
  const physics = BLOCK_PHYSICS[placement.itemType as BlockType];
  if (physics) {
    ctx.save();
    ctx.globalAlpha = 0.5;
    ctx.fillStyle = physics.color;
    ctx.fillRect(
      placement.ghostX - BLOCK_SIZE / 2,
      placement.ghostY - BLOCK_SIZE / 2,
      BLOCK_SIZE,
      BLOCK_SIZE,
    );
    // Dashed border to indicate "not yet placed"
    ctx.strokeStyle = '#FFFFFF';
    ctx.lineWidth = 2 / getScale();
    ctx.setLineDash([4, 4]);
    ctx.strokeRect(
      placement.ghostX - BLOCK_SIZE / 2,
      placement.ghostY - BLOCK_SIZE / 2,
      BLOCK_SIZE,
      BLOCK_SIZE,
    );
    ctx.restore();
  }
}
```

## Key Constraints
- `commitPlacement` must check `hasItem` (via `removeItem`'s internal guard) before creating any block objects — never create a block if the inventory deduction would fail.
- The 2000ms settle timer is a fixed delay for Phase 1. A better approach (detect when body velocity drops below a threshold) should be implemented in a later task to handle edge cases where blocks settle faster or slower.
- Placement is restricted to the user's own plot — `worldX` must be within `[plot.positionX, plot.positionX + plot.width]`. Attempting to place in another user's plot silently cancels.
- The ghost preview must be drawn in world space (inside the camera transform) so it scales and scrolls correctly with the rest of the world.
- `uuidv4()` from the `uuid` package must be used for block IDs — do not use `Math.random()` or `Date.now()` as they can produce collisions.
- If `saveBlock` fails after the block is already in the renderer and physics world, log the error and leave the block in-world — losing it silently would frustrate the player. A retry mechanism is a future enhancement.
- The `plotId` session value must be available when `commitPlacement` runs. The plot must be fetched after login (T11) and stored at the application level, not re-fetched on every placement.
