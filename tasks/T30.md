# T30 — [Phase 3] — Realtime reconciliation loop

## Overview
While the Realtime subscription in T29 handles live events, network gaps and missed events can leave the local block state inconsistent with the database. The reconciliation loop provides a safety net: every 30 seconds it performs a full load of the plot's blocks from Supabase and surgically syncs the local state — adding missing blocks, removing phantom blocks, and repositioning drifted ones. Spectators (non-owners viewing another player's plot) benefit most from this, as they never make writes but must see an accurate view.

## Scope
Add `startReconciliation(plotId: string, getLocalBlocks: () => Block[], applyDiff: ReconciliationDiff => void): void` to `src/supabase/realtime.ts`. The function sets a `setInterval` for every 30 seconds that calls `loadBlocks(plotId)` from T10, computes the diff between the returned DB blocks and the current local blocks array, then calls `applyDiff` with the result. The `ReconciliationDiff` object contains three arrays: `toAdd: Block[]` (in DB, not local), `toRemove: string[]` (in local, not in DB — IDs), and `toUpdate: Block[]` (in both but with differing x/y position). The caller's `applyDiff` handler is responsible for updating local state and the scene; the reconciliation module does not touch the block array directly. For spectators, block coordinates are set directly without physics re-simulation (use `Matter.Body.setPosition` if a body exists, otherwise just update `block.x`/`block.y`). Add `stopReconciliation(plotId: string): void` to clear the interval. Track intervals in a `Map<string, ReturnType<typeof setInterval>>`. Multiple plots can be reconciled simultaneously.

## Output Files
- `src/supabase/realtime.ts (modify)` — adds `startReconciliation`, `stopReconciliation`, `ReconciliationDiff` to the existing realtime module

## Depends On
- T29 — this is a direct extension of the realtime module; reconciliation is co-located in the same file because it shares the subscription infrastructure and plot-channel context

## Implementation Details

### ReconciliationDiff type

```typescript
// src/supabase/realtime.ts (addition)

import { loadBlocks } from './blocks'; // from T10
import Matter from 'matter-js';

export interface ReconciliationDiff {
  toAdd: Block[];
  toRemove: string[]; // block IDs
  toUpdate: Block[];  // DB version with new x/y
}
```

### computeDiff helper

```typescript
function computeDiff(
  dbBlocks: Block[],
  localBlocks: Block[]
): ReconciliationDiff {
  const localById = new Map<string, Block>(localBlocks.map((b) => [b.id, b]));
  const dbById = new Map<string, Block>(dbBlocks.map((b) => [b.id, b]));

  const toAdd: Block[] = [];
  const toRemove: string[] = [];
  const toUpdate: Block[] = [];

  // Blocks in DB not in local → add
  for (const [id, dbBlock] of dbById.entries()) {
    if (!localById.has(id)) {
      toAdd.push(dbBlock);
    }
  }

  // Blocks in local not in DB → remove
  for (const [id] of localById.entries()) {
    if (!dbById.has(id)) {
      toRemove.push(id);
    }
  }

  // Blocks in both with different position → update
  for (const [id, dbBlock] of dbById.entries()) {
    const localBlock = localById.get(id);
    if (localBlock && (localBlock.x !== dbBlock.x || localBlock.y !== dbBlock.y)) {
      toUpdate.push(dbBlock);
    }
  }

  return { toAdd, toRemove, toUpdate };
}
```

### startReconciliation

```typescript
const reconciliationIntervals = new Map<string, ReturnType<typeof setInterval>>();
const RECONCILIATION_INTERVAL_MS = 30_000;

export function startReconciliation(
  plotId: string,
  getLocalBlocks: () => Block[],
  applyDiff: (diff: ReconciliationDiff) => void
): void {
  // Clear any existing interval for this plot
  stopReconciliation(plotId);

  const intervalId = setInterval(async () => {
    try {
      const dbBlocks = await loadBlocks(plotId);
      const localBlocks = getLocalBlocks();
      const diff = computeDiff(dbBlocks, localBlocks);

      const hasChanges =
        diff.toAdd.length > 0 ||
        diff.toRemove.length > 0 ||
        diff.toUpdate.length > 0;

      if (hasChanges) {
        console.log(
          `[Reconciliation] Plot ${plotId}: +${diff.toAdd.length} -${diff.toRemove.length} ~${diff.toUpdate.length}`
        );
        applyDiff(diff);
      }
    } catch (err) {
      console.error(`[Reconciliation] Error for plot ${plotId}:`, err);
      // Do not clear the interval on error — keep retrying
    }
  }, RECONCILIATION_INTERVAL_MS);

  reconciliationIntervals.set(plotId, intervalId);
  console.log(`[Reconciliation] Started for plot: ${plotId}`);
}
```

### stopReconciliation

```typescript
export function stopReconciliation(plotId: string): void {
  const existing = reconciliationIntervals.get(plotId);
  if (existing !== undefined) {
    clearInterval(existing);
    reconciliationIntervals.delete(plotId);
    console.log(`[Reconciliation] Stopped for plot: ${plotId}`);
  }
}
```

### Recommended applyDiff handler (caller-side)

The caller passes an `applyDiff` handler that updates local block state and, for spectator mode, repositions physics bodies without simulation:

```typescript
// Example usage in the game scene for spectator mode:

import { startReconciliation, ReconciliationDiff } from '../supabase/realtime';

startReconciliation(
  currentPlotId,
  () => localBlockStore.getAll(),
  (diff: ReconciliationDiff) => {
    // Add missing blocks
    for (const block of diff.toAdd) {
      localBlockStore.add(block);
      scene.addBlockToScene(block); // creates physics body
    }

    // Remove phantom blocks
    for (const id of diff.toRemove) {
      scene.removeBlockFromScene(id);
      localBlockStore.remove(id);
    }

    // Update repositioned blocks (spectator: no physics re-run)
    for (const dbBlock of diff.toUpdate) {
      const local = localBlockStore.getById(dbBlock.id);
      if (local) {
        local.x = dbBlock.x;
        local.y = dbBlock.y;
        if (local.body) {
          // Direct position set — bypasses physics simulation
          Matter.Body.setPosition(local.body, { x: dbBlock.x + 16, y: dbBlock.y + 16 });
          Matter.Body.setVelocity(local.body, { x: 0, y: 0 });
        }
      }
    }
  }
);
```

### Interval lifecycle with subscriptions
Reconciliation intervals should be started alongside subscriptions and stopped alongside unsubscriptions. The recommended pattern is to always call both together:

```typescript
// Start viewing a plot:
subscribePlot(plotId, callbacks);
startReconciliation(plotId, getLocalBlocks, applyDiff);

// Stop viewing a plot:
unsubscribePlot(plotId);
stopReconciliation(plotId);
```

However, `startReconciliation` and `stopReconciliation` are independent of the channel lifecycle so they can be called separately for non-realtime contexts (e.g., polling-only mode on platforms without WebSocket support).

### Position-only update rationale
Reconciliation updates only sync `x` and `y` position for "toUpdate" blocks, not physics properties like velocity, mass, or pixelMask. These are considered ephemeral or managed by the owner's client. Position drift is the most common source of visual desync for spectators because block placement coordinates are the single most important spatial fact.

## Key Constraints
- `startReconciliation` must call `stopReconciliation` first to prevent duplicate intervals for the same plotId.
- `loadBlocks` is an async network call; errors must be caught and logged without crashing the interval.
- `computeDiff` is a pure function and must not mutate any block.
- `Matter.Body.setPosition` takes the center position (`x + 16, y + 16` for 32px blocks), not the top-left corner.
- Reconciliation must not be called more frequently than 30 seconds; it is a heavy full-load operation.
- Multiple plots can have simultaneous reconciliation intervals (e.g., if the player is viewing a gallery of plots).

## Acceptance Criteria
- [ ] `startReconciliation` triggers a diff computation every 30 seconds (verified via mock setInterval spy).
- [ ] `applyDiff` is called with `toAdd` containing blocks present in DB but absent locally.
- [ ] `applyDiff` is called with `toRemove` containing IDs of blocks present locally but absent in DB.
- [ ] `applyDiff` is called with `toUpdate` for blocks with matching IDs but different x/y.
- [ ] `applyDiff` is NOT called when DB and local state are identical (no spurious updates).
- [ ] `stopReconciliation` clears the interval; no further `applyDiff` calls occur after stopping.
- [ ] A `loadBlocks` error (rejected promise) does not clear the interval — the next tick still fires.
- [ ] Calling `startReconciliation` twice for the same plotId results in only one active interval (duplicate is cleared).
