# T39 — [Phase 3] — Collaborator DB & Invite System

## Overview
This task adds the full collaborator management layer to the existing plots module, enabling up to 3 co-editors per castle plot. It establishes the invite/remove lifecycle and, critically, exposes a `canEdit` function that becomes the single gate for all block placement and carving actions throughout the codebase.

## Scope
Modify `src/supabase/plots.ts` to add five new exported async functions: `inviteCollaborator(plotId, inviterId, targetUserId)` validates that the inviter is the plot owner and that the total collaborator count (excluding owner) will not exceed 3, then inserts into the `plot_collaborators` junction table; `removeCollaborator(plotId, ownerId, targetUserId)` deletes the row only if `ownerId` matches the plot owner (enforced by app logic and RLS); `getCollaborators(plotId)` returns an array of `User` objects representing all current collaborators; `isCollaborator(plotId, userId)` returns a boolean by checking for a matching row in `plot_collaborators`; `canEdit(plotId, userId)` returns `true` if `userId` is either the `owner_id` on the plots table OR a confirmed collaborator. Every existing block-placement function (imported from `src/systems/blocks.ts`) and carving function must call `canEdit` and throw `PermissionError` if it returns false.

## Output Files
- `src/supabase/plots.ts (modify)` — adds `inviteCollaborator`, `removeCollaborator`, `getCollaborators`, `isCollaborator`, `canEdit`

## Depends On

  constructor() {
    super('협업자는 최대 3명까지 초대할 수 있습니다.');
    this.name = 'CollaboratorLimitError';
  }

const MAX_COLLABORATORS = 3;

### inviteCollaborator

```typescript
export async function inviteCollaborator(
  plotId: string,
  inviterId: string,
  targetUserId: string
): Promise<void> {
  // 1. Verify inviter is the owner
  const { data: plot, error: plotErr } = await supabase
    .from('plots')
    .select('owner_id')
    .eq('id', plotId)
    .single();

  if (plotErr || !plot) throw new Error('성을 찾을 수 없습니다.');
  if (plot.owner_id !== inviterId) throw new PermissionError('소유자만 협업자를 초대할 수 있습니다.');
  if (targetUserId === inviterId) throw new Error('자기 자신을 초대할 수 없습니다.');

  // 2. Count current collaborators
  const { count, error: countErr } = await supabase
    .from('plot_collaborators')
    .select('id', { count: 'exact', head: true })
    .eq('plot_id', plotId);

  if (countErr) throw new Error(`협업자 조회 실패: ${countErr.message}`);
  if ((count ?? 0) >= MAX_COLLABORATORS) throw new CollaboratorLimitError();

  // 3. Check for duplicate
  const { data: existing } = await supabase
    .from('plot_collaborators')
    .select('id')
    .eq('plot_id', plotId)
    .eq('user_id', targetUserId)
    .maybeSingle();

  if (existing) return; // Already a collaborator — idempotent

  // 4. Insert
  const { error: insertErr } = await supabase
    .from('plot_collaborators')
    .insert({ plot_id: plotId, user_id: targetUserId });

  if (insertErr) throw new Error(`초대 실패: ${insertErr.message}`);
}
```

### removeCollaborator

```typescript
export async function removeCollaborator(
  plotId: string,
  ownerId: string,
  targetUserId: string
): Promise<void> {
  // Verify caller is owner (also enforced by RLS)
  const { data: plot } = await supabase
    .from('plots')
    .select('owner_id')
    .eq('id', plotId)
    .single();

  if (!plot || plot.owner_id !== ownerId) {
    throw new PermissionError('소유자만 협업자를 제거할 수 있습니다.');
  }

  const { error } = await supabase
    .from('plot_collaborators')
    .delete()
    .eq('plot_id', plotId)
    .eq('user_id', targetUserId);

  if (error) throw new Error(`협업자 제거 실패: ${error.message}`);
}
```

### getCollaborators

```typescript
export async function getCollaborators(plotId: string): Promise<User[]> {
  const { data, error } = await supabase
    .from('plot_collaborators')
    .select(`
      user_id,
      users!user_id ( id, nickname, avatar_url )
    `)
    .eq('plot_id', plotId);

  if (error) throw new Error(`협업자 목록 조회 실패: ${error.message}`);

  return (data ?? []).map((row: any) => ({
    id: row.users.id,
    nickname: row.users.nickname,
    avatarUrl: row.users.avatar_url ?? undefined,
  }));
}
```

### isCollaborator

```typescript
export async function isCollaborator(
  plotId: string,
  userId: string
): Promise<boolean> {
  const { data, error } = await supabase
    .from('plot_collaborators')
    .select('id')
    .eq('plot_id', plotId)
    .eq('user_id', userId)
    .maybeSingle();

  if (error) return false;
  return data !== null;
}
```

### canEdit — The Universal Edit Gate

```typescript
export async function canEdit(
  plotId: string,
  userId: string
): Promise<boolean> {
  const { data: plot } = await supabase
    .from('plots')
    .select('owner_id')
    .eq('id', plotId)
    .single();

  if (!plot) return false;
  if (plot.owner_id === userId) return true;

  return isCollaborator(plotId, userId);
}
```

### Gating Block Actions

All block mutation functions in `src/systems/blocks.ts` must be wrapped with the `canEdit` guard. Example:

### Required DB Schema

```sql
CREATE TABLE plot_collaborators (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plot_id    UUID NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
  user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (plot_id, user_id)
);

-- RLS: only plot owner may insert/delete
CREATE POLICY "owner_manage_collaborators" ON plot_collaborators
  FOR ALL USING (
    EXISTS (
      SELECT 1 FROM plots WHERE plots.id = plot_collaborators.plot_id
        AND plots.owner_id = auth.uid()
    )
  );

-- RLS: collaborators can read their own membership
CREATE POLICY "collab_self_select" ON plot_collaborators
  FOR SELECT USING (user_id = auth.uid());
```

## Key Constraints
- The maximum of 3 collaborators is per-plot and excludes the owner (so a plot has at most 4 total editors: 1 owner + 3 collaborators).
- `inviteCollaborator` is idempotent: inviting an already-added user silently returns without error or duplicate row insertion.
- `canEdit` must be awaited before every block placement and carving operation — synchronous bypasses are not acceptable.
- `canEdit` checks DB on every call; no in-memory cache is used in this task to avoid stale permission state during real-time collaboration.
- `removeCollaborator` does not notify the removed user in this task — that notification hook may be added in a later push notification task.
- The `plot_collaborators` table has a `UNIQUE(plot_id, user_id)` constraint as a secondary duplicate guard at the DB level.
- RLS policies must be enabled on `plot_collaborators` before deploying — unauthenticated access to this table must not be possible.

## Acceptance Criteria
- [ ] `inviteCollaborator` inserts a row when the inviter is the owner and collaborator count < 3.
- [ ] `inviteCollaborator` throws `CollaboratorLimitError` when attempting to add a 4th collaborator.
- [ ] `inviteCollaborator` is idempotent — calling it twice with the same user does not duplicate rows or throw.
- [ ] `removeCollaborator` throws `PermissionError` when called by a non-owner.
- [ ] `getCollaborators` returns `User[]` objects with `id`, `nickname`, and optionally `avatarUrl`.
- [ ] `canEdit` returns `true` for the plot owner.
- [ ] `canEdit` returns `true` for a confirmed collaborator and `false` for an unrelated user.
- [ ] Calling `placeBlock` or `carveBlock` as a non-collaborator non-owner throws `PermissionError` without modifying any DB state.
