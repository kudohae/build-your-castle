# T40 — [Phase 3] — Collaborator Realtime Cursors

## Overview
This task adds live cursor presence to collaborative plot editing sessions using Supabase Broadcast channels. Each active co-editor's cursor position is shared at 10 Hz and rendered on the canvas as a colored circle with a nickname initial, giving all participants real-time awareness of where their collaborators are working.

## Scope
Modify `src/supabase/realtime.ts` to add `startCursorBroadcast(plotId, userId)` which joins a Supabase Broadcast channel named `collab:{plotId}`, establishes a 100ms throttled interval that emits the current local cursor position as `{userId, x, y}`, and returns a cleanup function. Add `listenCursors(plotId, onCursor)` which subscribes to the same channel and invokes `onCursor({userId, x, y})` for every received payload, updating an internal `Map<string, {x, y, color}>` keyed by `userId`. Collaborator colors are deterministically derived from `userId` (hash → hue). Modify `src/canvas/renderer.ts` to render each remote cursor as a 10px filled circle at the cursor's world-to-screen coordinates, with a 1.5px white stroke outline and a small label showing the first character of the collaborator's nickname. Cursors for users who have not sent a position update in over 5 seconds are hidden.

## Output Files
- `src/supabase/realtime.ts (modify)` — adds `startCursorBroadcast`, `listenCursors`, cursor state map
- `src/canvas/renderer.ts (modify)` — adds `drawCollaboratorCursors(ctx, camera)` call in the render loop

## Implementation Details

### Cursor State Store

```typescript
// src/supabase/realtime.ts (additions)

import { RealtimeChannel } from '@supabase/supabase-js';
import { supabase } from './client';

interface RemoteCursor {
  userId: string;
  x: number;           // world-space X
  y: number;           // world-space Y
  color: string;       // HSL string derived from userId
  nickname: string;    // first char used as label
  lastSeen: number;    // performance.now() timestamp of last packet
}

const CURSOR_TIMEOUT_MS = 5000;
const BROADCAST_INTERVAL_MS = 100;

// Keyed by userId
const remoteCursors = new Map<string, RemoteCursor>();
const activeChannels = new Map<string, RealtimeChannel>();
let broadcastIntervalId: ReturnType<typeof setInterval> | null = null;

/** Deterministic color from userId string (hash → hue). */
function userIdToColor(userId: string): string {
  let hash = 0;
  for (let i = 0; i < userId.length; i++) {
    hash = (hash * 31 + userId.charCodeAt(i)) >>> 0;
  }
  const hue = hash % 360;
  return `hsl(${hue}, 80%, 55%)`;
}
```

### startCursorBroadcast

```typescript
export function startCursorBroadcast(
  plotId: string,
  userId: string,
  getNickname: () => string,
  getCursorWorldPos: () => { x: number; y: number }
): () => void {
  const channelKey = `collab:${plotId}`;

  // Reuse existing channel if already joined
  let channel = activeChannels.get(channelKey);
  if (!channel) {
    channel = supabase.channel(channelKey, {
      config: { broadcast: { self: false } },
    });
    channel.subscribe();
    activeChannels.set(channelKey, channel);
  }

  // Throttled broadcast at 10 Hz
  broadcastIntervalId = setInterval(() => {
    const pos = getCursorWorldPos();
    channel!.send({
      type: 'broadcast',
      event: 'cursor',
      payload: {
        userId,
        nickname: getNickname(),
        x: Math.round(pos.x),
        y: Math.round(pos.y),
      },
    });
  }, BROADCAST_INTERVAL_MS);

  // Return cleanup fn
  return () => {
    if (broadcastIntervalId !== null) {
      clearInterval(broadcastIntervalId);
      broadcastIntervalId = null;
    }
    // Send a final "leave" event so peers hide this cursor immediately
    channel!.send({
      type: 'broadcast',
      event: 'cursor_leave',
      payload: { userId },
    });
    supabase.removeChannel(channel!);
    activeChannels.delete(channelKey);
  };
}
```

`getCursorWorldPos` is a callback so the broadcast always reads the latest mouse/touch position without capturing stale state at subscription time.

### listenCursors

```typescript
export function listenCursors(
  plotId: string,
  onCursor: (cursor: RemoteCursor) => void
): () => void {
  const channelKey = `collab:${plotId}`;

  let channel = activeChannels.get(channelKey);
  if (!channel) {
    channel = supabase.channel(channelKey, {
      config: { broadcast: { self: false } },
    });
    channel.subscribe();
    activeChannels.set(channelKey, channel);
  }

  channel.on('broadcast', { event: 'cursor' }, ({ payload }) => {
    const cursor: RemoteCursor = {
      userId: payload.userId,
      x: payload.x,
      y: payload.y,
      color: userIdToColor(payload.userId),
      nickname: (payload.nickname as string)?.[0]?.toUpperCase() ?? '?',
      lastSeen: performance.now(),
    };
    remoteCursors.set(payload.userId, cursor);
    onCursor(cursor);
  });

  channel.on('broadcast', { event: 'cursor_leave' }, ({ payload }) => {
    remoteCursors.delete(payload.userId);
  });

  return () => {
    // Caller is responsible for calling the broadcast cleanup fn separately
    remoteCursors.clear();
  };
}

/** Read-only snapshot for renderer. */
export function getRemoteCursors(): ReadonlyMap<string, RemoteCursor> {
  return remoteCursors;
}
```

### Stale Cursor Pruning

A lightweight pruning pass runs in the renderer's frame loop:

```typescript
// Called each frame before drawing cursors
export function pruneStaleRemoteCursors(): void {
  const now = performance.now();
  for (const [userId, cursor] of remoteCursors) {
    if (now - cursor.lastSeen > CURSOR_TIMEOUT_MS) {
      remoteCursors.delete(userId);
    }
  }
}
```

### Renderer Additions — drawCollaboratorCursors

```typescript
// src/canvas/renderer.ts (additions)

import { getRemoteCursors, pruneStaleRemoteCursors } from '../supabase/realtime';

interface Camera {
  x: number;    // world-space left edge
  y: number;
  zoom: number;
}

export function drawCollaboratorCursors(
  ctx: CanvasRenderingContext2D,
  camera: Camera
): void {
  pruneStaleRemoteCursors();
  const cursors = getRemoteCursors();

  for (const cursor of cursors.values()) {
    // World to screen conversion
    const screenX = (cursor.x - camera.x) * camera.zoom;
    const screenY = (cursor.y - camera.y) * camera.zoom;

    // Outer circle
    ctx.beginPath();
    ctx.arc(screenX, screenY, 10, 0, Math.PI * 2);
    ctx.fillStyle = cursor.color;
    ctx.fill();

    // White outline
    ctx.strokeStyle = '#ffffff';
    ctx.lineWidth = 1.5;
    ctx.stroke();

    // Nickname initial label
    ctx.fillStyle = '#ffffff';
    ctx.font = 'bold 10px Pretendard, sans-serif';
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText(cursor.nickname, screenX, screenY);

    // Reset text baseline
    ctx.textBaseline = 'alphabetic';
    ctx.textAlign = 'left';
  }
}
```

The call site in the renderer's main `draw()` function:

```typescript
// At the end of the world-draw phase, before HUD:
drawCollaboratorCursors(ctx, camera);
```

## Key Constraints
- Broadcast is only started after `canEdit` (T39) confirms the user is an owner or collaborator — anonymous visitors must not join the channel.
- The channel name `collab:{plotId}` is shared between broadcaster and listener; both use the same `activeChannels` map to reuse the single `RealtimeChannel` instance, avoiding duplicate subscriptions.
- Cursor positions are sent as integer world-space coordinates (rounded to nearest pixel) to minimize payload size.
- Cursors not updated within 5 seconds are pruned from the map and no longer rendered — this handles disconnections gracefully.
- The broadcast interval (100ms / 10 Hz) is a deliberate balance between responsiveness and Supabase Broadcast rate limits; do not lower below 50ms.
- The `self: false` channel config prevents a user from seeing their own cursor echo from Supabase, since the local cursor is already rendered by the host browser.
- World-to-screen conversion must use the same `camera` object as the rest of the renderer to ensure cursor positions align with block positions.

## Acceptance Criteria
- [ ] When two confirmed collaborators are on the same plot, each sees the other's cursor moving in real time with less than 200ms lag.
- [ ] Cursor color is consistent for each user across sessions (same userId always produces the same color).
- [ ] Cursor disappears from the canvas within 5 seconds of the remote user closing the tab or navigating away.
- [ ] The local user's own cursor is never rendered as a remote cursor (no self-echo).
- [ ] Broadcast intervals do not stack: calling `startCursorBroadcast` twice for the same plot does not create two intervals.
- [ ] The nickname initial label is correctly centered within the cursor circle.
- [ ] A non-collaborator who calls `startCursorBroadcast` is rejected before the channel join (caller must check `canEdit` first).
- [ ] Cursor screen position correctly tracks world position as the camera pans (zoom and pan accounted for in world-to-screen).
