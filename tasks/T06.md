# T06 — [Phase 1] — Inventory data layer

## Overview
The inventory system is the bridge between the player's collected resources and the actions they can perform. This task implements the pure data layer — no UI, no rendering — so that other systems (shovel digging, block placement, bucket) can read and write inventory state through a stable API. The UI layer is built separately in T14.

## Scope
Define the `InventorySlot` and `Inventory` interfaces, implement all mutation and query functions, and export a singleton `inventory` instance pre-populated with the default starting items: one shovel, one branch, one bucket, and 17 empty slots for a total of 20 slots. Functions to implement: `addItem(type, qty)`, `removeItem(type, qty)` (returns `false` if insufficient), `findSlot(type)` (returns slot index or -1), `hasItem(type, qty)` (boolean check), `getSelectedItem()` (returns the current slot's contents or null), `setSelectedSlot(index)`. Also add `shovelLevel: 1 | 2 | 3 | 5` to the singleton and `setShovelLevel(level)` for the upgrade path used later by the payment system (T21).

## Output Files
- `src/systems/inventory.ts` — all inventory types, the singleton instance, and all manipulation functions

## Depends On
- T01 — needs `BlockType` for type-safe item type strings

## Implementation Details

### Types

```typescript
// src/systems/inventory.ts
import { BlockType } from '../constants/blocks';

// Item types extend BlockType to include tools
export type ItemType = BlockType | 'shovel' | 'branch' | 'bucket';

export interface InventorySlot {
  itemType: ItemType;
  quantity: number;
}

export interface Inventory {
  slots: Array<InventorySlot | null>; // null = empty slot
  maxSlots: number;
  selectedSlotIndex: number;
  shovelLevel: 1 | 2 | 3 | 5; // dig quantity per tap; upgradeable via payment
}
```

### Singleton initialization

The player starts with three tools in the first three slots and 17 empty slots. Tools have quantity 1 and are non-stackable (enforced in `addItem`).

```typescript
const TOOL_TYPES = new Set<ItemType>(['shovel', 'branch', 'bucket']);
const MAX_STACK = 99; // max quantity per material slot

export const inventory: Inventory = {
  slots: [
    { itemType: 'shovel', quantity: 1 },
    { itemType: 'branch', quantity: 1 },
    { itemType: 'bucket', quantity: 1 },
    ...Array(17).fill(null),
  ],
  maxSlots: 20,
  selectedSlotIndex: 0,
  shovelLevel: 1,
};
```

### addItem

Finds an existing partial stack of the same type and fills it first; if none exists, finds the first empty slot. Tools cannot be stacked (each tool occupies exactly one slot at quantity 1).

```typescript
export function addItem(type: ItemType, qty: number): boolean {
  if (qty <= 0) return false;

  // Tools: find if already exists (only one allowed)
  if (TOOL_TYPES.has(type)) {
    const existing = inventory.slots.findIndex(s => s?.itemType === type);
    if (existing !== -1) return false; // already have this tool
    const emptyIdx = inventory.slots.findIndex(s => s === null);
    if (emptyIdx === -1) return false; // inventory full
    inventory.slots[emptyIdx] = { itemType: type, quantity: 1 };
    return true;
  }

  // Materials: fill existing stack first
  let remaining = qty;
  for (let i = 0; i < inventory.slots.length && remaining > 0; i++) {
    const slot = inventory.slots[i];
    if (slot?.itemType === type && slot.quantity < MAX_STACK) {
      const space = MAX_STACK - slot.quantity;
      const added = Math.min(space, remaining);
      slot.quantity += added;
      remaining -= added;
    }
  }

  // Then fill new empty slots
  while (remaining > 0) {
    const emptyIdx = inventory.slots.findIndex(s => s === null);
    if (emptyIdx === -1) break; // inventory full; drop remainder
    const added = Math.min(MAX_STACK, remaining);
    inventory.slots[emptyIdx] = { itemType: type, quantity: added };
    remaining -= added;
  }

  return remaining === 0; // true if all items were added
}
```

### removeItem

Returns `false` without modifying state if the player does not have enough. This prevents partial deductions on failed placements.

```typescript
export function removeItem(type: ItemType, qty: number): boolean {
  if (!hasItem(type, qty)) return false;

  let remaining = qty;
  for (let i = 0; i < inventory.slots.length && remaining > 0; i++) {
    const slot = inventory.slots[i];
    if (slot?.itemType === type) {
      const taken = Math.min(slot.quantity, remaining);
      slot.quantity -= taken;
      remaining -= taken;
      if (slot.quantity === 0) {
        inventory.slots[i] = null; // clear empty slot
      }
    }
  }

  return true;
}
```

### Query helpers

```typescript
export function findSlot(type: ItemType): number {
  return inventory.slots.findIndex(s => s?.itemType === type);
}

export function hasItem(type: ItemType, qty: number = 1): boolean {
  let total = 0;
  for (const slot of inventory.slots) {
    if (slot?.itemType === type) total += slot.quantity;
  }
  return total >= qty;
}

export function getSelectedItem(): InventorySlot | null {
  return inventory.slots[inventory.selectedSlotIndex] ?? null;
}

export function setSelectedSlot(index: number): void {
  if (index < 0 || index >= inventory.maxSlots) return;
  inventory.selectedSlotIndex = index;
}
```

### Shovel level management

```typescript
export function setShovelLevel(level: 1 | 2 | 3 | 5): void {
  inventory.shovelLevel = level;
}
```

## Key Constraints
- `inventory.slots` is always exactly `maxSlots` (20) elements long — never push or pop; use index assignment only.
- Tool slots (`shovel`, `branch`, `bucket`) cannot be stacked or removed via `removeItem` — the player always retains their tools.
- `removeItem` must be atomic: it checks `hasItem` first and only modifies state if the full quantity is available. Partial deductions are not allowed.
- `MAX_STACK` of 99 prevents UI overflow in the quantity badge (T14).
- `shovelLevel` uses a discrete union `1 | 2 | 3 | 5` (not a continuous number) because these are the exact upgrade tiers sold via in-app purchase (T21).
- This module has no side effects beyond mutating the singleton — it does not dispatch events, call Supabase, or touch the canvas.

## Acceptance Criteria
- [ ] `addItem('dry_sand', 10)` increases the dry_sand slot quantity by 10.
- [ ] `addItem('dry_sand', 50)` followed by `addItem('dry_sand', 50)` results in one slot of 99 and one slot of 1 (overflow into second slot).
- [ ] `removeItem('dry_sand', 5)` returns `true` and reduces the total dry_sand count by 5.
- [ ] `removeItem('dry_sand', 9999)` returns `false` and leaves the inventory unchanged.
- [ ] `hasItem('shovel')` returns `true` on a freshly created inventory.
- [ ] `getSelectedItem()` returns the shovel slot when `selectedSlotIndex` is 0.
- [ ] `setShovelLevel(3)` updates `inventory.shovelLevel` to 3.
- [ ] TypeScript compiles with zero errors; no `any` types.
