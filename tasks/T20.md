# T20 — [Phase 2] — Carving tool input & DB sync

## Overview
This task wires pointer/touch events to the carving system and establishes a local-first sync pattern: carving feedback is instant (applied each animation frame as the user drags), while Supabase writes are batched to a single `updateBlockMask` call on pointer release. It also handles the two interaction modalities — single tap toggles between engrave and surface mode, long-press locks into surface mode — giving players intuitive control over the branch tool.

## Scope
Modify `src/canvas/renderer.ts` to register `pointerdown`, `pointermove`, `pointerup`, and `pointercancel` event listeners on the canvas element. These listeners are only active when the inventory's selected item is `'branch'`. During an active drag over a block in `CASTLE_ZONE` (x ≥ 1200), convert the canvas-space pointer position to block-local coordinates `(localX, localY)` and call `applyCarving()` from T19. The renderer's next `requestAnimationFrame` tick naturally picks up the mutated `pixelMask` through the existing `renderBlock` call (T18), so no explicit redraw trigger is needed. On `pointerup`/`pointercancel`, call `updateBlockMask(block.id, block.pixelMask)` to flush the mask to Supabase. Modify `src/carving/carvingTool.ts` to add `toggleMode(tool)` (swaps `'engrave'` ↔ `'surface'`) and `setMode(tool, mode)` for explicit mode assignment. Long-press detection: if `pointerdown` is held without `pointermove` exceeding 5px for 500ms, fire a long-press event that switches to `'surface'` mode for that drag session.

## Output Files
- `src/canvas/renderer.ts (modify)` — adds pointer event listeners, coordinate conversion, calls `applyCarving` on drag, calls `updateBlockMask` on release
- `src/carving/carvingTool.ts (modify)` — adds `toggleMode`, `setMode` helpers

## Depends On
- T19 — provides `applyCarving`, `CarvingTool`, `DEFAULT_TOOL` that input handler calls
- T18 — the pixel-aware renderer loop picks up mutated masks automatically each frame; this task depends on T18 being the active render path
- T10 — provides `updateBlockMask(blockId, mask)` Supabase function used on pointer release

## Implementation Details

### Active carving session state
Store mutable session state outside React (or UI framework) state to avoid re-render overhead during drag:

```typescript
// src/canvas/renderer.ts (additions)

import { applyCarving, DEFAULT_TOOL, CarvingTool, toggleMode, setMode } from '../carving/carvingTool';
import { updateBlockMask } from '../supabase/blocks';
import { getInventorySelectedItem } from '../systems/inventory';
import { CASTLE_ZONE_START_X } from '../types/zones';

interface CarvingSession {
  active: boolean;
  targetBlock: Block | null;
  tool: CarvingTool;
  pointerDownPos: { x: number; y: number } | null;
  longPressTimer: ReturnType<typeof setTimeout> | null;
  movedBeyondThreshold: boolean;
}

const carvingSession: CarvingSession = {
  active: false,
  targetBlock: null,
  tool: { ...DEFAULT_TOOL },
  pointerDownPos: null,
  longPressTimer: null,
  movedBeyondThreshold: false,
};
```

### Coordinate conversion
Convert a canvas `PointerEvent` position to block-local `(localX, localY)` by subtracting the block's screen-space origin (which accounts for camera pan):

```typescript
function canvasToBlockLocal(
  e: PointerEvent,
  canvas: HTMLCanvasElement,
  block: Block,
  cameraX: number
): { localX: number; localY: number } {
  const rect = canvas.getBoundingClientRect();
  const canvasX = (e.clientX - rect.left) * (canvas.width / rect.width);
  const canvasY = (e.clientY - rect.top) * (canvas.height / rect.height);

  const screenBlockX = block.x - cameraX;
  const screenBlockY = block.y; // Y is not panned in this game

  return {
    localX: Math.floor(canvasX - screenBlockX),
    localY: Math.floor(canvasY - screenBlockY),
  };
}
```

### Hit-testing a block under pointer
Iterate the block list to find the topmost block (last in painter's order) whose bounding box contains the pointer:

```typescript
function getBlockAtCanvas(
  canvasX: number,
  canvasY: number,
  blocks: Block[],
  cameraX: number
): Block | null {
  // Iterate in reverse so topmost-rendered block wins
  for (let i = blocks.length - 1; i >= 0; i--) {
    const b = blocks[i];
    const sx = b.x - cameraX;
    const sy = b.y;
    if (
      canvasX >= sx && canvasX < sx + 32 &&
      canvasY >= sy && canvasY < sy + 32 &&
      b.x >= CASTLE_ZONE_START_X
    ) {
      return b;
    }
  }
  return null;
}
```

### Event listener registration
Register once when the renderer initialises:

```typescript
function registerCarvingListeners(
  canvas: HTMLCanvasElement,
  getBlocks: () => Block[],
  getCameraX: () => number
): void {
  const LONG_PRESS_MS = 500;
  const MOVE_THRESHOLD_PX = 5;

  canvas.addEventListener('pointerdown', (e: PointerEvent) => {
    if (getInventorySelectedItem() !== 'branch') return;

    const rect = canvas.getBoundingClientRect();
    const cx = (e.clientX - rect.left) * (canvas.width / rect.width);
    const cy = (e.clientY - rect.top) * (canvas.height / rect.height);

    const block = getBlockAtCanvas(cx, cy, getBlocks(), getCameraX());
    if (!block) return;

    carvingSession.active = true;
    carvingSession.targetBlock = block;
    carvingSession.pointerDownPos = { x: e.clientX, y: e.clientY };
    carvingSession.movedBeyondThreshold = false;
    carvingSession.tool = { ...DEFAULT_TOOL }; // reset to default each session

    // Schedule long-press detection
    carvingSession.longPressTimer = setTimeout(() => {
      if (!carvingSession.movedBeyondThreshold) {
        setMode(carvingSession.tool, 'surface');
      }
    }, LONG_PRESS_MS);

    canvas.setPointerCapture(e.pointerId);
  });

  canvas.addEventListener('pointermove', (e: PointerEvent) => {
    if (!carvingSession.active || !carvingSession.targetBlock) return;

    // Check if pointer has moved beyond threshold (kills long-press to surface)
    if (carvingSession.pointerDownPos) {
      const dx = e.clientX - carvingSession.pointerDownPos.x;
      const dy = e.clientY - carvingSession.pointerDownPos.y;
      if (Math.sqrt(dx * dx + dy * dy) > MOVE_THRESHOLD_PX) {
        carvingSession.movedBeyondThreshold = true;
      }
    }

    const { localX, localY } = canvasToBlockLocal(
      e,
      canvas,
      carvingSession.targetBlock,
      getCameraX()
    );
    applyCarving(carvingSession.targetBlock, localX, localY, carvingSession.tool);
    // No explicit redraw — rAF loop in renderFrame() picks up mutation next tick
  });

  const endCarving = async (e: PointerEvent) => {
    if (!carvingSession.active) return;
    if (carvingSession.longPressTimer) clearTimeout(carvingSession.longPressTimer);

    const block = carvingSession.targetBlock;
    carvingSession.active = false;
    carvingSession.targetBlock = null;

    if (block) {
      await updateBlockMask(block.id, block.pixelMask);
    }
  };

  canvas.addEventListener('pointerup', endCarving);
  canvas.addEventListener('pointercancel', endCarving);
}
```

### Single-tap mode toggle
A tap is defined as a `pointerdown` followed by `pointerup` with no `pointermove` beyond 5px and duration < 300ms. Detect this inside the `pointerup` handler:

```typescript
// Inside endCarving, before the await:
const downTime = /* stored in carvingSession.downTimestamp */ Date.now() - carvingSession.downTimestamp;
if (!carvingSession.movedBeyondThreshold && downTime < 300) {
  toggleMode(carvingSession.tool); // flip mode for next session
  DEFAULT_TOOL.mode = carvingSession.tool.mode; // persist as new default
}
```

export function toggleMode(tool: CarvingTool): void {
  tool.mode = tool.mode === 'engrave' ? 'surface' : 'engrave';
}

export function setMode(tool: CarvingTool, mode: CarvingMode): void {
  tool.mode = mode;

## Key Constraints
- Carving only applies when the selected inventory item is `'branch'` — ignore all pointer events otherwise.
- Only blocks in `CASTLE_ZONE` (block `x >= 1200`) are carveable; blocks in BEACH or SHORELINE must not respond.
- `updateBlockMask` is called **once** on pointer release, not per-frame — this is the local-first design.
- `setPointerCapture` ensures the drag continues even if the pointer leaves the block's bounding box.
- Long-press (500ms without movement) activates surface mode for that drag session only; the next drag resets to engrave.
- Single tap (< 300ms, < 5px movement) toggles the persistent default mode.
- Never call `updateBlockMask` if `carvingSession.targetBlock` is null (e.g., drag started outside a block).

## Acceptance Criteria
- [ ] Dragging with the branch tool over a CASTLE_ZONE block updates the block's `pixelMask` and the change is visible in the next rendered frame.
- [ ] Releasing the pointer triggers exactly one `updateBlockMask` call with the block's final mask state.
- [ ] Dragging over a block in BEACH or SHORELINE zone has no effect (no carving, no DB call).
- [ ] Long-press (500ms, no movement) switches the session to surface mode; carved pixels become 0.0.
- [ ] Single tap (< 300ms) toggles the current carving mode, persisting for the next drag.
- [ ] Pointer capture prevents carving from stopping when the pointer briefly exits the block mid-drag.
- [ ] `toggleMode` and `setMode` correctly update the `mode` property of the passed `CarvingTool` object.
- [ ] `pointercancel` (e.g., system interrupt) flushes the mask to Supabase, preventing loss of work.
