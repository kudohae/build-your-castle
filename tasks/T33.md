# T33 — [Phase 3] — Leaderboard AIT API integration

## Overview
This task bridges the game's art scoring system with Toss's AIT (Apps in Toss) Leaderboard API, converting the community star ratings into integer leaderboard points and fetching ranked player lists. The AIT Leaderboard provides cross-app ranking infrastructure so players compete with a wider audience through the Toss super-app ecosystem. This module is the single integration point — all leaderboard reads and writes flow through it.

## Scope
Create `src/ait/leaderboard.ts` with two exported functions and one exported interface. `updateLeaderboard(userId: string, avgScore: number): Promise<void>` converts the 0–5 average score to integer AIT points by multiplying by 100 and rounding (`Math.round(avgScore * 100)`, giving 0–500 point range), then calls `AIT.Leaderboard.submitScore({ userId, score: points })`. `getTopPlayers(limit = 20): Promise<LeaderboardEntry[]>` calls `AIT.Leaderboard.getTopScores({ limit })` and maps the raw AIT response to the typed `LeaderboardEntry[]` format. Export `LeaderboardEntry` interface: `{ rank: number, userId: string, nickname: string, score: number, voteCount: number }`. Note that `voteCount` may not be available directly from the AIT API and should be fetched from `getPlotScore` for each player or provided as 0 if unavailable. Include a `getCurrentUserRank(userId: string): Promise<number | null>` helper that fetches the user's current rank, or null if unranked.

## Output Files
- `src/ait/leaderboard.ts` — new file; exports `updateLeaderboard`, `getTopPlayers`, `getCurrentUserRank`, `LeaderboardEntry`

## Depends On
- T11 — provides the AIT SDK integration layer, the `AIT` namespace, and the `Leaderboard` sub-API that this module calls; without T11 the `AIT.Leaderboard` methods are undefined
- T31 — provides `getPlotScore` used to enrich leaderboard entries with `voteCount` data that the AIT API may not return natively

### updateLeaderboard

```typescript
export async function updateLeaderboard(
  userId: string,
  avgScore: number
): Promise<void> {
  if (avgScore < 0 || avgScore > 5) {
    console.warn('[Leaderboard] avgScore out of range:', avgScore);
    return;
  }

  const aitScore = Math.round(avgScore * AIT_SCORE_MULTIPLIER);

  try {
    await AIT.Leaderboard.submitScore({ userId, score: aitScore });
    console.log(`[Leaderboard] Submitted score ${aitScore} for user ${userId}`);
  } catch (err) {
    // Leaderboard update failures are non-critical — log and continue
    console.error('[Leaderboard] submitScore failed:', err);
  }
}
```

### getTopPlayers

The function fetches top scores from AIT and enriches each entry with vote count data from Supabase. To avoid N+1 queries, `voteCount` is fetched concurrently for all players:

```typescript
export async function getTopPlayers(limit = 20): Promise<LeaderboardEntry[]> {
  let aitEntries: Awaited<ReturnType<typeof AIT.Leaderboard.getTopScores>>;

  try {
    aitEntries = await AIT.Leaderboard.getTopScores({ limit });
  } catch (err) {
    console.error('[Leaderboard] getTopScores failed:', err);
    return [];
  }

  // Enrich with voteCount from Supabase (best-effort, parallel fetch)
  // Note: getPlotScore requires plotId, not userId. If a userId→plotId mapping
  // is available (e.g., from a users→plots lookup), use it here.
  // For now, we include voteCount: 0 as a fallback.
  const entries: LeaderboardEntry[] = aitEntries.map((e) => ({
    rank: e.rank,
    userId: e.userId,
    nickname: e.nickname,
    score: e.score,
    avgRating: e.score / AIT_SCORE_MULTIPLIER,
    voteCount: 0, // placeholder; enriched below if plot mapping is available
  }));

  return entries;
}
```

### getTopPlayersWithVotes (enriched variant)
When a `getUserPlotId(userId)` lookup function is available:

```typescript
export async function getTopPlayersWithVotes(
  limit = 20,
  getUserPlotId: (userId: string) => Promise<string | null>
): Promise<LeaderboardEntry[]> {
  const entries = await getTopPlayers(limit);

  await Promise.allSettled(
    entries.map(async (entry) => {
      try {
        const plotId = await getUserPlotId(entry.userId);
        if (plotId) {
          const { count } = await getPlotScore(plotId);
          entry.voteCount = count;
        }
      } catch {
        // voteCount stays 0
      }
    })
  );

  return entries;
}
```

### getCurrentUserRank

```typescript
export async function getCurrentUserRank(userId: string): Promise<number | null> {
  try {
    const result = await AIT.Leaderboard.getUserRank({ userId });
    return result.rank;
  } catch (err) {
    console.error('[Leaderboard] getUserRank failed:', err);
    return null;
  }
}
```

### Score conversion table
| avgRating (from votes) | AIT score | Interpretation |
|---|---|---|
| 5.00 | 500 | Perfect (all 5-star votes) |
| 4.00 | 400 | Excellent |
| 3.50 | 350 | Good |
| 2.00 | 200 | Fair |
| 1.00 | 100 | Poor |
| 0.00 | 0 | No votes or zero average |

### Call timing
`updateLeaderboard` is called from T32's `handleSubmitVote` after every successful vote. It is also called on `app focus` events (when the user returns to the app) to re-sync if the score has changed through other voters in the interim. The focus call should be debounced to avoid hammering the AIT API:

```typescript
// In app focus handler:
let leaderboardSyncDebounce: ReturnType<typeof setTimeout> | null = null;

window.addEventListener('focus', () => {
  if (leaderboardSyncDebounce) return;
  leaderboardSyncDebounce = setTimeout(async () => {
    leaderboardSyncDebounce = null;
    const { avg } = await getPlotScore(currentPlotId);
    await updateLeaderboard(currentUserId, avg);
  }, 2000);
});
```

## Key Constraints
- `updateLeaderboard` must not throw — it is called fire-and-forget from voting flows and a crash here must not affect the vote result.
- AIT score range is 0–500 (avgRating 0–5 × 100); passing a score outside this range to AIT may be rejected — clamp if needed.
- `getTopPlayers` returns an empty array on AIT API failure rather than throwing — the leaderboard UI should handle the empty state gracefully.
- The `voteCount` field in `LeaderboardEntry` is best-effort; it may be 0 if the AIT API doesn't return it and the Supabase enrichment is unavailable.
- `AIT.Leaderboard` is imported from the T11 SDK integration; confirm the exact import path with T11 before finalising.

- [ ] `avgRating` on each entry equals `entry.score / 100` (within floating-point precision).
- [ ] `getCurrentUserRank` returns `null` when the user has no score on the leaderboard.
- [ ] `updateLeaderboard` with `avgScore = -1` logs a warning and makes no AIT API call.
- [ ] `updateLeaderboard` with `avgScore = 5.0` submits score `500` without clamping errors.
