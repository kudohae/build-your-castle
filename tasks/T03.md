# T03 тАФ [Phase 1] тАФ Canvas renderer core

## Overview
This task creates the central rendering engine that drives every visual frame of the game. It mounts a full-screen `<canvas>` element, starts and manages the `requestAnimationFrame` loop, and defines a layered draw order (background тЖТ zones тЖТ blocks тЖТ UI) with an open registration API so other modules can inject their draw calls without coupling to this file.

## Scope
Implement `src/canvas/renderer.ts` with the following responsibilities: mount a `<canvas>` to `document.body` sized to `window.innerWidth ├Ч window.innerHeight`; resize the canvas on `window.resize`; maintain a `requestAnimationFrame` loop with a delta-time accumulator; draw zone color bands from `MAP_ZONES` as the world background; maintain a list of block descriptors and draw each as a solid colored rectangle (using `BLOCK_PHYSICS[block.type].color`) offset by the camera; invoke all registered UI drawer callbacks after world drawing is complete. Export `start()`, `stop()`, `addBlock(block)`, `removeBlock(id)`, `registerUIDrawer(fn)`, `render()` (for manual frame trigger in tests), and `getCanvas()` / `getContext()` accessors. Camera offset is read from `src/canvas/camera.ts` (imported) тАФ this file does not own camera state.

## Output Files
- `src/canvas/renderer.ts` тАФ full rendering pipeline, frame loop, block registry, UI drawer registry

## Depends On
- T01 тАФ needs `MAP_ZONES`, `BLOCK_PHYSICS`, `Block` type, `BLOCK_SIZE`

## Implementation Details

### Canvas mounting and resize handling

```typescript
// src/canvas/renderer.ts
import { MAP_ZONES, GROUND_Y, MAP_HEIGHT } from '../constants/maps';
import { BLOCK_PHYSICS, BLOCK_SIZE, Block } from '../constants/blocks';
import { getOffset, getScale } from './camera'; // implemented in T04

const canvas = document.createElement('canvas');
const ctx = canvas.getContext('2d')!;
document.body.style.margin = '0';
document.body.style.overflow = 'hidden';
document.body.appendChild(canvas);

function resize() {
  canvas.width = window.innerWidth;
  canvas.height = window.innerHeight;
}
resize();
window.addEventListener('resize', resize);
```

### Block registry

Blocks are stored in a `Map` keyed by `block.id` for O(1) add/remove. The draw call iterates the map's values each frame.

```typescript
const blockMap = new Map<string, Block>();

export function addBlock(block: Block): void {
  blockMap.set(block.id, block);
}

export function removeBlock(id: string): void {
  blockMap.delete(id);
}
```

### UI drawer registry

Any module can register a draw callback. Callbacks receive `ctx` and `deltaMs` and are called after all world-space drawing, in insertion order.

```typescript
type UIDrawer = (ctx: CanvasRenderingContext2D, deltaMs: number) => void;
const uiDrawers: UIDrawer[] = [];

export function registerUIDrawer(fn: UIDrawer): void {
  uiDrawers.push(fn);
}
```

### Render loop and layered draw order

```typescript
let rafId = 0;
let lastTimestamp = 0;

export function render(timestamp = performance.now()): void {
  const deltaMs = Math.min(timestamp - lastTimestamp, 50); // cap at 50ms to prevent spiral
  lastTimestamp = timestamp;

  const { offsetX, offsetY } = getOffset();
  const scale = getScale();

  // 1. Clear
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  // 2. Zone background bands (world-space, camera-transformed)
  ctx.save();
  ctx.translate(-offsetX, -offsetY);
  ctx.scale(scale, scale);

  for (const zone of MAP_ZONES) {
    const endX = isFinite(zone.width) ? zone.startX + zone.width : zone.startX + 4000;
    // Sky band
    ctx.fillStyle = zone.bgColor;
    ctx.fillRect(zone.startX, 0, endX - zone.startX, GROUND_Y);
    // Ground band
    ctx.fillStyle = zone.groundColor;
    ctx.fillRect(zone.startX, GROUND_Y, endX - zone.startX, MAP_HEIGHT - GROUND_Y);
  }

  // 3. Blocks
  for (const block of blockMap.values()) {
    const physics = BLOCK_PHYSICS[block.type];
    ctx.fillStyle = block.isWet
      ? BLOCK_PHYSICS['wet_sand'].color  // tint wet blocks
      : physics.color;
    ctx.fillRect(block.x, block.y, BLOCK_SIZE, BLOCK_SIZE);
  }

  ctx.restore();

  // 4. UI layer (screen-space, no camera transform)
  for (const drawer of uiDrawers) {
    drawer(ctx, deltaMs);
  }
}

export function start(): void {
  lastTimestamp = performance.now();
  function loop(ts: number) {
    render(ts);
    rafId = requestAnimationFrame(loop);
  }
  rafId = requestAnimationFrame(loop);
}

export function stop(): void {
  cancelAnimationFrame(rafId);
  rafId = 0;
}
```

### Exported accessors

```typescript
export function getCanvas(): HTMLCanvasElement { return canvas; }
export function getContext(): CanvasRenderingContext2D { return ctx; }
```

## Key Constraints
- The renderer must not own camera state тАФ it imports read-only accessors from `camera.ts` (T04). This separation allows the camera to be unit-tested independently.
- Delta-time is capped at 50ms to prevent the "spiral of death" that occurs when the tab is backgrounded and then foregrounded.
- UI drawers run in screen space (no camera transform applied); they must not apply their own `ctx.save()/restore()` transforms to world space.
- When `CASTLE_ZONE.width` is `Infinity`, the renderer uses a fallback width of 4000px for the background band draw тАФ this must be revisited if the map grows beyond that.
- `addBlock` / `removeBlock` are the only mutation points for the block registry; no other module should hold a direct reference to `blockMap`.
- The canvas must have `touch-action: none` set in CSS (or via `canvas.style.touchAction = 'none'`) to prevent the browser from intercepting touch events for scrolling.
- `render()` is exported as a named function to allow unit tests to call a single frame without starting the RAF loop.
