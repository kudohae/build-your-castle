# T02 — [Phase 1] — Supabase client initialization

## Overview
This task creates the single shared Supabase client instance that all database and realtime modules import. Centralizing instantiation here prevents multiple connections from being opened and ensures environment variables are validated at startup rather than silently failing at query time.

## Scope
Create `src/supabase/client.ts` which reads `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` from `import.meta.env`, validates they are present, and calls `createClient()` from `@supabase/supabase-js`. The resulting `supabase` constant is exported as the module's default export and as a named export. Provide a `.env.example` file at the project root with commented placeholder values so developers can onboard quickly. No logic beyond instantiation belongs in this file — queries, subscriptions, and auth live in sibling modules.

## Output Files
- `src/supabase/client.ts` — Supabase singleton client, the single import target for all DB/realtime operations
- `.env.example` — environment variable template with comments explaining each variable

## Depends On
- none

## Implementation Details

### Client initialization with env validation

Vite exposes environment variables through `import.meta.env`. We guard against missing values at module load time so misconfiguration surfaces immediately rather than producing cryptic network errors later.

```typescript
// src/supabase/client.ts
import { createClient, SupabaseClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL as string | undefined;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY as string | undefined;

if (!supabaseUrl || !supabaseAnonKey) {
  throw new Error(
    '[Supabase] Missing required environment variables.\n' +
    'Copy .env.example to .env.local and fill in VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY.'
  );
}

export const supabase: SupabaseClient = createClient(supabaseUrl, supabaseAnonKey, {
  auth: {
    // Toss login manages session externally via AIT SDK (see src/ait/auth.ts).
    // Disable Supabase's built-in session persistence to avoid conflicts.
    persistSession: false,
    autoRefreshToken: false,
  },
  realtime: {
    params: {
      eventsPerSecond: 10, // rate-limit realtime events; tunable
    },
  },
});

export default supabase;
```

### Environment variable template

```bash
# .env.example
# Copy this file to .env.local (never commit .env.local to git)

# Supabase project URL — found in: Supabase Dashboard → Settings → API → Project URL
VITE_SUPABASE_URL=https://your-project-id.supabase.co

# Supabase anonymous (public) key — found in: Supabase Dashboard → Settings → API → anon / public
# This key is safe to expose in client-side code (RLS enforces access control)
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

### Usage pattern in sibling modules

Every other file in `src/supabase/` imports the singleton like this — they never call `createClient` themselves:

```typescript
// e.g., src/supabase/blocks.ts
import { supabase } from './client';

export async function loadBlocks(plotId: string) {
  const { data, error } = await supabase
    .from('blocks')
    .select('*')
    .eq('plot_id', plotId);
  // ...
}
```

## Key Constraints
- `persistSession: false` is mandatory — Toss login (AIT SDK) owns the auth lifecycle; letting Supabase independently manage sessions would cause conflicts.
- The `VITE_` prefix is required for Vite to expose variables to the browser bundle; server-only variables without this prefix will be `undefined` at runtime.
- This file must be a plain module (no React, no side-effects beyond the validation guard and `createClient` call) so it can be imported from any context including Web Workers if needed later.
- `.env.example` must be committed to git; `.env.local` must be listed in `.gitignore`.
- Do not export multiple clients — all modules share the single `supabase` export to avoid exhausting connection pool slots.
- `eventsPerSecond: 10` is a starting budget; the realtime system (T25) may tune this value once concurrent subscription counts are known.
