# T07 — [Phase 1] — Matter.js physics engine setup

## Overview
This task bootstraps the Matter.js simulation that gives blocks realistic weight, friction, and collision behavior. The engine is the single source of physical truth — all block bodies are added here, and its `update()` function drives frame-accurate physics that the renderer then mirrors. Getting gravity, ground, and boundary bodies right in this task prevents correctness issues in every downstream physics task.

## Scope
Initialize a Matter.js `Engine` with `gravity.y = 1.5` (slightly stronger than Earth for snappier sand physics). Create static boundary bodies: a ground rectangle at `GROUND_Y`, a left wall at `x = -10`, and a right wall at `x = MAP_WIDTH + 10` (where `MAP_WIDTH = 3000`). Export `update(delta: number)` to be called each render frame, the engine and world instances for external body registration, and an `onCollision(callback)` hook that forwards the Matter.js `collisionStart` event. Also export `MAP_WIDTH` for use by the camera and chunk systems.

## Output Files
- `src/physics/engine.ts` — Matter.js engine initialization, ground/wall bodies, frame update function, collision hook

## Depends On

## Implementation Details

### Engine initialization

```typescript
// src/physics/engine.ts
import Matter from 'matter-js';
import { GROUND_Y, MAP_HEIGHT } from '../constants/maps';

const { Engine, World, Bodies, Events, Runner } = Matter;

export const MAP_WIDTH = 3000; // BEACH(800) + SHORELINE(400) + initial CASTLE slots

export const engine = Engine.create({
  gravity: { x: 0, y: 1.5, scale: 0.001 },
  // scale: 0.001 converts pixel/ms² to the unit system Matter.js expects
});

export const world = engine.world;
```

### Static boundary bodies

The ground body is a wide, thin rectangle whose center is placed 10px below `GROUND_Y` so its top surface aligns exactly with `GROUND_Y`. Left and right walls extend the full world height to prevent blocks from leaving the simulation laterally.

```typescript
const ground = Bodies.rectangle(
  MAP_WIDTH / 2,      // center x
  GROUND_Y + 10,      // center y (top surface = GROUND_Y)
  MAP_WIDTH * 4,      // extra-wide to cover scrolled-out edges
  20,                 // height
  { isStatic: true, label: 'ground', friction: 0.5 }
);

const leftWall = Bodies.rectangle(
  -10,
  MAP_HEIGHT,
  20,
  MAP_HEIGHT * 2,
  { isStatic: true, label: 'left_wall' }
);

const rightWall = Bodies.rectangle(
  MAP_WIDTH + 10,
  MAP_HEIGHT,
  20,
  MAP_HEIGHT * 2,
  { isStatic: true, label: 'right_wall' }
);

World.add(world, [ground, leftWall, rightWall]);
```

### Frame update

Called from the renderer's RAF loop. `delta` is the elapsed milliseconds since the last frame, capped by the renderer at 50ms.

```typescript
export function update(delta: number): void {
  Engine.update(engine, delta);
}
```

### Collision hook

Exposes Matter.js collision events via a clean subscription API. The callback receives an array of collision pairs, each containing the two bodies involved.

```typescript
export type CollisionPair = {
  bodyA: Matter.Body;
  bodyB: Matter.Body;
};

export type CollisionCallback = (pairs: CollisionPair[]) => void;
const collisionListeners: CollisionCallback[] = [];

Events.on(engine, 'collisionStart', (event: Matter.IEventCollision<Matter.Engine>) => {
  const pairs: CollisionPair[] = event.pairs.map(pair => ({
    bodyA: pair.bodyA,
    bodyB: pair.bodyB,
  }));
  for (const cb of collisionListeners) {
    cb(pairs);
  }
});

export function onCollision(callback: CollisionCallback): void {
  collisionListeners.push(callback);
}
```

### Integration with renderer loop

In `renderer.ts`, the `render()` function should call `update(deltaMs)` each frame before drawing:

```typescript
// In renderer.ts (renderer imports physics engine):
import { update as physicsUpdate } from '../physics/engine';
// ...
// Inside render():
physicsUpdate(deltaMs);
```

## Key Constraints
- `gravity.y = 1.5` with `scale: 0.001` is tuned for the 32px block size at 60fps — do not change without re-testing block stacking stability.
- The ground body's center must be at `GROUND_Y + 10` so its top edge aligns exactly with `GROUND_Y`; any mismatch causes blocks to float or clip through the ground.
- `MAP_WIDTH * 4` ground width is intentionally larger than the visible world to ensure the ground extends under chunks loaded via scrolling.
- `collisionListeners` is not exposed directly — only `onCollision()` is exported to prevent listeners from being accidentally cleared.
- `update()` must be called exactly once per render frame; calling it multiple times per frame will advance the simulation faster than real time.
- Do not start Matter.js's built-in `Runner` — the renderer's RAF loop owns the update cadence.
