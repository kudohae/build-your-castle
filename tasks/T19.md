# T19 — [Phase 2] — Carving tool logic

## Overview
This task implements the core pixel manipulation logic that allows players to sculpt sand castle blocks using a branch tool. The carving system operates directly on a block's `pixelMask` Float32Array, enabling two distinct modes: surface carving (removes material entirely) and engraving (leaves a darkened imprint). This module is purely computational — it has no canvas, physics, or network side effects — making it easy to test in isolation and compose with input handling (T20) and rendering (T18).

## Scope
Create `src/carving/carvingTool.ts` exporting `applyCarving(block, localX, localY, tool)`, the `CarvingTool` interface, `CarvingMode` union type, and the `DEFAULT_TOOL` constant. The `applyCarving` function determines which pixels fall within a circle of radius `tool.brushSize` centered at `(localX, localY)` in block-local coordinates (0–31 range). For each pixel `(px, py)` in that circle: in `'surface'` mode, set the mask value to `0.0` (fully removes the pixel); in `'engrave'` mode, only modify pixels whose current value is greater than `0.0` (i.e., not already transparent) and set them to `0.5`. Helper `setPixel(mask, px, py, value)` writes a single pixel safely (bounds-checks before writing). Helper `getPixel(mask, px, py)` reads safely (returns 0.0 for out-of-bounds). The circular region is computed with standard `dx*dx + dy*dy <= r*r` integer arithmetic — no floating-point sqrt needed.

## Output Files
- `src/carving/carvingTool.ts` — new file; exports `CarvingTool`, `CarvingMode`, `DEFAULT_TOOL`, `applyCarving`, `setPixel`, `getPixel`

## Implementation Details

### Types and constants

```typescript
// src/carving/carvingTool.ts

import { Block, BLOCK_SIZE } from '../types/block';

export type CarvingMode = 'surface' | 'engrave';

export interface CarvingTool {
  mode: CarvingMode;
  brushSize: number; // radius in pixels, default 3
}

export const DEFAULT_TOOL: CarvingTool = {
  mode: 'engrave',
  brushSize: 3,
};
```

### Low-level pixel helpers
`setPixel` and `getPixel` use the row-major index formula. Out-of-bounds writes are silently dropped; out-of-bounds reads return 0.0. Both are exported so that T20's input handler can perform boundary checks before calling `applyCarving`.

```typescript
export function setPixel(
  mask: Float32Array,
  px: number,
  py: number,
  value: number
): void {
  if (px < 0 || px >= BLOCK_SIZE || py < 0 || py >= BLOCK_SIZE) return;
  mask[py * BLOCK_SIZE + px] = value;
}

export function getPixel(
  mask: Float32Array,
  px: number,
  py: number
): number {
  if (px < 0 || px >= BLOCK_SIZE || py < 0 || py >= BLOCK_SIZE) return 0.0;
  return mask[py * BLOCK_SIZE + px];
}
```

### applyCarving — circular brush
The function floors `localX` and `localY` to integer pixel coordinates, then iterates all integer `(px, py)` pairs within the bounding box `[cx-r, cx+r] × [cy-r, cy+r]`, using the circle equation to skip corners.

```typescript
export function applyCarving(
  block: Block,
  localX: number,
  localY: number,
  tool: CarvingTool = DEFAULT_TOOL
): void {
  const cx = Math.floor(localX);
  const cy = Math.floor(localY);
  const r = tool.brushSize;
  const r2 = r * r;

  for (let py = cy - r; py <= cy + r; py++) {
    for (let px = cx - r; px <= cx + r; px++) {
      const dx = px - cx;
      const dy = py - cy;
      if (dx * dx + dy * dy > r2) continue; // outside circle

      if (tool.mode === 'surface') {
        setPixel(block.pixelMask, px, py, 0.0);
      } else {
        // engrave: only affect solid or previously-solid pixels
        const current = getPixel(block.pixelMask, px, py);
        if (current > 0.0) {
          setPixel(block.pixelMask, px, py, 0.5);
        }
      }
    }
  }
}
```

### Brush size guidelines
| `brushSize` | Diameter | Use case |
|-------------|----------|----------|
| 1 | 1 px (single pixel) | Fine detail engraving |
| 3 | ~6 px circle | Default carving |
| 5 | ~10 px circle | Rough shaping |
| 8 | ~16 px circle | Large surface removal |

Brush sizes above 15 would exceed the 32px block boundary on all sides and are nonsensical — enforce a cap of 15 in `applyCarving` with an early clamp: `const r = Math.min(tool.brushSize, 15)`.

### Edge cases: brush center outside block
`localX`/`localY` are converted from canvas coordinates and may be negative or > 31 if the pointer is near a block edge. The `setPixel`/`getPixel` bounds checks handle this gracefully — pixels outside the block boundary are simply skipped.

### CarvingMode semantics
- `'surface'`: Pixels are set to `0.0`. The block gains a physical hole — air. Used for digging channels, windows, doors.
- `'engrave'`: Pixels are set to `0.5` (if not already transparent). No material is removed; a decorative groove is created. Used for patterns, text, and ornamentation.

In `'engrave'` mode, a pixel that is already `0.0` (already carved through) is intentionally left alone — engraving into a hole makes no semantic sense.

## Key Constraints
- `pixelMask` is a `Float32Array(1024)` — always validate index before write.
- `localX` and `localY` must be in the range 0–31 (block-local coordinates). Conversion from canvas coords is the caller's (T20's) responsibility.
- `applyCarving` must **not** trigger any re-render or network call; it is a pure data mutation.
- `brushSize` is clamped to `[1, 15]` inside the function to prevent degenerate loops.
- The function mutates `block.pixelMask` in place — it does not return a new array.
- Do not import from `src/canvas/` or `src/supabase/` to keep this module dependency-free.
