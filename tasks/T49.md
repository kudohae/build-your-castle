# T49 — [Phase 4] — Breakwater Item & DB

## Overview
This task builds the client-side breakwater management module that lets the game check whether a plot is currently protected against erosion, calculate the protection expiry date, and schedule a push notification reminder 3 days before expiry. It also modifies the erosion edge function to reference the same breakwater logic inline.

## Scope
Create `src/systems/erosion.ts` with three exported functions: `checkBreakwater(plotId)` queries the `breakwaters` table for a row where `plot_id = plotId AND expires_at > NOW()`, returning `boolean`; `getBreakwaterExpiry(plotId)` returns `Date | null` — the `expires_at` timestamp of the active breakwater or `null` if none; `scheduleExpiryAlert(plotId)` calls `getBreakwaterExpiry`, computes the 3-days-before-expiry timestamp (`expiresAt - 3 days`), and uses `setTimeout` to call `push.breakwaterExpiryAlert()` at that future time if the alert time is in the future. Modify `supabase/functions/erosion/index.ts` to replace its inline breakwater check with a call to the same query logic (duplicated as a local function, since Deno edge functions cannot import from the client src tree). Define the `breakwaters` table schema here if not already migrated.

## Output Files
- `src/systems/erosion.ts` — new file; `checkBreakwater`, `getBreakwaterExpiry`, `scheduleExpiryAlert`
- `supabase/functions/erosion/index.ts (modify)` — replaces ad-hoc breakwater check with the formalized query pattern; adds logging for expiry dates

## Depends On

## Implementation Details

### breakwaters Table Schema

```sql
-- supabase/migrations/YYYYMMDDHHMMSS_breakwaters.sql

CREATE TABLE IF NOT EXISTS breakwaters (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  plot_id     UUID NOT NULL REFERENCES plots(id) ON DELETE CASCADE,
  expires_at  TIMESTAMPTZ NOT NULL,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (plot_id)   -- one active breakwater per plot
);

-- RLS: users can read their own breakwater
CREATE POLICY "own_breakwater_select" ON breakwaters
  FOR SELECT USING (user_id = auth.uid());

-- RLS: allow insert/update for own user (called by applyPurchase via T42)
CREATE POLICY "own_breakwater_write" ON breakwaters
  FOR ALL USING (user_id = auth.uid());

-- Index for expiry queries
CREATE INDEX idx_breakwaters_expires ON breakwaters (plot_id, expires_at);
```

### src/systems/erosion.ts

```typescript
// src/systems/erosion.ts

import { supabase } from '../supabase/client';
import { breakwaterExpiryAlert } from '../ait/push';

const THREE_DAYS_MS = 3 * 24 * 60 * 60 * 1000;

// Active expiry alert handle — one per user session
let expiryAlertTimeoutId: ReturnType<typeof setTimeout> | null = null;

/**
 * Returns true if the given plot has an active (non-expired) breakwater.
 */
export async function checkBreakwater(plotId: string): Promise<boolean> {
  const { data, error } = await supabase
    .from('breakwaters')
    .select('id')
    .eq('plot_id', plotId)
    .gt('expires_at', new Date().toISOString())
    .maybeSingle();

  if (error) {
    console.warn('[erosion] checkBreakwater query failed:', error.message);
    return false; // fail open: assume unprotected if query fails
  }

  return data !== null;
}

/**
 * Returns the expiry Date of the active breakwater, or null if none.
 */
export async function getBreakwaterExpiry(plotId: string): Promise<Date | null> {
  const { data, error } = await supabase
    .from('breakwaters')
    .select('expires_at')
    .eq('plot_id', plotId)
    .gt('expires_at', new Date().toISOString())
    .order('expires_at', { ascending: false })
    .maybeSingle();

  if (error || !data) return null;

  return new Date(data.expires_at);
}

/**
 * Schedules a push notification 3 days before the active breakwater expires.
 * Cancels any previously scheduled expiry alert before setting a new one.
 */
export async function scheduleExpiryAlert(plotId: string): Promise<void> {
  // Cancel any existing alert
  if (expiryAlertTimeoutId !== null) {
    clearTimeout(expiryAlertTimeoutId);
    expiryAlertTimeoutId = null;
  }

  const expiry = await getBreakwaterExpiry(plotId);
  if (!expiry) {
    console.debug('[erosion] No active breakwater — expiry alert not scheduled.');
    return;
  }

  const alertTime = expiry.getTime() - THREE_DAYS_MS;
  const delay = alertTime - Date.now();

  if (delay <= 0) {
    // Less than 3 days remain — alert immediately
    console.debug('[erosion] Breakwater expiry < 3 days away — alerting now.');
    await breakwaterExpiryAlert();
    return;
  }

  expiryAlertTimeoutId = setTimeout(async () => {
    expiryAlertTimeoutId = null;
    await breakwaterExpiryAlert();
  }, delay);

  const alertDate = new Date(alertTime);
  console.debug(
    `[erosion] Breakwater expiry alert scheduled for ` +
    `${alertDate.toLocaleString('ko-KR')} ` +
    `(${Math.round(delay / 86400000)}일 후)`
  );
}

/**
 * Cancels any pending breakwater expiry alert timeout.
 * Call on session end or when a new breakwater is purchased.
 */
export function cancelExpiryAlert(): void {
  if (expiryAlertTimeoutId !== null) {
    clearTimeout(expiryAlertTimeoutId);
    expiryAlertTimeoutId = null;
  }
}
```

### Erosion Edge Function Modification

The edge function already implements an inline breakwater check (T48). Modify it to match the canonical query pattern and add expiry logging:

```typescript
// supabase/functions/erosion/index.ts — replace checkActiveBreakwater:

async function checkActiveBreakwater(plotId: string): Promise<{
  protected: boolean;
  expiresAt: string | null;
}> {
  const { data } = await adminClient
    .from('breakwaters')
    .select('expires_at')
    .eq('plot_id', plotId)
    .gt('expires_at', new Date().toISOString())
    .order('expires_at', { ascending: false })
    .maybeSingle();

  if (!data) return { protected: false, expiresAt: null };

  const daysLeft = Math.ceil(
    (new Date(data.expires_at).getTime() - Date.now()) / 86400000
  );
  console.log(
    `[erosion] Plot ${plotId} has active breakwater — expires in ${daysLeft} day(s).`
  );

  return { protected: true, expiresAt: data.expires_at };
}
```

## Key Constraints
- `checkBreakwater` fails open (returns `false`) on DB error — the edge function will then proceed with erosion on a protected plot rather than silently skipping. This is a conservative design that errs toward correct protection. Consider failing closed (returning `true`) in a security-sensitive context.
- Only one `expiryAlertTimeoutId` may be active per session; `scheduleExpiryAlert` always cancels the previous one before scheduling a new one.
- The 3-days-before-expiry threshold is computed from `getBreakwaterExpiry()` result at call time, not from purchase time — this handles mid-session expiry updates correctly.
- The `breakwaters` table has `UNIQUE (plot_id)` — there is always at most one row per plot, enforced at the DB level.
- The edge function cannot import from `src/systems/erosion.ts` (different runtime) — the breakwater check logic is duplicated in the Deno function. Keep the two in sync manually.
- `cancelExpiryAlert()` must be called before `scheduleExpiryAlert()` when a new breakwater is purchased to prevent two simultaneous timers.
- `scheduleExpiryAlert` is idempotent with respect to multiple calls: if no breakwater exists, it returns without side effects.

## Acceptance Criteria
- [ ] `checkBreakwater` returns `true` for a plot with `expires_at` 24 hours in the future.
- [ ] `checkBreakwater` returns `false` for a plot with no `breakwaters` row.
- [ ] `checkBreakwater` returns `false` for a plot with `expires_at` in the past.
- [ ] `getBreakwaterExpiry` returns the correct `Date` object matching the DB `expires_at`.
- [ ] `scheduleExpiryAlert` sets a `setTimeout` with a delay equal to `expiresAt - 3 days - now` when > 0.
- [ ] `scheduleExpiryAlert` calls `breakwaterExpiryAlert()` immediately when expiry is less than 3 days away.
- [ ] Calling `scheduleExpiryAlert` twice cancels the first timeout before setting the second.
- [ ] The modified erosion edge function logs the expiry date and skips the plot when a breakwater is active.
