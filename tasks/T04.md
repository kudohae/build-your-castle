# T04 — [Phase 1] — Camera & scroll system

## Overview
This task implements all camera state management and coordinate transforms for the game world. Touch scrolling and pinch-zoom are the primary input modalities on mobile, so smooth momentum-based scrolling and a proper world↔screen coordinate mapping are critical infrastructure that nearly every other input handler depends on.

## Scope
Implement `src/canvas/camera.ts` to manage `offsetX`, `offsetY`, and `scale`. Wire up `touchstart`, `touchmove`, and `touchend` event listeners on the canvas element obtained from the renderer. Single-finger drag updates `offsetX`/`offsetY` with clamping at map boundaries. Two-finger pinch updates `scale` clamped between `0.5` and `2.0`, adjusting offsets to zoom toward the pinch midpoint. On `touchend`, apply momentum: preserve the last-frame velocity and decay it exponentially over ~300ms using a `requestAnimationFrame` loop. Export read-only accessors `getOffset(): { offsetX: number; offsetY: number }` and `getScale(): number` for the renderer, plus coordinate helpers `worldToScreen(wx, wy)` and `screenToWorld(sx, sy)`.

## Output Files
- `src/canvas/camera.ts` — all camera state, touch event wiring, coordinate helpers, momentum system

## Depends On
- T03 — needs `getCanvas()` to attach touch listeners to the correct DOM element

let offsetX = 0;

let scale = 1.0;

let velY = 0;

### Coordinate helpers

`worldToScreen` applies scale then subtracts the camera offset; `screenToWorld` is the inverse. Both return integer-rounded pixel coordinates.

```typescript
export function worldToScreen(wx: number, wy: number): { x: number; y: number } {
  return {
    x: Math.round(wx * scale - offsetX),
    y: Math.round(wy * scale - offsetY),
  };
}

export function screenToWorld(sx: number, sy: number): { x: number; y: number } {
  return {
    x: (sx + offsetX) / scale,
    y: (sy + offsetY) / scale,
  };
}
```

### Map boundary clamping

```typescript
const TOTAL_MAP_WIDTH = 4000; // fallback width; update if map grows beyond this

function clampOffset(): void {
  const minX = 0;
  const maxX = Math.max(0, TOTAL_MAP_WIDTH * scale - window.innerWidth);
  offsetX = Math.max(minX, Math.min(maxX, offsetX));

  const minY = 0;
  const maxY = Math.max(0, MAP_HEIGHT * scale - window.innerHeight);
  offsetY = Math.max(minY, Math.min(maxY, offsetY));
}
```

### Touch event handlers

```typescript
function onTouchStart(e: TouchEvent): void {
  e.preventDefault();
  cancelAnimationFrame(momentumRafId);
  velX = 0;
  velY = 0;
  lastTouches = Array.from(e.touches);
}

function onTouchMove(e: TouchEvent): void {
  e.preventDefault();
  const touches = Array.from(e.touches);

  if (touches.length === 1 && lastTouches.length === 1) {
    // Single-finger pan
    const dx = touches[0].clientX - lastTouches[0].clientX;
    const dy = touches[0].clientY - lastTouches[0].clientY;
    velX = -dx; // record velocity for momentum (inverted: moving right reduces offsetX)
    velY = -dy;
    offsetX -= dx;
    offsetY -= dy;
    clampOffset();
  } else if (touches.length === 2 && lastTouches.length === 2) {
    // Two-finger pinch-zoom
    const prevDist = Math.hypot(
      lastTouches[1].clientX - lastTouches[0].clientX,
      lastTouches[1].clientY - lastTouches[0].clientY,
    );
    const newDist = Math.hypot(
      touches[1].clientX - touches[0].clientX,
      touches[1].clientY - touches[0].clientY,
    );
    if (prevDist > 0) {
      const ratio = newDist / prevDist;
      const newScale = Math.max(0.5, Math.min(2.0, scale * ratio));

      // Zoom toward midpoint
      const midX = (touches[0].clientX + touches[1].clientX) / 2;
      const midY = (touches[0].clientY + touches[1].clientY) / 2;
      offsetX = midX * newScale - midX * scale + offsetX * (newScale / scale);
      offsetY = midY * newScale - midY * scale + offsetY * (newScale / scale);
      scale = newScale;
      clampOffset();
    }
  }

  lastTouches = touches;
}

function onTouchEnd(e: TouchEvent): void {
  e.preventDefault();
  lastTouches = Array.from(e.touches);
  if (lastTouches.length === 0) {
    startMomentum();
  }
}
```

### Momentum system

Decays velocity by 90% each frame (~60fps), stopping when velocity drops below 0.5px/frame.

```typescript
function startMomentum(): void {
  const DECAY = 0.88;
  const STOP_THRESHOLD = 0.5;

  function step(): void {
    velX *= DECAY;
    velY *= DECAY;
    offsetX += velX;
    offsetY += velY;
    clampOffset();

    if (Math.abs(velX) > STOP_THRESHOLD || Math.abs(velY) > STOP_THRESHOLD) {
      momentumRafId = requestAnimationFrame(step);
    }
  }

  momentumRafId = requestAnimationFrame(step);
}
```

### Initialization

Call `initCamera()` once from the app entry point after the renderer canvas is mounted.

```typescript
export function initCamera(): void {
  const canvas = getCanvas();
  canvas.addEventListener('touchstart', onTouchStart, { passive: false });
  canvas.addEventListener('touchmove', onTouchMove, { passive: false });
  canvas.addEventListener('touchend', onTouchEnd, { passive: false });
}
```

## Key Constraints
- `{ passive: false }` is required on all touch listeners to allow `e.preventDefault()`, which stops the browser's native scroll/zoom from interfering.
- Pinch-zoom must clamp `scale` to `[0.5, 2.0]` — values outside this range produce unreadable block sizes.
- `offsetX` / `offsetY` must never go negative or exceed `(mapDimension * scale - viewportDimension)`, or the camera will show empty space outside the map.
- Momentum RAF must be cancelled on `touchstart` to prevent two momentum loops running simultaneously.
- `worldToScreen` and `screenToWorld` must be mathematically inverse — verified by the round-trip acceptance criterion.
- Do not import from `src/systems/` or `src/ui/` — camera is low-level infrastructure; higher-level systems depend on camera, not the other way around.
- `TOTAL_MAP_WIDTH = 4000` is a placeholder constant; if the map grows beyond this, clamping will break. T09 (plots) should update this dynamically in a later pass.

## Acceptance Criteria
- [ ] Dragging a single finger across the canvas scrolls the world smoothly with no jitter.
- [ ] Lifting the finger produces a momentum glide that decays to a stop within ~300ms.
- [ ] Pinching inward zooms out to a minimum scale of `0.5`; pinching outward zooms in to a maximum of `2.0`.
- [ ] Camera cannot scroll past the left or top edge of the map (offsetX and offsetY stay ≥ 0).
- [ ] `screenToWorld(worldToScreen(wx, wy).x, worldToScreen(wx, wy).y)` returns approximately `{ x: wx, y: wy }` (within 1px rounding).
- [ ] `initCamera()` throws no errors when called after `renderer.start()`.
- [ ] TypeScript compiles with zero errors.
