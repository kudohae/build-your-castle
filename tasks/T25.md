# T25 — [Phase 2] — Tide system

## Overview
The tide system drives one of the most dynamic environmental forces in the game: a sinusoidal water level that rises and falls on a 6-hour cycle, periodically submerging castle zone blocks. High tide converts exposed dry_sand blocks to wet_sand over time, changing their physics properties and raising the structural wet bonus. This system creates time pressure and strategic gameplay — players must plan their castle height around the tide cycle.

## Scope
Create `src/systems/weather.ts` as the single source of truth for all environmental time-series data. Implement `getTideLevel(now: Date): number` returning a value in `[0.0, 1.0]` where 0.0 is low tide and 1.0 is high tide, computed as `(Math.sin(phase * Math.PI * 2) + 1) / 2` with `phase = (now.getTime() % (6*3600*1000)) / (6*3600*1000)`. Implement `getWaterY(tideLevel: number): number` mapping `0.0 → GROUND_Y - 20` (low tide, water just below ground) and `1.0 → GROUND_Y - 120` (high tide, water 120px above ground), using linear interpolation. Implement `applyTideEffect(blocks: Block[], now: Date, updateFn: (block: Block) => Promise<void>): void` which is called once per animation frame: compute the current `waterY`, then for each block in the CASTLE_ZONE (`block.x >= CASTLE_ZONE_START_X`) where `block.y + BLOCK_SIZE > waterY` (block bottom is submerged), if `block.type === 'dry_sand'` apply a stochastic conversion: `if (Math.random() < 0.005)` set `block.type = 'wet_sand'`, `block.isWet = true`, and call `updateFn(block)` to sync to Supabase. The 0.005 probability per frame at 60fps gives ~26% conversion per second and ~50% over ~100 frames (~1.67 seconds).

## Output Files
- `src/systems/weather.ts` — new file; exports `getTideLevel`, `getWaterY`, `applyTideEffect`, `TideInfo`

## Implementation Details

### Tide math and constants

```typescript
// src/systems/weather.ts

import { Block, BLOCK_SIZE } from '../types/block';
import {
  CASTLE_ZONE_START_X,
  GROUND_Y,
} from '../types/zones';

const TIDE_PERIOD_MS = 6 * 60 * 60 * 1000; // 6 hours in milliseconds
const TIDE_LOW_WATER_Y_OFFSET = 20;   // GROUND_Y - 20 at low tide
const TIDE_HIGH_WATER_Y_OFFSET = 120; // GROUND_Y - 120 at high tide

export interface TideInfo {
  level: number;   // 0.0 (low) to 1.0 (high)
  waterY: number;  // canvas y-coordinate of water surface
  isRising: boolean;
  nextChangeMs: number; // ms until next high or low tide
}
```

### getTideLevel

```typescript
export function getTideLevel(now: Date): number {
  const elapsed = now.getTime() % TIDE_PERIOD_MS;
  const phase = elapsed / TIDE_PERIOD_MS; // 0.0 → 1.0 over 6 hours
  return (Math.sin(phase * Math.PI * 2) + 1) / 2;
}
```

At `phase=0`: `sin(0)=0` → level=0.5 (mid-tide rising).
At `phase=0.25`: `sin(π/2)=1` → level=1.0 (high tide).
At `phase=0.5`: `sin(π)=0` → level=0.5 (mid-tide falling).
At `phase=0.75`: `sin(3π/2)=-1` → level=0.0 (low tide).

### getWaterY

```typescript
export function getWaterY(tideLevel: number): number {
  // tideLevel 0.0 → GROUND_Y - 20 (low tide, water barely visible)
  // tideLevel 1.0 → GROUND_Y - 120 (high tide, 4 blocks submerged)
  const lowY = GROUND_Y - TIDE_LOW_WATER_Y_OFFSET;
  const highY = GROUND_Y - TIDE_HIGH_WATER_Y_OFFSET;
  return lowY + (highY - lowY) * tideLevel;
  // Equivalently: GROUND_Y - 20 - 100 * tideLevel
}
```

At low tide: `waterY = GROUND_Y - 20` (2/3 of a block above ground).
At high tide: `waterY = GROUND_Y - 120` (3.75 blocks above ground — ~3-4 block layers submerged).

### getTideInfo (convenience wrapper)

```typescript
export function getTideInfo(now: Date): TideInfo {
  const level = getTideLevel(now);
  const waterY = getWaterY(level);

  // Determine if tide is currently rising
  // Derivative of sin is cos; rising when cos(phase * 2π) > 0, i.e., phase in (0.5, 1.0) or (0, 0.5)
  const elapsed = now.getTime() % TIDE_PERIOD_MS;
  const phase = elapsed / TIDE_PERIOD_MS;
  const isRising = Math.cos(phase * Math.PI * 2) > 0;

  // Time to next extremum (high or low tide)
  // Next high tide: phase=0.25 → target elapsed = TIDE_PERIOD_MS * 0.25
  // Next low tide: phase=0.75 → target elapsed = TIDE_PERIOD_MS * 0.75
  const nextHighMs = ((0.25 - phase + 1) % 1) * TIDE_PERIOD_MS;
  const nextLowMs = ((0.75 - phase + 1) % 1) * TIDE_PERIOD_MS;
  const nextChangeMs = isRising ? nextHighMs : nextLowMs;

  return { level, waterY, isRising, nextChangeMs };
}
```

### applyTideEffect (per-frame stochastic conversion)

```typescript
const CONVERSION_PROBABILITY_PER_FRAME = 0.005;

export function applyTideEffect(
  blocks: Block[],
  now: Date,
  updateFn: (block: Block) => Promise<void>
): void {
  const tideLevel = getTideLevel(now);
  const waterY = getWaterY(tideLevel);

  for (const block of blocks) {
    // Only process CASTLE_ZONE blocks
    if (block.x < CASTLE_ZONE_START_X) continue;

    // Block must be submerged: bottom edge below waterY
    const blockBottom = block.y + BLOCK_SIZE;
    if (blockBottom <= waterY) continue;

    // Only convert dry sand
    if (block.type !== 'dry_sand') continue;

    // Already wet — skip
    if (block.isWet) continue;

    // Stochastic conversion
    if (Math.random() < CONVERSION_PROBABILITY_PER_FRAME) {
      block.type = 'wet_sand';
      block.isWet = true;
      // Fire-and-forget Supabase update (errors should be caught in updateFn)
      updateFn(block).catch((err) => {
        console.error('[Tide] Failed to update block:', block.id, err);
      });
    }
  }
}
```

### Supabase update function signature
The `updateFn` parameter decouples `weather.ts` from direct Supabase imports. The caller (renderer.ts or a game loop coordinator) passes in:

```typescript
import { updateBlock } from '../supabase/blocks';

// When calling applyTideEffect:
applyTideEffect(blocks, new Date(), async (block) => {
  await updateBlock(block.id, { type: block.type, isWet: block.isWet });
});
```

### Conversion probability analysis
- Per frame (60fps): 0.5% chance
- After 100 frames (1.67s): `1 - (1-0.005)^100 ≈ 39.5%` conversion probability
- After 200 frames (3.33s): `1 - (1-0.005)^200 ≈ 63.2%`
- After 460 frames (~7.7s): `≈ 90%` — effectively all blocks convert in under 10 seconds of submersion

This is intentional — blocks should wet quickly but not instantly, giving the player a moment of anticipation.

## Key Constraints
- `applyTideEffect` is called every animation frame; it must complete in < 1ms for 200 blocks.
- `Math.random()` calls inside `applyTideEffect` are intentional non-determinism — do not seed or mock them in production.
- Blocks that are already `isWet === true` are skipped even if submerged again (the conversion is idempotent).
- `updateFn` is async and fire-and-forget — the local block state is updated synchronously, Supabase sync follows.
- `getTideLevel` uses `Date.getTime()` (UTC milliseconds) — the 6-hour cycle is relative to Unix epoch, not local midnight.
- Do not import from `src/canvas/` or `src/ui/` — weather.ts is a pure data/logic module.

## Acceptance Criteria
- [ ] `getTideLevel(new Date(0))` returns 0.5 (at Unix epoch, phase=0, sin(0)=0, level=0.5).
- [ ] `getTideLevel` returns 1.0 at phase=0.25 (1.5 hours into the cycle) and 0.0 at phase=0.75 (4.5 hours in).
- [ ] `getWaterY(0.0)` returns `GROUND_Y - 20` and `getWaterY(1.0)` returns `GROUND_Y - 120`.
- [ ] `applyTideEffect` does not modify blocks in BEACH or SHORELINE zones.
- [ ] `applyTideEffect` does not modify blocks that are already `isWet === true`.
- [ ] A submerged `dry_sand` block converts to `wet_sand` within ~10 seconds of submersion on average (testable with a high-frequency call loop).
- [ ] `updateFn` is called at most once per block per conversion event (not once per frame).
- [ ] `getTideInfo` correctly identifies `isRising === true` during the first 3 hours and `false` during the latter 3 hours of the cycle.
