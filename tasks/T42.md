# T42 — [Phase 4] — Purchase Application Logic

## Overview
This task implements the `applyPurchase` switch statement that translates a completed AIT receipt into concrete in-game effects — upgrading tools, expanding plots, adding inventory items, and creating breakwater protection records. It also extends the inventory system with the new slot-expansion and item-addition capabilities required by several product types.

## Scope
Implement `applyPurchase(productId, receipt)` inside `src/ait/payment.ts` as a `switch` statement covering all 11 product cases. Cases and their effects: `shovel_lv2/3/5` → call `inventory.setShovelLevel(level)`; `bucket_extra` → call `inventory.addBucketSlot()`; `plot_expand` → call `expandPlot(userId)` from `src/supabase/plots.ts` which increments the plot's `width` by one CASTLE_ZONE block-width (800px / 25 blocks) and recalculates the right MAP boundary; `wood_plank_pack` → call `inventory.addItem('wood_plank', 50)`; `stone_pack` → call `inventory.addItem('stone', 30)`; `breakwater_7d` → insert a `breakwaters` row with `expires_at = NOW() + 7 days`; `breakwater_30d` → insert with `expires_at = NOW() + 30 days`; `inventory_expand` → call `inventory.expandMaxSlots(10)`. All side effects are persisted to Supabase. Modify `src/systems/inventory.ts` to add `setShovelLevel`, `addBucketSlot`, `addItem`, `expandMaxSlots` if not already present.

## Output Files
- `src/ait/payment.ts (modify)` — fills in the `applyPurchase` switch statement
- `src/systems/inventory.ts (modify)` — adds `setShovelLevel`, `addBucketSlot`, `addItem`, `expandMaxSlots`, persisted to Supabase

### Full applyPurchase Switch

```typescript
// src/ait/payment.ts — replace stub with this implementation

import { inventory } from '../systems/inventory';
import { expandPlot } from '../supabase/plots';
import { supabase } from '../supabase/client';
import { getCurrentUserId } from '../ait/auth';

export async function applyPurchase(
  productId: ProductId,
  _receipt: string   // kept for future server-side verification
): Promise<void> {
  const userId = getCurrentUserId();

  switch (productId) {
    // ── Shovel upgrades ────────────────────────────────────────────
    case PRODUCTS.shovel_lv2:
      await inventory.setShovelLevel(2);
      break;

    case PRODUCTS.shovel_lv3:
      await inventory.setShovelLevel(3);
      break;

    case PRODUCTS.shovel_lv5:
      await inventory.setShovelLevel(5);
      break;

    // ── Bucket slot ────────────────────────────────────────────────
    case PRODUCTS.bucket_extra:
      await inventory.addBucketSlot();
      break;

    // ── Plot expansion ─────────────────────────────────────────────
    case PRODUCTS.plot_expand:
      await expandPlot(userId);
      break;

    // ── Block packs ────────────────────────────────────────────────
    case PRODUCTS.wood_plank_pack:
      await inventory.addItem('wood_plank', 50);
      break;

    case PRODUCTS.stone_pack:
      await inventory.addItem('stone', 30);
      break;

    // ── Breakwater protection ──────────────────────────────────────
    case PRODUCTS.breakwater_7d:
      await createBreakwater(userId, 7);
      break;

    case PRODUCTS.breakwater_30d:
      await createBreakwater(userId, 30);
      break;

    // ── Inventory expansion ────────────────────────────────────────
    case PRODUCTS.inventory_expand:
      await inventory.expandMaxSlots(10);
      break;

    // ── Season pack ───────────────────────────────────────────────
    case PRODUCTS.season_pack:
      await applySeasonPack(userId);
      break;

    default:
      throw new Error(`알 수 없는 상품: ${productId}`);
  }
}
```

### createBreakwater Helper

```typescript
async function createBreakwater(userId: string, days: number): Promise<void> {
  const plotResult = await supabase
    .from('plots')
    .select('id')
    .eq('owner_id', userId)
    .single();

  if (!plotResult.data) throw new Error('성을 찾을 수 없습니다.');

  const expiresAt = new Date();
  expiresAt.setDate(expiresAt.getDate() + days);

  // Upsert: if a breakwater already exists, extend it
  const { error } = await supabase
    .from('breakwaters')
    .upsert(
      {
        user_id:    userId,
        plot_id:    plotResult.data.id,
        expires_at: expiresAt.toISOString(),
      },
      { onConflict: 'plot_id' }  // one breakwater per plot
    );

  if (error) throw new Error(`방파제 생성 실패: ${error.message}`);
}
```

`upsert` on `plot_id` conflict means purchasing a second breakwater during an active one replaces (extends) the expiry rather than creating a duplicate row.

### expandPlot in plots.ts

```typescript
// src/supabase/plots.ts (addition)

const PLOT_EXPAND_WIDTH_PX = 800; // one CASTLE_ZONE segment width

export async function expandPlot(userId: string): Promise<void> {
  const { data: plot, error: fetchErr } = await supabase
    .from('plots')
    .select('id, width')
    .eq('owner_id', userId)
    .single();

  if (fetchErr || !plot) throw new Error('성 구역을 찾을 수 없습니다.');

  const newWidth = plot.width + PLOT_EXPAND_WIDTH_PX;

  const { error: updateErr } = await supabase
    .from('plots')
    .update({ width: newWidth })
    .eq('id', plot.id);

  if (updateErr) throw new Error(`구역 확장 실패: ${updateErr.message}`);

  // Notify the local MAP system to recalculate right boundary
  window.dispatchEvent(
    new CustomEvent('plot:expanded', { detail: { plotId: plot.id, newWidth } })
  );
}
```

The `CustomEvent` allows the client-side MAP/camera system to react without a direct import cycle between `supabase/plots.ts` and the canvas layer.

### Inventory Additions in inventory.ts

```typescript
// src/systems/inventory.ts (additions)

import { supabase } from '../supabase/client';
import { getCurrentUserId } from '../ait/auth';

export class Inventory {
  private shovelLevel: number = 1;
  private bucketSlots: number = 1;    // default 1 bucket slot
  private maxSlots: number = 20;      // default max inventory slots
  private items: Map<string, number> = new Map();

  // ── Shovel ────────────────────────────────────────────────────────
  async setShovelLevel(level: 2 | 3 | 5): Promise<void> {
    if (level <= this.shovelLevel) return; // never downgrade
    this.shovelLevel = level;
    await this.persistShovelLevel();
  }

  getShovelLevel(): number { return this.shovelLevel; }

  private async persistShovelLevel(): Promise<void> {
    const userId = getCurrentUserId();
    await supabase
      .from('user_settings')
      .upsert({ user_id: userId, shovel_level: this.shovelLevel }, { onConflict: 'user_id' });
  }

  // ── Bucket slots ──────────────────────────────────────────────────
  async addBucketSlot(): Promise<void> {
    const MAX_BUCKET_SLOTS = 3;
    if (this.bucketSlots >= MAX_BUCKET_SLOTS) return;
    this.bucketSlots++;
    await this.persistUserSettings();
  }

  getBucketSlots(): number { return this.bucketSlots; }

  // ── Items ─────────────────────────────────────────────────────────
  async addItem(itemType: string, quantity: number): Promise<void> {
    const current = this.items.get(itemType) ?? 0;
    this.items.set(itemType, current + quantity);
    await this.persistItem(itemType, current + quantity);
  }

  getItemCount(itemType: string): number {
    return this.items.get(itemType) ?? 0;
  }

  private async persistItem(itemType: string, quantity: number): Promise<void> {
    const userId = getCurrentUserId();
    await supabase
      .from('inventory_items')
      .upsert(
        { user_id: userId, item_type: itemType, quantity },
        { onConflict: 'user_id,item_type' }
      );
  }

  // ── Max slots ─────────────────────────────────────────────────────
  async expandMaxSlots(by: number): Promise<void> {
    this.maxSlots += by;
    await this.persistUserSettings();
  }

  getMaxSlots(): number { return this.maxSlots; }

  // ── Generic persist ───────────────────────────────────────────────
  private async persistUserSettings(): Promise<void> {
    const userId = getCurrentUserId();
    await supabase.from('user_settings').upsert(
      {
        user_id:       userId,
        shovel_level:  this.shovelLevel,
        bucket_slots:  this.bucketSlots,
        max_inv_slots: this.maxSlots,
      },
      { onConflict: 'user_id' }
    );
  }

  // ── Hydration from DB ─────────────────────────────────────────────
  async hydrate(): Promise<void> {
    const userId = getCurrentUserId();

    const [settingsRes, itemsRes] = await Promise.all([
      supabase
        .from('user_settings')
        .select('shovel_level, bucket_slots, max_inv_slots')
        .eq('user_id', userId)
        .maybeSingle(),
      supabase
        .from('inventory_items')
        .select('item_type, quantity')
        .eq('user_id', userId),
    ]);

    if (settingsRes.data) {
      this.shovelLevel = settingsRes.data.shovel_level ?? 1;
      this.bucketSlots = settingsRes.data.bucket_slots ?? 1;
      this.maxSlots    = settingsRes.data.max_inv_slots ?? 20;
    }

    for (const row of itemsRes.data ?? []) {
      this.items.set(row.item_type, row.quantity);
    }
  }
}

export const inventory = new Inventory();
```

## Key Constraints
- `setShovelLevel` never downgrades: calling `setShovelLevel(2)` when level is already 3 is a no-op.
- `addBucketSlot` is capped at 3 total slots (original + 2 extras); attempting a 4th slot silently returns.
- `expandPlot` dispatches a `CustomEvent` rather than importing the canvas layer to avoid circular dependencies.
- Breakwater `upsert` uses `onConflict: 'plot_id'` — buying a second breakwater extends (overwrites) the active one rather than creating a second row. The edge function (T48) must handle this single-row design.
- The `season_pack` case uses a dynamic `import()` to defer the T50 dependency until it is available; do not convert this to a static import.
- All inventory mutations call `persistUserSettings` or `persistItem` immediately — there is no deferred/batched write in this task.
- `applyPurchase` is always called by `purchase()` in payment.ts after the receipt is stored; it must never be called directly from the UI.
