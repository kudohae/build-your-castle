# T27 — [Phase 2] — Wind system

## Overview
The wind system adds a persistent environmental force that challenges structural stability by pushing unsettled blocks. Wind direction and strength change every 5 minutes using a deterministic seeded pseudo-random approach, ensuring all players on the same map experience the same wind conditions simultaneously. Wind is categorized into four Korean strength labels and physically applied to Matter.js bodies each animation frame, making tall or unsupported structures visibly sway and potentially topple during strong winds.

## Scope
Add `getWind(now: Date): Wind` to `src/systems/weather.ts`. Compute a seed from the current 5-minute window: `seed = Math.floor(now.getTime() / 300_000)`. Use `seededRandom(seed)` to get a deterministic direction (0 to 2π radians) and `seededRandom(seed + 1)` to get a strength in `[0, 1)`. Map strength to the label: `'무풍'` (< 0.1), `'약풍'` (0.1–0.39), `'중풍'` (0.4–0.69), `'강풍'` (≥ 0.7). Add `applyWindForce(body: Matter.Body, wind: Wind): void` which calls `Matter.Body.applyForce` with `{ x: Math.cos(wind.direction) * wind.strength * body.mass * 0.002, y: Math.sin(wind.direction) * wind.strength * body.mass * 0.002 }`. The force formula scales by body mass so heavier blocks resist wind proportionally. The game loop calls `applyWindForce` each frame for every block body that is not static and not `isSettled` (blocks flagged as settled after coming to rest should be immune to small wind forces). The `seededRandom` implementation uses a simple mulberry32 PRNG that produces deterministic output from a 32-bit seed.

## Implementation Details

### seededRandom — mulberry32 PRNG

A fast, seedable PRNG that produces the same output for any given seed, ensuring all clients share the same wind without network coordination:

```typescript
// src/systems/weather.ts (addition)

function mulberry32(seed: number): () => number {
  return function () {
    seed |= 0;
    seed = (seed + 0x6d2b79f5) | 0;
    let z = Math.imul(seed ^ (seed >>> 15), 1 | seed);
    z = (z + Math.imul(z ^ (z >>> 7), 61 | z)) ^ z;
    return ((z ^ (z >>> 14)) >>> 0) / 0x100000000;
  };
}

/**
 * Returns a deterministic pseudo-random number in [0, 1) for the given integer seed.
 */
export function seededRandom(seed: number): number {
  return mulberry32(seed)();
}
```

### Wind interface and getWind

```typescript
export type WindLabel = '무풍' | '약풍' | '중풍' | '강풍';

export interface Wind {
  direction: number;   // radians, 0 = East, π/2 = South
  strength: number;    // 0.0 to 1.0
  label: WindLabel;
  seed: number;        // the 5-min window seed, for debugging
}

const WIND_WINDOW_MS = 5 * 60 * 1000; // 5 minutes

export function getWind(now: Date): Wind {
  const seed = Math.floor(now.getTime() / WIND_WINDOW_MS);
  const direction = seededRandom(seed) * Math.PI * 2;
  const strength = seededRandom(seed + 1);

  let label: WindLabel;
  if (strength < 0.1) {
    label = '무풍';
  } else if (strength < 0.4) {
    label = '약풍';
  } else if (strength < 0.7) {
    label = '중풍';
  } else {
    label = '강풍';
  }

  return { direction, strength, label, seed };
}
```

### applyWindForce

The force constant `0.002` is calibrated so that at `strength=1.0` (maximum) and `mass=80` (dry sand), the force is `0.16` — enough to move an unsupported block but not overwhelm a properly built castle. Wet sand at `mass=120` resists wind by 50% more (heavier), incentivising wetting blocks.

```typescript
import Matter from 'matter-js';

const WIND_FORCE_SCALE = 0.002;

export function applyWindForce(body: Matter.Body, wind: Wind): void {
  if (body.isStatic) return;

  const force = {
    x: Math.cos(wind.direction) * wind.strength * body.mass * WIND_FORCE_SCALE,
    y: Math.sin(wind.direction) * wind.strength * body.mass * WIND_FORCE_SCALE,
  };

  Matter.Body.applyForce(body, body.position, force);
}
```

### Integration in the game loop

The wind is computed once per frame (not per body — `getWind` is cheap since it's deterministic):

```typescript
// In the physics update step (called before Matter.Engine.update):

function applyWindToAllBodies(blocks: Block[], wind: Wind): void {
  for (const block of blocks) {
    if (!block.body) continue;
    if (block.isSettled) continue;   // settled blocks ignore small forces
    applyWindForce(block.body, wind);
  }
}

// Per frame:
const wind = getWind(new Date());
applyWindToAllBodies(activeBlocks, wind);
Matter.Engine.update(engine, 1000 / 60);
```

### isSettled flag definition
A block is considered settled when its velocity magnitude is below a threshold for several consecutive frames. The physics engine (T08) should set `block.isSettled = true` once `Math.sqrt(vx² + vy²) < 0.1` for 30 consecutive frames. Wind force bypasses settled blocks to avoid constant micro-vibration in a stable castle, which would both look wrong and waste CPU.

```typescript
// Expected block field (ensure in T08's Block type):
interface Block {
  // ... existing fields
  isSettled: boolean; // true = body has come to rest; immune to wind
}
```

### Wind transition (between 5-min windows)
Because `seed` changes abruptly every 5 minutes, the wind direction/strength can snap instantaneously. To soften this, interpolate between old and new wind over a 10-second crossfade:

```typescript
let previousWind: Wind | null = null;
let windTransitionStart = 0;
const WIND_TRANSITION_MS = 10_000;

export function getInterpolatedWind(now: Date): Wind {
  const currentWind = getWind(now);

  if (!previousWind || previousWind.seed !== currentWind.seed) {
    previousWind = currentWind;
    windTransitionStart = now.getTime();
    return currentWind;
  }

  const t = Math.min(1, (now.getTime() - windTransitionStart) / WIND_TRANSITION_MS);
  // Interpolate strength; direction snaps (circular interpolation is complex)
  return {
    ...currentWind,
    strength: previousWind.strength + (currentWind.strength - previousWind.strength) * t,
  };
}
```

## Key Constraints
- `seededRandom(seed)` must return identical values across all clients for the same seed — no `Math.random()` involvement.
- `getWind` may be called hundreds of times per second (once per frame); it must be fast — no loops, just two PRNG evaluations.
- `applyWindForce` must not be called for `isStatic === true` bodies (ground, walls) — static bodies in Matter.js ignore applied forces but the guard still prevents wasted cycles.
- `'강풍'` (strength ≥ 0.7) should be rare — with a uniform distribution, it occurs ~30% of 5-minute windows. This is intentional challenge.
- Wind direction 0 = East (positive x). Direction π/2 = South (positive y = downward in canvas). Southward wind pushes blocks down, which is interesting but physically unusual — designers may want to clamp the y component.
- `WIND_FORCE_SCALE = 0.002` is a tuning constant; document it clearly and make it easy to adjust during playtesting.
