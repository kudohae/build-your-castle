# T21 — [Phase 2] — Wet sand conversion (shore dipping)

## Overview
This task implements the mechanic by which dry sand becomes wet sand when the player drags it into the water area of the SHORELINE zone. This is a purely inventory-level transformation — no new physics body is created and no DB block row is written. Instead the selected inventory slot's `blockType` changes from `'dry_sand'` to `'wet_sand'`, accompanied by a splash particle animation to make the interaction feel satisfying. The wet sand can then be placed as a stickier, heavier block in the CASTLE_ZONE (T22 continues the bucket mechanic for liquid water).

## Scope
Modify `src/systems/inventory.ts` to export `dipSandInWater(inventory, slotIndex): boolean`. The function checks that the targeted slot contains a block of type `'dry_sand'`; if so, mutates that slot's `blockType` to `'wet_sand'` and returns `true`. It does not change quantity or any other slot property. Modify `src/canvas/renderer.ts` to listen for drag-move events when the selected item is a `'dry_sand'` stack: if the drag position is within the SHORELINE zone (x in `[800, 1200)`) **and** the canvas-y coordinate is greater than the current `waterY` (obtained from `getWaterY(getTideLevel(new Date()))` from T25's weather module — if T25 is not yet merged, use a stub returning `GROUND_Y - 50`), call `dipSandInWater` and fire the splash animation at the drop point. Update the inventory display in the next frame to show `'wet_sand'` color for the converted slot. The conversion is one-way and immediate: once converted, the slot shows wet_sand for the rest of the drag.

## Output Files
- `src/systems/inventory.ts (modify)` — adds `dipSandInWater(inventory, slotIndex)` function
- `src/canvas/renderer.ts (modify)` — detects water-zone drag, triggers `dipSandInWater`, spawns splash particles

## Implementation Details

### dipSandInWater function

```typescript
// src/systems/inventory.ts (addition)

import { Inventory, InventorySlot, BlockType } from '../types/inventory';

/**
 * Converts the dry_sand in the given inventory slot to wet_sand.
 * Returns true if conversion happened, false if the slot was not dry_sand.
 */
export function dipSandInWater(inventory: Inventory, slotIndex: number): boolean {
  const slot: InventorySlot | undefined = inventory.slots[slotIndex];
  if (!slot || slot.blockType !== 'dry_sand' || slot.quantity <= 0) {
    return false;
  }
  slot.blockType = 'wet_sand';
  return true;
}
```

### Splash particle system
A lightweight particle emitter for the splash effect. Particles are stored in a module-level array so the existing `renderFrame` can iterate and update them without needing a separate animation loop.

```typescript
// src/canvas/renderer.ts (additions)

interface SplashParticle {
  x: number;
  y: number;
  vx: number;
  vy: number;
  alpha: number;
  life: number;       // frames remaining
  maxLife: number;
}

const splashParticles: SplashParticle[] = [];

function emitSplash(canvasX: number, canvasY: number, count = 12): void {
  for (let i = 0; i < count; i++) {
    const angle = (Math.PI * 2 * i) / count + (Math.random() - 0.5) * 0.4;
    const speed = 0.8 + Math.random() * 2.0;
    splashParticles.push({
      x: canvasX,
      y: canvasY,
      vx: Math.cos(angle) * speed,
      vy: Math.sin(angle) * speed - 2.5, // upward bias
      alpha: 1.0,
      life: 30 + Math.floor(Math.random() * 20),
      maxLife: 50,
    });
  }
}

function updateAndRenderSplash(ctx: CanvasRenderingContext2D): void {
  for (let i = splashParticles.length - 1; i >= 0; i--) {
    const p = splashParticles[i];
    p.x += p.vx;
    p.y += p.vy;
    p.vy += 0.15; // gravity
    p.life--;
    p.alpha = p.life / p.maxLife;

    if (p.life <= 0) {
      splashParticles.splice(i, 1);
      continue;
    }

    ctx.save();
    ctx.globalAlpha = p.alpha;
    ctx.fillStyle = '#64b4ff'; // light blue water
    ctx.beginPath();
    ctx.arc(p.x, p.y, 2, 0, Math.PI * 2);
    ctx.fill();
    ctx.restore();
  }
}
```

### Drag detection in renderer

The renderer's `pointermove` handler must be extended (or a new handler added) for the sand-dipping mechanic. This is separate from the carving handler in T20 and should be gated by a different selected-item check:

```typescript
// src/canvas/renderer.ts — extend pointermove handler

import { dipSandInWater } from '../systems/inventory';
import { getTideLevel, getWaterY } from '../systems/weather';
import { SHORELINE_START_X, SHORELINE_END_X, GROUND_Y } from '../types/zones';

function handleSandDipMove(
  e: PointerEvent,
  canvas: HTMLCanvasElement,
  inventory: Inventory,
  selectedSlotIndex: number
): void {
  const slot = inventory.slots[selectedSlotIndex];
  if (!slot || slot.blockType !== 'dry_sand') return;

  const rect = canvas.getBoundingClientRect();
  const canvasX = (e.clientX - rect.left) * (canvas.width / rect.width);
  const canvasY = (e.clientY - rect.top) * (canvas.height / rect.height);

  // Must be in SHORELINE x-range
  if (canvasX < SHORELINE_START_X || canvasX >= SHORELINE_END_X) return;

  // Must be below current waterY
  const tideLevel = getTideLevel(new Date());
  const waterY = getWaterY(tideLevel);
  if (canvasY <= waterY) return;

  const converted = dipSandInWater(inventory, selectedSlotIndex);
  if (converted) {
    emitSplash(canvasX, canvasY);
    // Inventory HUD will pick up the blockType change in next render
  }
}
```

## Key Constraints
- `dipSandInWater` may only be called once per drag session per slot — once the slot is `wet_sand`, repeat calls are no-ops (guarded by the `blockType !== 'dry_sand'` check).
- Conversion only happens in SHORELINE zone (x: 800–1199), not in BEACH or CASTLE_ZONE.
- The canvas-y comparison uses the **live** tide level, not a hardcoded y-value; during low tide the water is higher on screen (`waterY` is a smaller number) so less of the shoreline triggers conversion.
- No Supabase call is made during this transformation — the inventory lives in local state only.
- The splash particle emitter must not cause frame rate drops; limit `splashParticles` array size to 200 at any moment (drop oldest when exceeded).
- Block color for the inventory slot must update immediately (same frame as conversion).

## Acceptance Criteria
- [ ] Dragging a `dry_sand` slot into the water portion of SHORELINE converts it to `wet_sand` and the HUD slot color updates in the same frame.
- [ ] The conversion does not occur if the drag is above the waterline (canvasY <= waterY).
- [ ] The conversion does not occur in BEACH or CASTLE_ZONE x-ranges.
- [ ] A splash particle animation (≥ 8 particles) emits at the drop point and fades out within ~50 frames.
- [ ] `dipSandInWater` returns `false` and makes no change if the slot is not `dry_sand`.
- [ ] Inventory slots other than the selected slot are unaffected.
- [ ] After conversion, subsequent drag frames over water do not trigger additional conversion calls (slot is already `wet_sand`).
- [ ] The splash particle array is bounded (no memory leak during repeated dipping).
