# T29 — [Phase 3] — Supabase Realtime subscription manager

## Overview
This task creates the realtime collaboration backbone of the game, enabling players to see each other's block placements, deletions, and modifications as they happen. The subscription manager wraps Supabase's Realtime channels behind a clean interface with LRU eviction to stay within Supabase's simultaneous channel limits. This is Phase 3's foundational module — all multiplayer viewing features build on top of it.

## Scope
Create `src/supabase/realtime.ts` with `subscribePlot(plotId, callbacks)` and `unsubscribePlot(plotId)`. `subscribePlot` opens (or re-uses if already open) a Supabase Realtime channel named `'plot:{plotId}'` and subscribes to `postgres_changes` events on the `blocks` table filtered by `plot_id = plotId`. The callbacks object provides three handlers: `onInsert(block: Block)`, `onUpdate(block: Block)`, `onDelete(id: string)`. Track active channels in a `Map<string, ActiveChannel>` where `ActiveChannel` stores the Supabase `RealtimeChannel` reference and an LRU access timestamp. Enforce a maximum of 5 simultaneous subscriptions: before opening a new channel when at the limit, find the oldest accessed channel (minimum timestamp), call `unsubscribePlot` on that plotId, then proceed. On each `subscribePlot` call, update the LRU timestamp for that plotId. `unsubscribePlot` calls `supabase.removeChannel(channel)` and deletes the map entry.

## Output Files
- `src/supabase/realtime.ts` — new file; exports `subscribePlot`, `unsubscribePlot`, `PlotCallbacks`, `ActiveChannel`

## Depends On

## Implementation Details

### Types and constants

```typescript
// src/supabase/realtime.ts

import { supabase } from './client';
import { Block } from '../types/block';
import { parseBlockRow } from './blocks';
import type { RealtimeChannel } from '@supabase/supabase-js';

export interface PlotCallbacks {
  onInsert: (block: Block) => void;
  onUpdate: (block: Block) => void;
  onDelete: (id: string) => void;
}

interface ActiveChannel {
  plotId: string;
  channel: RealtimeChannel;
  callbacks: PlotCallbacks;
  lastAccessed: number; // Date.now() timestamp for LRU
}

const MAX_SUBSCRIPTIONS = 5;
const activeChannels = new Map<string, ActiveChannel>();
```

### LRU eviction helper

```typescript
function evictLRU(): void {
  if (activeChannels.size < MAX_SUBSCRIPTIONS) return;

  let oldestKey = '';
  let oldestTime = Infinity;

  for (const [plotId, entry] of activeChannels.entries()) {
    if (entry.lastAccessed < oldestTime) {
      oldestTime = entry.lastAccessed;
      oldestKey = plotId;
    }
  }

  if (oldestKey) {
    console.log(`[Realtime] Evicting LRU subscription for plot: ${oldestKey}`);
    unsubscribePlot(oldestKey);
  }
}
```

### subscribePlot

```typescript
export function subscribePlot(
  plotId: string,
  callbacks: PlotCallbacks
): void {
  // If already subscribed, update callbacks and touch LRU timestamp
  const existing = activeChannels.get(plotId);
  if (existing) {
    existing.callbacks = callbacks;
    existing.lastAccessed = Date.now();
    return;
  }

  // Enforce max subscription limit via LRU eviction
  if (activeChannels.size >= MAX_SUBSCRIPTIONS) {
    evictLRU();
  }

  const channelName = `plot:${plotId}`;

  const channel = supabase
    .channel(channelName)
    .on(
      'postgres_changes',
      {
        event: 'INSERT',
        schema: 'public',
        table: 'blocks',
        filter: `plot_id=eq.${plotId}`,
      },
      (payload) => {
        try {
          const block = parseBlockRow(payload.new);
          activeChannels.get(plotId)?.callbacks.onInsert(block);
        } catch (err) {
          console.error('[Realtime] INSERT parse error:', err, payload);
        }
      }
    )
    .on(
      'postgres_changes',
      {
        event: 'UPDATE',
        schema: 'public',
        table: 'blocks',
        filter: `plot_id=eq.${plotId}`,
      },
      (payload) => {
        try {
          const block = parseBlockRow(payload.new);
          activeChannels.get(plotId)?.callbacks.onUpdate(block);
        } catch (err) {
          console.error('[Realtime] UPDATE parse error:', err, payload);
        }
      }
    )
    .on(
      'postgres_changes',
      {
        event: 'DELETE',
        schema: 'public',
        table: 'blocks',
        filter: `plot_id=eq.${plotId}`,
      },
      (payload) => {
        const id = payload.old?.id as string | undefined;
        if (id) {
          activeChannels.get(plotId)?.callbacks.onDelete(id);
        }
      }
    )
    .subscribe((status) => {
      if (status === 'SUBSCRIBED') {
        console.log(`[Realtime] Subscribed to plot: ${plotId}`);
      } else if (status === 'CHANNEL_ERROR') {
        console.error(`[Realtime] Channel error for plot: ${plotId}`);
      } else if (status === 'TIMED_OUT') {
        console.warn(`[Realtime] Subscription timed out for plot: ${plotId}`);
      }
    });

  activeChannels.set(plotId, {
    plotId,
    channel,
    callbacks,
    lastAccessed: Date.now(),
  });
}
```

### unsubscribePlot

```typescript
export function unsubscribePlot(plotId: string): void {
  const entry = activeChannels.get(plotId);
  if (!entry) return;

  supabase
    .removeChannel(entry.channel)
    .then(() => {
      console.log(`[Realtime] Unsubscribed from plot: ${plotId}`);
    })
    .catch((err) => {
      console.error(`[Realtime] Error unsubscribing from plot: ${plotId}`, err);
    });

  activeChannels.delete(plotId);
}
```

### getActiveSubscriptionCount (utility for debugging)

```typescript
export function getActiveSubscriptionCount(): number {
  return activeChannels.size;
}

export function isSubscribed(plotId: string): boolean {
  return activeChannels.has(plotId);
}
```

### Error resilience and reconnection
Supabase Realtime channels reconnect automatically on network drops. However, if the subscription status becomes `'CHANNEL_ERROR'`, the channel should be removed and re-subscribed after a delay to avoid zombie channels:

```typescript
// Inside the .subscribe() callback:
if (status === 'CHANNEL_ERROR') {
  console.error(`[Realtime] Re-subscribing to plot: ${plotId} after error`);
  activeChannels.delete(plotId);
  setTimeout(() => subscribePlot(plotId, callbacks), 2000);
}
```

### Supabase Realtime filter note
The `filter: 'plot_id=eq.{plotId}'` syntax enables server-side row filtering so only changes to the target plot are received. This requires the `blocks` table to have Realtime enabled and the `plot_id` column indexed. The Supabase project must have Realtime enabled for the `blocks` table (confirm in Supabase dashboard → Table Editor → Realtime toggle, or via `ALTER TABLE blocks REPLICA IDENTITY FULL`).

## Key Constraints
- Maximum 5 simultaneous subscriptions at any time; the LRU channel is evicted synchronously before the new one is opened.
- `subscribePlot` called for an already-subscribed plotId must update the callbacks and refresh the LRU timestamp — not open a second channel for the same plot.
- `parseBlockRow` may throw if the Supabase payload is malformed; all event callbacks must be wrapped in try/catch.
- `unsubscribePlot` is safe to call for a plotId that has no active subscription (no-op).
- The `postgres_changes` filter reduces Realtime traffic — do not subscribe without the `filter` clause to avoid receiving all block events across all plots.
- Do not store block state inside this module — it is a transport layer only. State management is the caller's responsibility.

## Acceptance Criteria
- [ ] `subscribePlot('plot-1', cbs)` opens a Supabase channel named `'plot:plot-1'` and the `onInsert` callback fires when a row is inserted into `blocks` with `plot_id='plot-1'`.
- [ ] `unsubscribePlot('plot-1')` removes the channel and deletes the map entry; subsequent inserts to that plot do not trigger callbacks.
- [ ] When 5 subscriptions are active and `subscribePlot` is called for a 6th plotId, the oldest accessed channel is evicted first.
- [ ] Calling `subscribePlot` for an already-subscribed plotId updates callbacks without opening a duplicate channel (`getActiveSubscriptionCount()` stays unchanged).
- [ ] `onDelete` fires with the correct block `id` string when a row is deleted.
- [ ] A malformed payload in `onInsert` (that causes `parseBlockRow` to throw) does not crash the subscription and subsequent events still fire.
- [ ] `isSubscribed('non-existent')` returns `false`; `isSubscribed('plot-1')` returns `true` after subscribing.
- [ ] `CHANNEL_ERROR` status triggers automatic re-subscription after a 2-second delay.
