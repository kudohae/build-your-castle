# T12 — [Phase 1] — Shovel tool (beach digging)

## Overview
The shovel is the player's first and most fundamental interaction: tapping the beach to collect dry sand, which is then carried to the castle zone for building. This task wires up the touch/click input handler for shovel digging, gates it to the BEACH zone, and plays a short visual feedback animation at the dig point. It also extends the inventory singleton with the `shovelLevel` upgrade field needed by the payment system later.

## Scope
Modify `src/canvas/renderer.ts` to register a `touchstart`/`click` event handler on the canvas. When the selected inventory item is `'shovel'` and the tap world-coordinate falls within the BEACH zone (`x: 0–800`), call `inventory.addItem('dry_sand', inventory.shovelLevel)` to add sand to the inventory. Trigger a digging animation: render a shovel icon placeholder (colored circle for now) at the tap position that scales up and fades out over 400ms. Modify `src/systems/inventory.ts` to add `shovelLevel: 1 | 2 | 3 | 5` to the singleton (with default value `1`) and export `setShovelLevel(level)`.

## Output Files
- `src/canvas/renderer.ts (modify)` — adds canvas input handler and shovel animation system
- `src/systems/inventory.ts (modify)` — adds `shovelLevel` field and `setShovelLevel()` function

## Implementation Details

### Animation system in renderer.ts

Animations are stored as a module-level array and drawn each frame in the UI layer (screen-space). Each animation tracks its world position, age, and duration.

```typescript
// In renderer.ts — add to module-level declarations
interface ShovelAnimation {
  worldX: number;
  worldY: number;
  startedAt: number; // performance.now() timestamp
  duration: number;  // ms
}

const activeAnimations: ShovelAnimation[] = [];

/** Called by the render loop — draws and expires animations. */
function drawAnimations(ctx: CanvasRenderingContext2D, now: number): void {
  const { offsetX, offsetY } = getOffset();
  const scale = getScale();

  for (let i = activeAnimations.length - 1; i >= 0; i--) {
    const anim = activeAnimations[i];
    const elapsed = now - anim.startedAt;
    const t = Math.min(1, elapsed / anim.duration); // 0→1

    if (t >= 1) {
      activeAnimations.splice(i, 1);
      continue;
    }

    // Scale up 1→2, fade out 1→0
    const animScale = 1 + t;
    const alpha = 1 - t;

    // Convert world coords to screen coords
    const sx = anim.worldX * scale - offsetX;
    const sy = anim.worldY * scale - offsetY;

    ctx.save();
    ctx.globalAlpha = alpha;
    ctx.fillStyle = '#8B6914'; // shovel brown
    ctx.beginPath();
    ctx.arc(sx, sy, 12 * animScale, 0, Math.PI * 2);
    ctx.fill();
    ctx.restore();
  }
}
```

### Input handler in renderer.ts

The handler is registered once in `start()` or `initInputHandlers()`. It checks zone membership before adding inventory items.

```typescript
// In renderer.ts — add after canvas creation
import { inventory, addItem, getSelectedItem } from '../systems/inventory';
import { getZoneAtX } from '../constants/maps';
import { screenToWorld } from './camera';

function initInputHandlers(): void {
  const canvas = getCanvas();

  function handleDig(clientX: number, clientY: number): void {
    const { x: worldX, y: worldY } = screenToWorld(clientX, clientY);
    const zone = getZoneAtX(worldX);

    if (zone.id !== 'BEACH') return; // shovel only works on the beach
    const selected = getSelectedItem();
    if (selected?.itemType !== 'shovel') return;

    // Add sand to inventory
    addItem('dry_sand', inventory.shovelLevel);

    // Spawn animation
    activeAnimations.push({
      worldX,
      worldY,
      startedAt: performance.now(),
      duration: 400,
    });
  }

  canvas.addEventListener('click', (e: MouseEvent) => {
    handleDig(e.clientX, e.clientY);
  });

  canvas.addEventListener('touchstart', (e: TouchEvent) => {
    e.preventDefault();
    const touch = e.changedTouches[0];
    // Only handle single-finger taps that are not camera drags
    // (drag detection handled by camera.ts; short taps fall through here)
    handleDig(touch.clientX, touch.clientY);
  }, { passive: false });
}
```

> Note: The `touchstart` handler here will fire on every touch, including drags. A subsequent task should add a tap-vs-drag discriminator (e.g. only fire `handleDig` if the touch duration was < 200ms and the finger moved < 10px). For Phase 1 this simplified version is acceptable.

## Key Constraints
- The shovel interaction must be blocked outside the BEACH zone — the dig handler must check `zone.id === 'BEACH'` before calling `addItem`.
- `touchstart` with `{ passive: false }` must call `e.preventDefault()` to prevent browser-native scroll from firing simultaneously with the dig action.
- Animations are drawn in screen-space (after `ctx.restore()`) to avoid applying camera transforms to the animation scale effect.
- `activeAnimations` array must be pruned each frame — use reverse-iteration splice to avoid index shifting bugs.
- `setShovelLevel` is the only way to change shovel level; it will be called by the in-app purchase callback in T21.
- The dig animation is intentionally simple (colored circle) in Phase 1; a proper shovel sprite is a design asset delivered later.
- Do not call `addItem` if the inventory is full — `addItem` already returns `false` in that case, but the animation should still play (the player successfully dug, even if they couldn't carry it all).
