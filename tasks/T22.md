# T22 — [Phase 2] — Bucket system

## Overview
The bucket system gives players a way to collect liquid water from the SHORELINE zone and pour it onto placed blocks in the CASTLE_ZONE, making them wet. Wet blocks are heavier and stickier — wetting a sand castle improves structural cohesion and is rewarded by the stability calculator. This task creates the bucket module, including physics body updates via Matter.js when a block becomes wet, plus a brief shimmer visual effect to confirm the action.

## Scope
Create `src/systems/bucket.ts` with two exported functions. `fillBucket(inventory: Inventory): FillResult` verifies that the player's current zone is SHORELINE (determined by passing in the player's world x-position), searches for the first inventory slot with `itemType === 'bucket'` and `water === false`, sets that slot's `water` to `true`, and returns `{ success: true, slotIndex }` or `{ success: false, reason }`. `pourWater(block: Block, body: Matter.Body, inventory: Inventory): boolean` finds the first filled bucket slot, sets `block.isWet = true`, updates the Matter.js physics body's mass and friction to match `wet_sand` physics constants (using `Matter.Body.set`), empties that bucket slot (`water = false`), starts a shimmer animation on the block (stores shimmerUntil timestamp), and returns `true`. If no filled bucket is found, returns `false`. Also define `BucketSlot` extension of `InventorySlot` and a `FillResult` type. The shimmer effect is a separate `updateBlockShimmer(block, ctx)` function that draws a moving highlight stripe across the block for ~60 frames after wetting.

## Output Files
- `src/systems/bucket.ts` — new file; exports `fillBucket`, `pourWater`, `updateBlockShimmer`, `BucketSlot`, `FillResult`

## Depends On

## Implementation Details

### Types

```typescript
// src/systems/bucket.ts

import Matter from 'matter-js';
import { Inventory, InventorySlot } from '../types/inventory';
import { Block } from '../types/block';
import { WET_SAND_MASS, WET_SAND_FRICTION, WET_SAND_RESTITUTION } from '../physics/constants';
import { SHORELINE_START_X, SHORELINE_END_X } from '../types/zones';

export interface BucketSlot extends InventorySlot {
  itemType: 'bucket';
  water: boolean;
}

export type FillResult =
  | { success: true; slotIndex: number }
  | { success: false; reason: 'not_in_shoreline' | 'no_bucket' | 'all_full' };

// Shimmer state per block (keyed by block.id)
const shimmerMap = new Map<string, number>(); // blockId → expiry timestamp (ms)
const SHIMMER_DURATION_MS = 1000; // ~60 frames at 60fps
```

### fillBucket

```typescript
export function fillBucket(
  inventory: Inventory,
  playerWorldX: number
): FillResult {
  // Zone check
  if (playerWorldX < SHORELINE_START_X || playerWorldX >= SHORELINE_END_X) {
    return { success: false, reason: 'not_in_shoreline' };
  }

  // Find first bucket slot
  const bucketSlotIndex = inventory.slots.findIndex(
    (s) => s.itemType === 'bucket'
  );
  if (bucketSlotIndex === -1) {
    return { success: false, reason: 'no_bucket' };
  }

  // Find first empty (water:false) bucket
  const emptyBucketIndex = inventory.slots.findIndex(
    (s) => s.itemType === 'bucket' && !(s as BucketSlot).water
  );
  if (emptyBucketIndex === -1) {
    return { success: false, reason: 'all_full' };
  }

  (inventory.slots[emptyBucketIndex] as BucketSlot).water = true;
  return { success: true, slotIndex: emptyBucketIndex };
}
```

### pourWater

```typescript
export function pourWater(
  block: Block,
  body: Matter.Body,
  inventory: Inventory
): boolean {
  // Find first filled bucket
  const filledIndex = inventory.slots.findIndex(
    (s) => s.itemType === 'bucket' && (s as BucketSlot).water === true
  );
  if (filledIndex === -1) return false;

  // Update block data
  block.isWet = true;

  // Update physics body properties
  Matter.Body.set(body, {
    mass: WET_SAND_MASS,
    friction: WET_SAND_FRICTION,
    restitution: WET_SAND_RESTITUTION,
    // Matter.js recalculates inertia when mass changes
  });
  // Also scale density so that mass/area ratio stays consistent
  Matter.Body.set(body, 'density', WET_SAND_MASS / (32 * 32));

  // Empty the bucket
  (inventory.slots[filledIndex] as BucketSlot).water = false;

  // Start shimmer
  shimmerMap.set(block.id, Date.now() + SHIMMER_DURATION_MS);

  return true;
}
```

### Shimmer visual effect
The shimmer is a diagonal highlight stripe that sweeps across the block in ~1 second. It is rendered by the caller (renderer.ts) by calling `updateBlockShimmer` after `renderBlock`:

```typescript
export function updateBlockShimmer(
  blockId: string,
  blockScreenX: number,
  blockScreenY: number,
  ctx: CanvasRenderingContext2D
): void {
  const expiry = shimmerMap.get(blockId);
  if (!expiry) return;

  const remaining = expiry - Date.now();
  if (remaining <= 0) {
    shimmerMap.delete(blockId);
    return;
  }

  const progress = 1 - remaining / SHIMMER_DURATION_MS; // 0→1 over duration
  // Diagonal stripe sweeps from top-left to bottom-right
  const stripeX = blockScreenX - 32 + progress * 80; // sweeps 80px total

  ctx.save();
  ctx.globalAlpha = 0.4 * (remaining / SHIMMER_DURATION_MS); // fades out
  ctx.fillStyle = '#a0d8ff'; // light blue shimmer
  // Draw a narrow diagonal parallelogram
  ctx.beginPath();
  ctx.moveTo(stripeX,      blockScreenY);
  ctx.lineTo(stripeX + 8,  blockScreenY);
  ctx.lineTo(stripeX + 8 + 32, blockScreenY + 32);
  ctx.lineTo(stripeX + 32, blockScreenY + 32);
  ctx.closePath();
  ctx.fill();
  ctx.restore();
}
```

## Key Constraints
- `fillBucket` only succeeds in SHORELINE zone; calling it from BEACH or CASTLE_ZONE returns `{ success: false, reason: 'not_in_shoreline' }`.
- A player may carry multiple bucket slots; only the first empty/filled one is used per call (FIFO).
- `Matter.Body.set` with `mass` also updates the body's `inertia` — do not set `inertia` manually.
- `pourWater` does not validate that the block is in CASTLE_ZONE; the caller (renderer.ts) must gate the interaction to that zone.
- The shimmer map may hold entries for blocks that have since been removed; clean up on block deletion.
- Do not call `pourWater` if `block.isWet` is already `true` — wetting a wet block wastes a bucket charge.

## Acceptance Criteria
- [ ] `fillBucket` returns `{ success: true }` when the player is in SHORELINE and has an empty bucket slot.
- [ ] `fillBucket` returns `{ success: false, reason: 'not_in_shoreline' }` when called from BEACH or CASTLE_ZONE.
- [ ] `fillBucket` returns `{ success: false, reason: 'all_full' }` when all bucket slots have `water:true`.
- [ ] `pourWater` sets `block.isWet = true` and empties the bucket slot on success.
- [ ] After `pourWater`, the Matter.js body's `mass` equals `WET_SAND_MASS` (verified via `body.mass`).
- [ ] The shimmer animation is visible for ~1 second after wetting and then disappears.
- [ ] `pourWater` returns `false` without throwing if no filled bucket is available.
- [ ] Multiple blocks can shimmer simultaneously (shimmerMap holds independent entries per block id).
