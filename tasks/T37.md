# T37 — [Phase 3] — Castle Tour Mode Logic

## Overview
This task implements the data-fetching and camera automation logic for the castle tour feature, which automatically pans through the top-10 highest-rated castle plots in a looping slideshow. It is the non-visual "engine" half of the tour system, decoupled from the UI overlay that T38 will add on top.

## Scope
Create `src/ui/tourMode.ts` (logic layer only; T38 will extend this same file with the UI overlay). Export `TourState` interface and a `TourController` class with `startTour()` and `stopTour()` methods. `startTour()` calls the Supabase `plot_avg_scores` view to fetch the top-10 plots (minimum 3 votes, ordered by `avg_score DESC`), stores them in `topPlots[]`, then begins the cycling loop: for each plot, a camera lerp tween smoothly moves the viewport's `cameraX` to the plot's center coordinate over 800ms, the view holds for 7 seconds, then advances `currentIndex`. After the 10th plot, the index wraps to 0. `stopTour()` clears the interval/timeout chain and leaves `cameraX` at its current position. The camera system from T04 is used for `cameraX` mutation; the score view from T31 provides the ranked data source.

### TourState Interface & Module Exports

```typescript
// src/ui/tourMode.ts

import { supabase } from '../supabase/client';
import { camera } from '../canvas/camera';   // T04 export

export interface TourPlot {
  plotId: string;
  ownerNickname: string;
  centerX: number;   // world-space X of plot center
  avgScore: number;  // 1.0–5.0
  voteCount: number;
}

export interface TourState {
  isActive: boolean;
  currentIndex: number;         // 0–9
  topPlots: TourPlot[];
  intervalId: ReturnType<typeof setTimeout> | null;
}

const HOLD_DURATION_MS  = 7000;   // time spent viewing each plot
const TWEEN_DURATION_MS = 800;    // camera lerp duration
const MIN_VOTES         = 3;      // minimum votes to qualify
const TOP_N             = 10;
```

### Fetching Top-10 Plots

```typescript
async function fetchTopPlots(): Promise<TourPlot[]> {
  const { data, error } = await supabase
    .from('plot_avg_scores')   // view from T31
    .select(`
      plot_id,
      avg_score,
      vote_count,
      plots!plot_id (
        x,
        width,
        owner_id,
        users!owner_id ( nickname )
      )
    `)
    .gte('vote_count', MIN_VOTES)
    .order('avg_score', { ascending: false })
    .limit(TOP_N);

  if (error || !data) return [];

  return data.map((row: any) => {
    const plot = row.plots;
    const centerX = plot.x + Math.floor(plot.width / 2);
    return {
      plotId: row.plot_id,
      ownerNickname: plot.users?.nickname ?? '알 수 없음',
      centerX,
      avgScore: parseFloat(row.avg_score),
      voteCount: row.vote_count,
    };
  });
}
```

The view is queried fresh each time `startTour()` is called so the rankings reflect recent votes. No caching is applied — the tour is an infrequent user action.

### Camera Lerp Tween

```typescript
function tweenCameraTo(targetX: number): Promise<void> {
  return new Promise((resolve) => {
    const startX = camera.x;
    const startTime = performance.now();

    function step(now: number) {
      const t = Math.min((now - startTime) / TWEEN_DURATION_MS, 1);
      // Smooth-step easing: 3t² - 2t³
      const eased = t * t * (3 - 2 * t);
      camera.x = startX + (targetX - startX) * eased;

      if (t < 1) {
        requestAnimationFrame(step);
      } else {
        camera.x = targetX;
        resolve();
      }
    }

    requestAnimationFrame(step);
  });
}
```

Using `Promise`-based tween makes sequencing (`await tweenCameraTo(...)`) clean within the async tour loop, avoiding callback nesting.

### TourController Class

```typescript
export class TourController {
  private state: TourState = {
    isActive: false,
    currentIndex: 0,
    topPlots: [],
    intervalId: null,
  };

  /** Returns current state snapshot (read-only intent). */
  getState(): Readonly<TourState> {
    return this.state;
  }

  async startTour(): Promise<void> {
    if (this.state.isActive) return;

    const plots = await fetchTopPlots();
    if (plots.length === 0) {
      // Not enough rated plots to start tour
      console.warn('[TourMode] 투어 가능한 성이 없습니다 (최소 투표 3개 필요).');
      return;
    }

    this.state.topPlots = plots;
    this.state.currentIndex = 0;
    this.state.isActive = true;

    this.runCycle();
  }

  stopTour(): void {
    this.state.isActive = false;
    if (this.state.intervalId !== null) {
      clearTimeout(this.state.intervalId);
      this.state.intervalId = null;
    }
    // cameraX stays at current position — no reset
  }

  private async runCycle(): Promise<void> {
    if (!this.state.isActive) return;

    const plot = this.state.topPlots[this.state.currentIndex];

    // Move camera to this plot's center
    await tweenCameraTo(plot.centerX);

    if (!this.state.isActive) return; // stopTour() may have been called during tween

    // Schedule advance after hold duration
    this.state.intervalId = setTimeout(() => {
      if (!this.state.isActive) return;
      this.state.currentIndex =
        (this.state.currentIndex + 1) % this.state.topPlots.length;
      this.runCycle();
    }, HOLD_DURATION_MS);
  }

  /** Advance to next plot manually (used by T38 UI controls). */
  skipToNext(): void {
    if (!this.state.isActive) return;
    if (this.state.intervalId !== null) clearTimeout(this.state.intervalId);
    this.state.currentIndex =
      (this.state.currentIndex + 1) % this.state.topPlots.length;
    this.runCycle();
  }

  /** Retreat to previous plot manually. */
  skipToPrev(): void {
    if (!this.state.isActive) return;
    if (this.state.intervalId !== null) clearTimeout(this.state.intervalId);
    this.state.currentIndex =
      (this.state.currentIndex - 1 + this.state.topPlots.length) %
      this.state.topPlots.length;
    this.runCycle();
  }
}

// Singleton instance shared by tourMode UI (T38) and HUD
export const tourController = new TourController();
```

## Key Constraints
- `startTour()` is a no-op if a tour is already active — calling it twice does not create two concurrent loops.
- `stopTour()` must clear the pending `setTimeout` immediately; if called mid-tween, the `isActive` guard in `runCycle` halts the loop at the next checkpoint.
- Plots with fewer than 3 votes are excluded to prevent unrated or low-sample plots from dominating the tour.
- If fewer than 1 qualifying plot exists, `startTour()` logs a warning and returns without activating the tour.
- The tween uses `performance.now()` (not `Date.now()`) for sub-millisecond frame timing accuracy.
- `topPlots` is not persisted to `localStorage` — it is always fetched fresh on `startTour()`.
- `skipToNext` / `skipToPrev` are intentionally exposed on the controller but not called by this task — they are hooks for T38's UI buttons.

## Acceptance Criteria
- [ ] `startTour()` fetches from `plot_avg_scores` and populates `topPlots` with ≤ 10 entries having `voteCount ≥ 3`.
- [ ] After calling `startTour()`, `camera.x` visibly lerps to the first plot's `centerX` over ~800ms.
- [ ] After 7 seconds on a plot, the camera automatically advances to the next plot.
- [ ] After the 10th plot, the tour loops back to index 0 without stopping.
- [ ] `stopTour()` halts the cycle and leaves `camera.x` at the position it was when called.
- [ ] Calling `startTour()` while already active is idempotent (no duplicate timers).
- [ ] If `fetchTopPlots()` returns an empty array, `startTour()` does not activate and logs a warning.
- [ ] `getState()` accurately reflects `isActive`, `currentIndex`, and `topPlots` at all times during the tour.
